# 12: 数据库与持久化综合练习

这一节把 `database/sql`、事务、迁移、repository、测试和文件持久化串起来。

---

## 练习1: 用户表迁移

写迁移 SQL：

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT NOT NULL UNIQUE,
    created_at TIMESTAMP NOT NULL,
    updated_at TIMESTAMP NOT NULL
);
```

要求：

1. 写 `Migrate(ctx, db) error`。
2. 创建 email 唯一索引。
3. 创建 created_at 普通索引。
4. 用测试验证迁移可以执行。

## 练习2: UserStore

实现：

```go
type UserStore struct {
    db DBTX
}

func (s *UserStore) Create(ctx context.Context, user User) (int64, error)
func (s *UserStore) FindByID(ctx context.Context, id int64) (User, error)
func (s *UserStore) List(ctx context.Context, limit, offset int) ([]User, error)
func (s *UserStore) UpdateEmail(ctx context.Context, id int64, email string) error
func (s *UserStore) Delete(ctx context.Context, id int64) error
```

要求：

1. 所有方法使用 context。
2. 所有 SQL 参数化。
3. `FindByID` 把 `sql.ErrNoRows` 转为 `ErrNotFound`。
4. `UpdateEmail` 和 `Delete` 检查影响行数。

## 练习8: JSON 文件持久化

实现：

```go
type Todo struct {
    ID    int64  `json:"id"`
    Title string `json:"title"`
    Done  bool   `json:"done"`
}

func SaveTodos(path string, todos []Todo) error
func LoadTodos(path string) ([]Todo, error)
```

要求：

1. 使用缩进 JSON。
2. 保存时使用原子写入。
3. 加载文件不存在时返回空列表还是错误，由你明确决定。
4. 用 `t.TempDir()` 测试。

## 验证建议

建议运行：

```bash
go test ./...
go test -race ./...
```

数据库练习要重点验证错误路径：唯一约束、找不到数据、事务回滚、非法分页参数、SQL 动态排序白名单。
