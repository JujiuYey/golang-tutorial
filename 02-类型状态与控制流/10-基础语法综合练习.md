## 练习1: 预测输出

> 考点:变量遮蔽、`:=` vs `=`(02-02 第 6-7 节)

**不要先运行**。先写下你认为的输出,再运行验证。

```go
package main

import "fmt"

func main() {
    count := 1

    if true {
        count := count + 1
        fmt.Println("inside:", count)
    }

    fmt.Println("outside:", count)
}
```

问题:

1. 这段代码能编译吗?
2. 如果能,输出是什么?
3. `if` 里面的 `count := count + 1` 创建了几个 `count`?

提示:`:=` 右边的 `count` 取值的时候,新的 `count` 存在了吗?

## 练习3: 百分比计算

> 考点:整数除法截断、显式类型转换、边界处理(02-04、02-06)

实现函数:

```go
func percent(part int, total int) float64
```

要求:

1. `total == 0` 时返回 `0`。
2. 正常情况下返回百分比。
3. `percent(1, 3)` 不能因为整数除法变成 `0`。

提示:`part / total * 100` 这个顺序哪一步就已经把结果砍成 0 了?转换要放在除法**之前**还是之后?

## 练习5: 简易计算器

> 考点:多返回值风格的前身(`值 + ok`)、switch 分发、浮点除零(02-04、02-09)

实现一个简易计算器函数:

```go
func calculate(a float64, op string, b float64) (float64, bool)
```

规则:

1. 支持 `+`、`-`、`*`、`/`。
2. 除数为 `0` 时返回 `0, false`。
3. 不支持的运算符返回 `0, false`。
4. 正常计算返回结果和 `true`。

示例:

```go
result, ok := calculate(10, "/", 2)
if !ok {
    fmt.Println("invalid calculation")
    return
}
fmt.Println(result)
```

提示:这里第二个返回值 `bool` 表示"成没成"——这正是 03 模块 `(value, error)` 模式的简化版,先在这里体会一下"结果和状态一起返回"的手感。

## 练习6: 订单状态

> 考点:const 块、iota 起始值控制(02-03)

使用 `const` 和 `iota` 定义订单状态:

- 待支付
- 已支付
- 已发货
- 已取消

要求:

1. 状态从 `1` 开始。
2. 写一个 `statusText(status int) string` 函数返回中文状态。
3. 非法状态返回 `"未知状态"`。

提示:iota 默认从 0 开始,想从 1 开始有两种常见写法——`iota + 1`,或者第一行用 `_` 占掉 0。任选其一,想想两种各自的含义。
