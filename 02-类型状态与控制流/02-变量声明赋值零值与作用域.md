# 02: 变量声明、赋值、零值与作用域

这篇解决两个问题：Go 的几种变量声明写法分别什么时候用；一个变量到底"活"在哪个范围里。学完你能避开两个高频坑——`:=` 和 `=` 混用、内层变量悄悄遮住外层变量。

先对齐一下心智模型：JS 里你主要用 `let`/`const`，Go 里对应的是 `var` 和 `:=`。区别在于 Go 的变量声明后**必有值、必有类型、必须被用到**——三个"必须"都由编译器盯着。

---

## 1. var 声明

`var` 是最基础的声明方式，大致对应 JS 的 `let`：

```go
var name string
var age int
var ok bool
```

可以声明时赋值：

```go
var name string = "Go"
var age int = 18
```

也可以省掉类型，让编译器推断（上一篇讲过的类型推断）：

```go
var name = "Go"
var age = 18
```

批量声明常用于包级变量或一组相关变量：

```go
var (
    appName = "demo"
    version = "1.0.0"
    debug   = true
)
```

---

## 2. 短变量声明

函数内部有更顺手的写法——`:=`，声明加赋值一步到位：

```go
func main() {
    name := "Go"
    age := 18

    fmt.Println(name, age)
}
```

```text
Go 18
```

### 🕳️ 坑：`:=` 只能在函数内部用

包级变量（函数外面）只能用 `var`：

```go
package main

name := "Go" // 函数外使用 :=
```

```text
编译错误：syntax error: non-declaration statement outside function body
```

**一句话总结：函数里用 `:=`，函数外用 `var`。**

---

## 3. 赋值与多变量赋值

声明之后再修改，用 `=`（注意没有冒号）：

```go
func main() {
    count := 1
    count = 2
    fmt.Println(count)
}
```

```text
2
```

记忆锚点：**`:=` 是"生孩子"（创建新变量），`=` 是"改名字"（修改已有变量）**。JS 里 `let` 和 `=` 分工也是如此，只是 Go 把"生"压缩成了一个符号。

Go 支持多变量同时赋值，交换两个值不需要临时变量：

```go
func main() {
    a, b := 1, 2
    a, b = b, a
    fmt.Println(a, b)
}
```

```text
2 1
```

眼熟吗？JS 的解构交换 `[a, b] = [b, a]` 干的是同一件事，Go 连方括号都不用写。

---

## 4. 短变量声明的重复规则

`:=` 左边**至少要有一个新变量**，才允许旧变量混在里面：

```go
func main() {
    name := "Go"
    age := 18

    name, city := "Gopher", "Beijing" // city 是新变量，所以合法
    fmt.Println(name, age, city)
}
```

如果左边全是旧变量，编译器认为你在"重复生同一个孩子"：

```go
func main() {
    name := "Go"
    name := "Gopher" // 左边没有任何新变量
    fmt.Println(name)
}
```

```text
编译错误：no new variables on left side of :=
```

改成 `name = "Gopher"`（纯赋值）就行。这条规则在后面写 `n, err := ...` 连续处理错误时会经常打交道。

---

## 5. 零值

JS 里没赋值的变量是 `undefined`，用它做运算随时炸出 `NaN`。Go 没有 `undefined` 这个概念：**变量声明后一定有值**。没显式赋值时，编译器自动填入该类型的零值：

| 类型 | 零值 |
|------|------|
| `int` / `int64` / `float64` | `0` |
| `bool` | `false` |
| `string` | `""` |
| 指针、slice、map、channel、function | `nil` |

```go
package main

import "fmt"

func main() {
    var count int
    var name string
    var ok bool

    fmt.Printf("count=%d name=%q ok=%v\n", count, name, ok)
}
```

```text
count=0 name="" ok=false
```

零值是 Go 设计里的重要概念，很多类型会刻意设计成"零值直接可用"。**一句话总结：Go 变量出生即有值，不存在 `undefined`。**

---

## 6. 作用域

作用域决定变量在哪里可见。Go 和现代 JS（`let`/`const`）一样是**块级作用域**——`{}` 就是边界：

```go
package main

import "fmt"

var global = "global"

func main() {
    local := "local"

    if ok := true; ok {
        msg := "inside if"
        fmt.Println(global, local, msg)
    }

    // fmt.Println(ok)  // 编译错误：undefined: ok
    // fmt.Println(msg) // 编译错误：undefined: msg
}
```

```text
global local inside if
```

### 看不懂 `if ok := true; ok` ？拆开看

这一行用了 Go 特有的 **if 初始化语句** 语法（02-07 会正式讲），结构是：

```text
if <初始化语句>; <条件> {
   └─ ok := true   └─ ok（必须是 bool）
```

分号把 if 拆成两部分：**先执行分号前的声明，再判断分号后的条件**。它完全等价于下面这个普通写法：

```go
ok := true
if ok {
    // ...
}
```

区别只有一个：**变量 `ok` 的作用域不同**。

- 普通写法：`ok` 声明在 `main` 里，if 结束后依然存在
- if 初始化写法：`ok` 只活在这条 if 语句里（包括 else 分支），if 结束就消失

所以上面例子里 `fmt.Println(ok)` 会编译错误——`ok` 出了 if 就不存在了。这正是本节要演示的作用域层级：

| 变量 | 声明位置 | 能见范围 |
|------|----------|----------|
| `global` | 函数外（包级） | 整个包 |
| `local` | `main` 函数内 | `main` 全函数 |
| `ok` | if 初始化语句 | 仅这条 if 语句 |
| `msg` | if 代码块内 | 仅 `{}` 内 |

这个语法的价值在 02-07 会展开：Go 的错误处理惯用法 `if err := doSomething(); err != nil { ... }` 就是它——把 `err` 锁在 if 里，不污染外层作用域。

## 7. 变量遮蔽

内层作用域可以声明和外层**同名**的变量，内层的会把外层的"挡住"，这叫变量遮蔽（shadowing）。JS 的 `let` 也有一模一样的行为，但 Go 里它更容易被误触发——因为 `:=` 太顺手了。

```go
package main

import "fmt"

func main() {
    count := 10

    if true {
        count := 20
        fmt.Println("inside:", count)
    }

    fmt.Println("outside:", count)
}
```

```text
inside: 20
outside: 10
```

### 🕳️ 坑：想改外层变量，却手滑写了 `:=`

以为会怎样：if 里把 `count` 改成了 20，出来还是 20。
实际怎样：外层 `count` 纹丝不动，还是 10。
为什么：`:=` 在 if 块里**新建**了一个同名的 `count`，改的是新的那个；出了 `{}` 它就销毁了。

想修改外层变量，用 `=`：

```go
func main() {
    count := 10

    if true {
        count = 20 // 没有冒号，改的是外层的 count
    }

    fmt.Println(count) // 20
}
```

**一句话总结：块里用 `:=` 是新建"影子"，用 `=` 才是修改本尊。**

---

## 8. 未使用变量

JS 里声明了不用的变量顶多被 ESLint 警告一下。Go 直接把它升级成编译错误——声明了就必须用：

```go
func main() {
    name := "Go"
}
```

```text
编译错误：declared and not used: name
```

临时想忽略某个值，用空白标识符 `_`（相当于一个"垃圾桶"变量）：

```go
func main() {
    name := "Go"
    _ = name
}
```

`_` 要谨慎用。它能让代码通过编译，但也可能掩盖你本该处理的值（尤其是 error）。

---

## 本篇重点

- [ ] 函数内用 `:=`（声明+赋值），函数外只能 `var`；修改已有变量用 `=`。
- [ ] `:=` 左边必须至少有一个新变量，否则编译错误 `no new variables on left side of :=`。
- [ ] Go 没有 `undefined`：变量声明即有零值（`0`、`false`、`""`、`nil`）。
- [ ] 作用域以 `{}` 为界；if 初始化语句声明的变量只活在这条 if 里。
- [ ] 块内 `:=` 会遮蔽外层同名变量——想改外层变量用 `=`，这是高频坑。
