# 04: 数字类型与运算

数字类型看起来简单，但 Go 对数字类型的边界、转换和运算规则很严格。

---

## 1. 整数类型

有符号整数：

```go
var a int = 10
var b int8 = 127
var c int16 = 32767
var d int32 = 2147483647
var e int64 = 9223372036854775807
```

无符号整数：

```go
var a uint = 10
var b uint8 = 255
var c uint16 = 65535
var d uint32 = 4294967295
var e uint64 = 18446744073709551615
```

`int` 和 `uint` 的大小依赖平台，通常在 64 位机器上是 64 位。写业务代码时一般用 `int`，和外部协议、数据库、二进制格式对齐时再使用明确位数的类型。

---

## 2. byte、rune、uintptr

```go
var b byte = 255 // byte 是 uint8 的别名
var r rune = '你' // rune 是 int32 的别名
```

`uintptr` 用于保存指针的整数表示，普通业务代码很少使用。

---

## 3. 整数溢出

超出类型范围会溢出或编译失败，取决于上下文。

```go
func main() {
    var n uint8 = 255
    n++
    fmt.Println(n) // 0
}
```

`uint8` 最大是 `255`，再加 `1` 会回到 `0`。

常量赋值超出范围会直接编译失败：

```go
// var n uint8 = 256 // 编译错误
```

---

## 4. 浮点数

Go 常用 `float64`。

```go
func main() {
    price := 19.9
    fmt.Printf("%T\n", price) // float64
}
```

浮点数不能精确表示所有小数：

```go
func main() {
    fmt.Println(0.1 + 0.2)
}
```

输出可能不是你直觉里的精确 `0.3`。涉及金额时，不要直接用浮点数保存“分”这种精确单位，可以用整数表示最小货币单位。

---

## 5. 复数

Go 支持复数：

```go
func main() {
    z := 3 + 4i
    fmt.Println(real(z))
    fmt.Println(imag(z))
}
```

日常业务代码很少使用复数，知道 Go 支持即可。

---

## 6. 算术运算

```go
func main() {
    a, b := 10, 3

    fmt.Println(a + b) // 13
    fmt.Println(a - b) // 7
    fmt.Println(a * b) // 30
    fmt.Println(a / b) // 3，整数除法
    fmt.Println(a % b) // 1
}
```

整数除法会丢掉小数部分。需要小数结果时，先转换类型：

```go
func main() {
    a, b := 10, 3
    fmt.Println(float64(a) / float64(b))
}
```

---

## 7. 自增和自减

Go 支持 `++` 和 `--`，但它们只能作为语句，不能作为表达式。

```go
func main() {
    n := 1
    n++
    n--
    fmt.Println(n)
}
```

下面这种写法在 Go 中不合法：

```go
// x := n++ // 编译错误
```

---

## 8. 位运算

```go
func main() {
    a, b := 5, 3 // 5=101, 3=011

    fmt.Println(a & b)  // 1
    fmt.Println(a | b)  // 7
    fmt.Println(a ^ b)  // 6
    fmt.Println(a << 1) // 10
    fmt.Println(a >> 1) // 2
}
```

位运算常见于权限标志、二进制协议、性能敏感的底层代码。

---

## 练习

写一个百分比计算函数：

```go
func percent(part int, total int) float64
```

要求：

1. `total == 0` 时返回 `0`。
2. 正常情况下返回 `part / total * 100` 的浮点结果。
3. 测试 `percent(1, 3)`，确认没有被整数除法截断。
