# 01: 值、类型与字面量

这篇解决一个问题：从 JS 的"变量随便装"切换到 Go 的"每个值生下来就有类型"。学完你能看懂 Go 的类型报错，也知道 `123`、`3.14`、`'你'` 这些写在代码里的值在 Go 眼里分别是什么。

先说和 JS 最大的差别：JS 的变量像贴纸，今天贴在数字上、明天撕下来贴到字符串上都行；Go 的变量像模具，浇进去是什么类型，一辈子就是什么类型。类型对不上，程序压根跑不起来——编译器在运行前就拦下（静态类型检查）。

---

## 1. 值与类型

值是程序处理的数据，类型决定这个值能参与哪些操作（能不能相加、能不能比较、占多少内存）。

想知道一个值的类型，用 `%T`（Type）打出来看——初学阶段这是你最好的朋友：

```go
package main

import "fmt"

func main() {
    age := 18
    name := "Go"
    active := true

    fmt.Printf("%T %v\n", age, age)
    fmt.Printf("%T %v\n", name, name)
    fmt.Printf("%T %v\n", active, active)
}
```

```text
int 18
string Go
bool true
```

`%T` 打印类型，`%v` 打印值。**对类型没把握时别猜，`%T` 打出来看。**

---

## 2. 基本类型概览

| 类型 | 说明 | 示例 |
|------|------|------|
| `bool` | 布尔值，只能是 `true` 或 `false` | `ok := true` |
| `string` | 字符串 | `name := "golang"` |
| `int` | 常用整数，位数依赖平台 | `count := 10` |
| `int8` / `int16` / `int32` / `int64` | 指定位数有符号整数 | `var id int64 = 1001` |
| `uint` / `uint8` / `uint16` / `uint32` / `uint64` | 无符号整数 | `var n uint = 10` |
| `byte` | `uint8` 的别名，常表示字节 | `var b byte = 'A'` |
| `rune` | `int32` 的别名，常表示 Unicode 码点 | `var r rune = '你'` |
| `float32` / `float64` | 浮点数 | `price := 19.9` |
| `complex64` / `complex128` | 复数 | `z := 1 + 2i` |

对比 JS：JS 的数字只有一个 `number`（外加 `bigint`），Go 把数字按"有没有符号、占几位、是不是小数"切成了十几种。不用慌——**日常代码里 `bool`、`string`、`int`、`int64`、`float64`、`byte`、`rune` 这几个就够用**，其余用到再查。

---

## 3. 字面量

字面量就是直接写在代码里、肉眼可见的值（literal）——`123`、`3.14`、`"hello"` 这种。

```go
func main() {
    fmt.Println(123)     // 整数字面量
    fmt.Println(3.14)    // 浮点数字面量
    fmt.Println("hello") // 字符串字面量
    fmt.Println('你')    // rune 字面量
    fmt.Println(true)    // 布尔字面量
}
```

```text
123
3.14
hello
20320
true
```

### 🕳️ 坑：`'你'` 打印出来是 20320，不是"你"

JS 里 `'你'` 和 `"你"` 都是字符串，单双引号随便换。**Go 里单引号是另一种东西**：`'你'` 是 rune 字面量——这个字符的 Unicode 码点，本质是个数字，所以打印出来是 `20320`。想要字符串，必须用双引号 `"你"`。

字面量本身可以是"无类型"的，只有在赋给变量、参与表达式、传给函数时，编译器才给它定下**默认类型**：

| 字面量 | 默认类型 |
|--------|----------|
| `123` | `int` |
| `3.14` | `float64` |
| `"hello"` | `string` |
| `'你'` | `rune`（就是 `int32`） |
| `true` | `bool` |

## 4. 类型推断

`:=` 会让编译器根据右边的值推断类型。验证一下上面那张表：

```go
func main() {
    i := 123
    f := 3.14
    r := '你'

    fmt.Printf("%T\n", i)
    fmt.Printf("%T\n", f)
    fmt.Printf("%T\n", r)
}
```

```text
int
float64
int32
```

### 🕳️ 坑：类型推断 ≠ 动态类型

你可能以为 `:=` 让 Go 变得和 JS 一样灵活——不是的。推断只发生在声明那一刻，**定下来就焊死了**。JS 里 `let n = 10; n = "hello"` 毫无问题，Go 里同样的操作直接编译失败：

```go
func main() {
    n := 10
    n = "hello"
}
```

```text
编译错误：cannot use "hello" (untyped string constant) as int value in assignment
```

**一句话总结：`:=` 帮你省的是"写类型名"的功夫，不是"守类型"的规矩。**

---

## 5. 类型决定可用操作

同一个运算符，不同类型待遇不同。数字能加减，字符串只能"加"（拼接）：

```go
func main() {
    fmt.Println(1 + 2)
    fmt.Println("Go" + "lang")
}
```

```text
3
Golang
```

字符串做减法？编译器直接拒绝：

```go
fmt.Println("Go" - "lang")
```

```text
编译错误：invalid operation: operator - not defined on "Go" (untyped string constant)
```

对比 JS：`"Go" - "lang"` 在 JS 里会安静地算出 `NaN`，等你运行时踩雷；Go 在编译期就把这类错误拍死。

**一句话总结：类型不是负担，是编译器替你提前排雷的依据。**

---

## 本篇重点

- [ ] Go 是静态类型：变量类型在声明时确定，之后不能换——`:=` 是省写类型名，不是动态类型。
- [ ] 对类型没把握时用 `fmt.Printf("%T", x)` 打出来验证，不要靠猜。
- [ ] 单引号 `'你'` 是 rune（码点数字），双引号 `"你"` 才是字符串——和 JS 完全不同。
- [ ] 字面量有默认类型：整数 → `int`，小数 → `float64`，字符 → `rune`。
- [ ] 类型决定可用操作，非法操作（如字符串减法）在编译期就会报错，不会拖到运行时。

## 练习

先判断每个变量的类型，再运行验证。

```go
package main

import "fmt"

func main() {
    a := 1
    b := 1.0
    c := "1"
    d := '1'
    e := true

    fmt.Printf("a: %T %v\n", a, a)
    fmt.Printf("b: %T %v\n", b, b)
    fmt.Printf("c: %T %v\n", c, c)
    fmt.Printf("d: %T %v\n", d, d)
    fmt.Printf("e: %T %v\n", e, e)
}
```

问题：

1. `a` 和 `b` 的类型一样吗？
2. `c` 和 `d` 的类型一样吗？
3. 为什么 `'1'` 打印出来的值不是字符串 `"1"`？

提示：回看第 3 节的"坑"，想想 `'1'` 这个字符的码点是多少。
