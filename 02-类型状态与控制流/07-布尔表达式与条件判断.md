# 07: 布尔表达式与条件判断

这篇解决的问题：把 JS 的 truthy/falsy 直觉彻底关掉——Go 的 `if` 条件**只认 `bool`**，`if (n)`、`if (str)` 这类写法一律编译不过。学完你还会拿到 Go 最高频的惯用法：`if err := ...; err != nil`。

---

## 1. 关系运算符

六个比较运算符，结果都是 `bool`：

```go
func main() {
    a, b := 10, 20

    fmt.Println(a == b)
    fmt.Println(a != b)
    fmt.Println(a > b)
    fmt.Println(a < b)
    fmt.Println(a >= b)
    fmt.Println(a <= b)
}
```

```text
false
true
false
true
false
true
```

对比 JS：Go 只有 `==`，没有 `===`——因为没有隐式转换（上一篇），不同类型的值压根不允许比较，自然不需要一个"更严格的等于"。**Go 的 `==` 天生就是 JS 的 `===`。**

---

## 2. 逻辑运算符

`&&`（都满足）、`||`（满足其一）、`!`（取反），写法和 JS 相同：

```go
func main() {
    x, y := true, false

    fmt.Println(x && y)
    fmt.Println(x || y)
    fmt.Println(!x)
}
```

```text
false
true
false
```

短路求值也和 JS 一样：`&&` 左边为 false 就不看右边，`||` 左边为 true 就不看右边。但注意——JS 里 `a || defaultValue` 那种"取默认值"的写法在 Go 行不通：Go 的 `&&`/`||` 只吃 `bool`、只吐 `bool`，不会返回操作数本身。

---

## 3. if 条件必须是 bool

### 🕳️ 坑：没有 truthy / falsy

以为会怎样：`if n` 在 n 非零时执行（JS 直觉）。
实际怎样：编译错误。
为什么：Go 的 if 条件位置只接受 `bool` 类型，数字、字符串、nil 都没有"自动转布尔"一说。

```go
func main() {
    n := 1
    if n {
        fmt.Println("ok")
    }
}
```

```text
编译错误：non-boolean condition in if statement
```

必须把意图写明白：

```go
if n != 0 {
    fmt.Println("ok")
}
```

这其实是福利：JS 里 `if (arr.length)`、`if (str)` 这些简写埋着 `0`、`""`、`NaN` 的歧义（空字符串是没有值还是值恰好为空？），Go 强迫你写 `n != 0`、`s != ""`，条件里没有任何脑筋急转弯。

顺带：Go 的 if 条件**不需要圆括号**，但花括号**必须有**——单行 if 不能省 `{}`。

---

## 4. if / else if / else

结构和 JS 一致：

```go
func grade(score int) string {
    if score >= 90 {
        return "A"
    } else if score >= 80 {
        return "B"
    } else if score >= 60 {
        return "C"
    }
    return "D"
}

func main() {
    fmt.Println(grade(95), grade(83), grade(61), grade(30))
}
```

```text
A B C D
```

### 🕳️ 坑：`else` 必须紧跟在 `}` 后面，不能换行

```go
if score >= 60 {
    fmt.Println("pass")
} else {              // ✅ else 和 } 同行
    fmt.Println("fail")
}

// } 
// else {            // ❌ 编译错误
```

原因：Go 编译器会在行尾自动补分号，`}` 独占一行后面就被补了分号，if 语句被认为已结束。JS 里这只是风格问题，Go 里是硬性语法。

---

## 5. if 初始化语句

`if` 可以带初始化语句，变量只在这个 `if` 结构内有效。

```go
func main() {
    if n := len("golang"); n > 3 {
        fmt.Println(n)
    }

    // fmt.Println(n) // 编译错误：undefined: n
}
```

```text
6
```

### 记忆锚点：这就是 for 循环的第一段

觉得这个写法陌生？其实你写了无数遍——**JS 的 for 循环开头就是它**：

```js
for (let i = 0; i < n; i++) { ... }
//   └─ 声明 ─┘  └─ 条件 ─┘        i 只活在循环里
```

Go 只是把"分号前先声明、分号后做判断、变量只活在结构内"这套规则**从 for 搬到了 if**：

```go
if n := len("golang"); n > 3 { ... }
// └─── 声明 ────────┘ └─ 条件 ┘     n 只活在 if 里
```

所以看到 `if xxx; yyy {` 不用慌，在心里把分号拆成两行读：

```go
n := len("golang")   // 分号前：先执行这个
if n > 3 { ... }     // 分号后：再判断这个
```

### 写的时候记不住怎么办

不用硬记。**先写两行普通写法，再"收"进 if**：

```go
// 第一步：怎么顺手怎么写
err := doSomething()
if err != nil { ... }

// 第二步：把第一行搬进 if，用分号接起来
if err := doSomething(); err != nil { ... }
```

这两种写法都是合法的 Go，功能一样，唯一区别是变量出了 if 还在不在。写成第二种只是更地道（Go 工具链的 linter 有时会建议你收进去）。写多了 err-handling（下一节），手自然就熟了——这不是需要背的语法，是会长进肌肉记忆的高频动作。

---

## 6. if 初始化语句的最常见应用：err-handling 模式

第 5 节用 `len("golang")` 演示了 if 初始化语句的语法。在真实 Go 代码里，**这个特性几乎全部用在 error handling 上**。

### 语法拆解

```go
if err := doSomething(); err != nil {
    return err
}
```

三部分：

- `err := doSomething()` —— 初始化（声明 + 赋值）
- `err != nil` —— 条件
- `;` —— 分隔初始化和条件

### 核心好处：作用域限制

`err` 只在 `if/else` 块里活，出 `{}` 就消失。**这跟函数参数一样——用完即弃，不污染外层 scope。**

所以 Go 代码里 `if err != nil` 满屏飞是正常的。Go 函数通常返回 `(value, err)`，调用方立刻用 `if err != nil` 处理掉。

### 示例

```go
package main

import (
    "fmt"
    "strconv"
)

func main() {
    if n, err := strconv.Atoi("42"); err == nil {
        fmt.Println("parsed:", n)
    } else {
        fmt.Println("error:", err)
    }
}
```

```text
parsed: 42
```

## 7. 拆分复杂条件

条件一长就容易写反。技巧：把每个子条件**起个名字**存进变量，让 if 读起来像句子：

```go
func main() {
    score := 86
    attendance := 0.9

    passedScore := score >= 60
    passedAttendance := attendance >= 0.8

    if passedScore && passedAttendance {
        fmt.Println("pass")
    }
}
```

```text
pass
```

`if passedScore && passedAttendance` 一眼能读出业务含义，也方便调试时单独打印某个子条件。**一句话总结：条件超过两个子句，就值得拆出来命名。**

---

## 本篇重点

- [ ] Go 没有 truthy/falsy：if 条件必须是 `bool`，`if n`、`if s` 直接编译错误，写成 `n != 0`、`s != ""`。
- [ ] Go 的 `==` 就是 JS 的 `===`（没有隐式转换，所以不需要三等号）。
- [ ] if 条件不用圆括号，花括号必须有；`else` 必须和 `}` 同行。
- [ ] `if x := ...; 条件 {}` = JS for 循环第一段搬到 if：先执行、再判断，变量只活在 if 里。
- [ ] 它的头号用途是 `if err := doSomething(); err != nil {}`——Go 最高频的惯用法。

## 练习

实现一个函数：

```go
func canRegister(age int, agreedTerms bool) bool
```

要求：

1. 年龄必须大于等于 `18`。
2. 必须同意条款。
3. 两个条件都满足才返回 `true`。

提示：`agreedTerms` 本身就是 bool，不需要写 `agreedTerms == true`——直接参与 `&&` 即可。整个函数体可以只有一行 return。
