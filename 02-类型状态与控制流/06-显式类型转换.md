# 06: 显式类型转换

Go 不会偷偷帮你转换类型。需要转换时，必须明确写出来。

---

## 1. 数字类型转换

不同数字类型不能直接混算：

```go
package main

import "fmt"

func main() {
    var a int = 10
    var b int64 = 20

    // fmt.Println(a + b) // 编译错误
    fmt.Println(int64(a) + b)
}
```

显式转换让你自己承担“转换是否合理”的责任。

---

## 2. 转换可能丢失信息

```go
func main() {
    var n int64 = 300
    var b byte = byte(n)

    fmt.Println(b)
}
```

`byte` 只能表示 `0-255`，转换可能得到你意料之外的值。类型转换不是安全检查。

---

## 3. 浮点数和整数转换

```go
func main() {
    f := 3.9
    n := int(f)

    fmt.Println(n) // 3
}
```

浮点数转整数会截断小数部分。

---

## 4. 数字和字符串转换

数字和字符串之间不能用普通类型转换表达“十进制文本转换”，要用 `strconv`。

```go
package main

import (
    "fmt"
    "strconv"
)

func main() {
    n := 123
    s := strconv.Itoa(n)
    fmt.Printf("%T %q\n", s, s)

    parsed, err := strconv.Atoi("456")
    if err != nil {
        fmt.Println("parse error:", err)
        return
    }
    fmt.Printf("%T %v\n", parsed, parsed)
}
```

`string(65)` 的含义不是把数字 `65` 变成字符串 `"65"`，而是把 Unicode 码点 65 变成字符 `"A"`。

```go
func main() {
    fmt.Println(string(65))       // A
    fmt.Println(strconv.Itoa(65)) // 65
}
```

---

## 5. bool 不能和数字互转

Go 不允许把布尔值当数字用。

```go
func main() {
    ok := true
    // n := int(ok) // 编译错误
    _ = ok
}
```

如果需要根据布尔值得到数字，要显式写条件：

```go
func boolToInt(ok bool) int {
    if ok {
        return 1
    }
    return 0
}
```

## 练习

下面的代码不能编译。请修正它，并解释为什么必须显式转换。

```go
package main

import "fmt"

func main() {
    var count int = 10
    var total int64 = 100

    fmt.Println(count + total)
}
```

要求：

1. 保留 `count` 和 `total` 的原始类型。
2. 输出正确的相加结果。
3. 解释为什么 Go 不允许直接相加。
