# 12: 并发模型综合练习

这一节把 goroutine、channel、select、context、sync 和 race detector 串起来。每道题都要说明 goroutine 如何退出、channel 由谁关闭、共享状态如何保护。

---

## 练习1: 并发平方

实现：

```go
func Squares(nums []int) []int
```

要求：

1. 为每个数字启动一个 goroutine。
2. 使用 `WaitGroup` 等待全部完成。
3. 返回结果顺序和输入顺序一致。
4. 不产生数据竞争。

提示：可以预先创建结果 slice，每个 goroutine 只写自己的下标。

---

## 练习2: 并发求和

实现：

```go
func ParallelSum(nums []int, parts int) int
```

要求：

1. 把 nums 切成 `parts` 段。
2. 每段一个 goroutine 求局部和。
3. 用 channel 汇总局部结果。
4. 处理 `parts <= 0` 和空 slice。

---

## 练习3: 可取消的 producer

实现：

```go
func Produce(ctx context.Context, n int) <-chan int
```

要求：

1. 依次产生 `0` 到 `n-1`。
2. 返回的 channel 最终会关闭。
3. 如果 ctx 取消，producer 能退出。
4. 发送时监听 `ctx.Done()`。

---

## 练习4: 带超时的接收

实现：

```go
func ReceiveWithTimeout[T any](ch <-chan T, timeout time.Duration) (T, bool)
```

要求：

1. 超时前收到值，返回值和 `true`。
2. 超时返回零值和 `false`。
3. channel 关闭时返回零值和 `false`。
4. 使用 `select`。

## 练习5: 安全计数器

实现：

```go
type Counter struct {
    // 自己设计字段
}

func (c *Counter) Inc()
func (c *Counter) Add(n int)
func (c *Counter) Value() int
```

要求：

1. 使用 `sync.Mutex`。
2. 并发调用 `Inc` 1000 次，最终结果正确。
3. 用 `go test -race` 验证没有数据竞争。

---

## 练习6: 并发安全缓存

实现：

```go
type Cache struct {
    // 自己设计字段
}

func (c *Cache) Set(key, value string)
func (c *Cache) Get(key string) (string, bool)
func (c *Cache) Delete(key string)
```

要求：

1. 使用 `sync.RWMutex` 保护 map。
2. 零值可用，或者提供构造函数并说明原因。
3. 并发读写时通过 `go test -race`。

---

## 练习7: worker pool

实现：

```go
func RunJobs(ctx context.Context, jobs []Job, workerCount int) []Result
```

要求：

1. 最多同时运行 `workerCount` 个 worker。
2. 支持 ctx 取消。
3. 每个 job 都产生一个 result，除非 ctx 提前取消。
4. 不向已关闭 channel 发送。
5. 说明结果顺序是否稳定。

---

## 练习8: fan-in

实现：

```go
func Merge[T any](ctx context.Context, inputs ...<-chan T) <-chan T
```

要求：

1. 把多个输入 channel 合并成一个输出 channel。
2. 所有输入关闭后，输出 channel 关闭。
3. ctx 取消时，所有转发 goroutine 能退出。
4. 发送到输出 channel 时监听 ctx。

---

## 练习9: 找出并发 bug

阅读下面代码，列出问题并修复：

```go
func Bad(urls []string) []string {
    var results []string
    var wg sync.WaitGroup

    for _, url := range urls {
        wg.Add(1)
        go func() {
            defer wg.Done()
            results = append(results, fetch(url))
        }()
    }

    wg.Wait()
    return results
}
```

至少考虑：

1. 循环变量捕获。
2. 多 goroutine append 同一个 slice。
3. 结果顺序。
4. fetch 失败或超时。

## 练习10: 验证并发代码

为前面的练习写测试：

```bash
go test ./...
go test -race ./...
```

要求：

1. 测试正常路径。
2. 测试取消路径。
3. 测试空输入。
4. 测试并发读写。
5. 能解释 race detector 的报告。

---

## 验证建议

并发练习不要只看输出。建议每个练习都至少做三件事：

1. 写单元测试。
2. 运行 `go test -race ./...`。
3. 人工检查 goroutine 生命周期和 channel 关闭责任。
