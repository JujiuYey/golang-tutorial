# 01: 标准库学习方法与文档阅读

标准库是 Go 的重要组成部分。很多 Go 程序之所以简单，是因为标准库已经提供了稳定、清晰、可组合的基础能力。

---

## 1. 标准库的价值

标准库覆盖了很多高频任务：

- 文件和目录：`os`、`io`、`bufio`、`path/filepath`
- 编码：`encoding/json`、`encoding/csv`、`encoding/base64`
- 文本处理：`strings`、`strconv`、`bytes`、`unicode`
- 时间：`time`
- 命令行：`flag`、`os`
- 日志：`log`、`log/slog`
- 测试：`testing`、`httptest`
- 并发：`sync`、`context`
- 网络：`net`、`net/http`

本章先讲最常用于日常项目和 CLI 工具的部分。网络和 HTTP 会在后面章节展开。

---

## 2. 先看包级文档

每个标准库包都有包级说明。阅读顺序建议是：

1. 包解决什么问题。
2. 最小示例怎么写。
3. 核心类型有哪些。
4. 常用函数的错误语义是什么。

例如看到 `io` 包，先抓住两个核心接口：

```go
type Reader interface {
    Read(p []byte) (n int, err error)
}

type Writer interface {
    Write(p []byte) (n int, err error)
}
```

理解这两个接口，很多标准库 API 都会变得顺。

---

## 3. 看函数签名

函数签名会告诉你很多信息：

```go
func ReadFile(name string) ([]byte, error)
```

`os.ReadFile` 读取整个文件，返回字节切片和错误。

```go
func Copy(dst Writer, src Reader) (written int64, err error)
```

`io.Copy` 从 `Reader` 拷贝到 `Writer`，不关心具体是文件、网络连接、内存 buffer 还是压缩流。

读签名时要问：

- 参数是具体类型还是接口。
- 返回值有没有 error。
- 调用方是否需要关闭资源。
- 数据是一次性读完，还是流式处理。

---

## 4. 标准库大量使用接口

很多包通过小接口组合能力：

```go
type ReadCloser interface {
    Reader
    Closer
}
```

这就是 Go 标准库的风格：接口小，组合灵活。

如果一个函数接收 `io.Reader`，你可以传入：

- 文件：`*os.File`
- 字符串 reader：`strings.NewReader`
- bytes reader：`bytes.NewReader`
- HTTP 响应体
- 压缩流 reader

掌握这些接口，会让你少写很多重复代码。

---

## 5. 错误处理不能省

标准库函数大量返回 error。

```go
data, err := os.ReadFile("config.json")
if err != nil {
    return err
}
```

教学示例里偶尔会省略错误处理，但你自己写代码时不要养成忽略 error 的习惯。

错误处理要做到：

- 该返回就返回。
- 需要上下文时用 `%w` 包装。
- 判断特定错误时用 `errors.Is` 或 `errors.As`。

---

## 6. 资源要关闭

打开文件、网络连接、响应体后，通常需要关闭。

```go
f, err := os.Open("data.txt")
if err != nil {
    return err
}
defer f.Close()
```

只有资源成功创建后才 defer 关闭。

写文件、压缩流、网络连接等场景，`Close` 本身也可能返回重要错误。简单读文件可以忽略 close 错误，写入场景要更谨慎。

---

## 7. 一次性和流式

一次性读取：

```go
data, err := os.ReadFile("data.txt")
```

适合小文件、配置文件。

流式处理：

```go
f, err := os.Open("large.log")
if err != nil {
    return err
}
defer f.Close()

scanner := bufio.NewScanner(f)
for scanner.Scan() {
    handle(scanner.Text())
}
if err := scanner.Err(); err != nil {
    return err
}
```

适合大文件、网络流、持续输入。

---

## 8. 包名通常很短

Go 标准库包名通常短而明确：

- `fmt`
- `os`
- `io`
- `bufio`
- `json`
- `http`
- `time`
- `sync`

使用时通常通过包名表达来源：

```go
json.Marshal(v)
time.Now()
os.Open(name)
```

不要给标准库包起奇怪别名，除非发生命名冲突。

---

## 9. 文档示例值得运行

标准库文档中的 Example 通常可以直接运行。学习时可以复制到本地，改参数、改输入、观察输出。

建议做法：

1. 先运行原示例。
2. 改一个参数。
3. 故意传错误输入。
4. 看错误是什么。
5. 把它改造成自己的小函数。

---

## 10. 建立包索引

不要试图一次背完标准库。先建立索引：

- 读写数据：`io`、`os`、`bufio`
- 路径：`path/filepath`
- 文本：`strings`、`strconv`
- JSON：`encoding/json`
- 时间：`time`
- CLI：`flag`
- 日志：`log/slog`
- 测试：`testing`、`httptest`

遇到问题时能想到该查哪个包，就已经迈过了标准库学习的第一道门。

---

## 练习

1. 打开一个标准库包文档，记录包级说明、三个常用函数、两个核心类型。
2. 找出 `io.Reader` 和 `io.Writer` 在其他包里的使用场景。
3. 写一个函数读取文件内容，并正确处理 error。
4. 写一个函数打开文件后用 `defer Close` 关闭。
5. 总结一次性读取和流式处理各适合什么场景。
