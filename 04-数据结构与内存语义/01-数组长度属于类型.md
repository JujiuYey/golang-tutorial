# 01: 数组：长度属于类型

这篇解决一个坐标校准问题：JS 的 `Array` 其实是"动态列表"，而 Go 的数组是**长度焊死的一排格子**——长度直接写进类型里，赋值时整箱复制。看懂这一点，你就明白为什么 Go 日常代码里几乎都在用 slice（下一篇），以及 slice 到底建立在什么之上。

---

## 1. 声明数组

```go
var a [3]int
fmt.Println(a)
```

```text
[0 0 0]
```

`[3]int` 读作"3 个 int 的数组"。注意长度写在**类型**里，方括号在前——和 JS 的 `new Array(3)` 不同，这 3 个格子从出生起就存在，并且填好了零值（02 模块讲过的规则，这里复习）：

| 元素类型 | 零值 |
|------|------|
| `int` | `0` |
| `bool` | `false` |
| `string` | `""` |
| 指针、slice、map、函数、channel、interface | `nil` |

---

## 2. 初始化数组

字面量初始化，花括号里给元素：

```go
nums := [3]int{1, 2, 3}
names := [2]string{"alice", "bob"}
```

不想自己数个数，用 `...` 让编译器数（长度推断）：

```go
nums := [...]int{1, 2, 3}
fmt.Printf("%T\n", nums)
```

```text
[3]int
```

`%T` 打印类型——注意推断出来的类型里长度是 `3`，不是"任意长度"。

还可以指定下标初始化，没点名的格子保持零值：

```go
nums := [5]int{
    0: 10,
    3: 40,
}
fmt.Println(nums)
```

```text
[10 0 0 40 0]
```

拆开看这个语法：

```text
[5]int{ 0: 10, 3: 40 }
//      ┬────  ┬────
//   下标0放10  下标3放40，其余格子=零值
```

---

## 3. 长度属于类型

这是 Go 数组最"反直觉"的设定：**`[3]int` 和 `[4]int` 是两个完全不同的类型**，互相不能赋值。

```go
package main

import "fmt"

func main() {
	var a [3]int
	var b [4]int

	a = b

	fmt.Println(a)
}
```

```text
# command-line-arguments
./main.go:9:6: cannot use b (variable of type [4]int) as [3]int value in assignment
```

编译器说得很直白：`b` 是 `[4]int`，不能当 `[3]int` 用。就像 3 格的收纳盒和 4 格的收纳盒是两种商品，不能互换包装。

对比 JS：JS 数组长度是运行时的一个属性（`arr.length`），随便变；Go 数组长度是编译期的类型信息，焊死不动。**Go 的数组不是"可变长容器"，它是一个固定大小的值。**

---

## 4. 访问与越界

下标从 0 开始，和 JS 一样：

```go
nums := [3]int{10, 20, 30}

fmt.Println(nums[0])
fmt.Println(nums[2])
```

```text
10
30
```

### 🕳️ 坑：越界不会给你 undefined

以为会怎样：像 JS 一样 `nums[3]` 返回 `undefined`，程序继续跑。
实际怎样：编译期直接拦下，或运行时 panic。
为什么：Go 对越界零容忍——宁可当场炸，也不让脏数据流到后面。

下标是**常量**时，编译器直接看穿：

```go
package main

import "fmt"

func main() {
	nums := [3]int{10, 20, 30}
	fmt.Println(nums[3])
}
```

```text
# command-line-arguments
./main.go:7:19: invalid argument: index 3 out of bounds [0:3]
```

下标来自**变量**时，编译器算不出来，就推迟到运行时检查。越界时程序会立刻停止并打印 `panic` 信息；`panic` 的完整含义会在第 06 章学习：

```go
package main

import "fmt"

func main() {
	nums := [3]int{10, 20, 30}
	i := 3
	fmt.Println(nums[i])
}
```

```text
panic: runtime error: index out of range [3] with length 3

goroutine 1 [running]:
main.main()
	./main.go:8 +0x24
exit status 2
```

**一句话总结：常量越界编译期报错，变量越界运行时 panic，没有静默失败这条路。**

---

## 5. 遍历数组

普通 `for`（02 模块的老朋友）：

```go
nums := [3]int{10, 20, 30}

for i := 0; i < len(nums); i++ {
    fmt.Println(i, nums[i])
}
```

`range` 更顺手：

```go
for i, v := range nums {
    fmt.Println(i, v)
}
```

```text
0 10
1 20
2 30
```

`i` 是下标，`v` 是元素值的**副本**——改 `v` 不会改数组。这个"副本"语义是本模块的贯穿主题，第 09 篇会专门拆它。

---

## 6. 数组赋值会复制整个数组

对照实验来了。先回忆 JS：

```js
const a = [1, 2, 3];
const b = a;      // b 和 a 指向同一个数组
b[0] = 100;
console.log(a);   // [100, 2, 3] —— a 跟着变了
```

同样的代码翻译成 Go 数组：

```go
a := [3]int{1, 2, 3}
b := a

b[0] = 100

fmt.Println(a)
fmt.Println(b)
```

```text
[1 2 3]
[100 2 3]
```

`a` 纹丝不动。因为 **Go 的数组是值**：`b := a` 那一刻，3 个元素被整箱复制了一份，之后 `a` 和 `b` 各过各的。JS 复制的是"指向数组的引用"，Go 复制的是"数组本身"——这是两种语言最根本的直觉差异之一，后面 slice、map、struct 的行为全都要拿它当参照物。

---

## 7. 数组作为函数参数

函数传参也是赋值（值传递），所以传数组 = 复制整个数组：

```go
func change(nums [3]int) {
    nums[0] = 100     // 改的是复制进来的那一份
}

func main() {
    a := [3]int{1, 2, 3}
    change(a)
    fmt.Println(a)
}
```

```text
[1 2 3]
```

想改调用方的原数组，传数组的**指针**（第 08 篇细讲指针，这里先见个面）：

```go
func change(nums *[3]int) {
    nums[0] = 100     // 通过地址找到原数组去改
}

func main() {
    a := [3]int{1, 2, 3}
    change(&a)
    fmt.Println(a)
}
```

```text
[100 2 3]
```

不过实际业务里很少这么写——需要"一段可变长度、传来传去不复制数据"的场景，用的是 slice，下一篇正主登场。

---

## 8. 多维数组

多维数组就是"数组的元素还是数组"：

```go
matrix := [2][3]int{
    {1, 2, 3},
    {4, 5, 6},
}

fmt.Println(matrix[0][1])
```

```text
2
```

遍历用双层循环：

```go
for row := 0; row < len(matrix); row++ {
    for col := 0; col < len(matrix[row]); col++ {
        fmt.Print(matrix[row][col], " ")
    }
    fmt.Println()
}
```

```text
1 2 3 
4 5 6 
```

`[2][3]int` 读法：2 行，每行是一个 `[3]int`。

---

## 9. 什么时候使用数组

数组适合**长度固定且长度本身有意义**的场景：

- 固定大小的哈希值：`[32]byte`（SHA-256 就是 32 字节，一个都不多）。
- 固定维度的坐标：`[3]float64`。
- 底层实现中需要确定大小的数据块。

大多数普通列表、集合、查询结果、请求数据——长度天生不固定——应该用 slice。**记住这个分工：数组是地基，slice 是日常。**

---

## 本篇重点

- [ ] Go 数组长度写在类型里：`[3]int` 和 `[4]int` 是两个类型，不能互相赋值。
- [ ] 越界没有 `undefined`：常量下标编译期报错，变量下标运行时 panic。
- [ ] 数组是值：赋值、传参都会**整箱复制**，改副本不影响原数组——和 JS 数组"复制引用"完全相反。
- [ ] 想在函数里改原数组，传 `*[3]int` 指针；但日常可变长数据用 slice。
- [ ] 数组适合长度固定有意义的场景（哈希、坐标），列表类需求交给 slice。
