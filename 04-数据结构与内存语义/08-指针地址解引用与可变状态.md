# 08: 指针：地址、解引用与可变状态

指针保存的是某个值的地址。Go 的指针比 C 简单：可以取地址、解引用、判断 nil，但不能做指针运算。

---

## 1. 取地址

```go
x := 10
p := &x

fmt.Println(p)  // 一个地址
fmt.Println(*p) // 10
```

`&x` 取变量 `x` 的地址。`p` 的类型是 `*int`。

---

## 2. 解引用

```go
x := 10
p := &x

*p = 20

fmt.Println(x) // 20
```

`*p` 表示访问指针指向的值。通过 `*p` 修改，会影响原变量。

---

## 3. 指针的零值是 nil

```go
var p *int
fmt.Println(p == nil) // true
```

解引用 nil 指针会 panic：

```go
var p *int
// fmt.Println(*p) // panic
```

使用指针前，先确认它不是 nil。

---

## 4. new

`new(T)` 会创建一个 T 类型的零值，并返回它的指针。

```go
p := new(int)

fmt.Println(*p) // 0

*p = 10
fmt.Println(*p) // 10
```

日常代码里，更多时候会直接用取地址：

```go
u := User{Name: "alice"}
p := &u
```

---

## 5. 传指针修改调用方变量

```go
func reset(n *int) {
    *n = 0
}

func main() {
    x := 10
    reset(&x)
    fmt.Println(x) // 0
}
```

Go 的函数参数依然是值传递。这里传入的是地址值的副本。函数通过这个地址找到原变量并修改它。

---

## 6. 指向 struct

```go
type User struct {
    Name string
    Age  int
}

u := User{Name: "alice", Age: 18}
p := &u

p.Age++

fmt.Println(u.Age) // 19
```

`p.Age` 是 Go 的简化写法，等价于 `(*p).Age`。

---

## 7. 返回局部变量的地址

Go 允许返回局部变量的地址。

```go
func NewUser(name string) *User {
    u := User{Name: name}
    return &u
}
```

编译器会分析变量生命周期。需要逃逸到函数外的值，会被安排在合适的位置。你不需要手动分配和释放内存。

---

## 8. 指针不能运算

Go 不支持指针加减。

```go
// p++
// p = p + 1
```

这让 Go 的指针比 C 的指针安全很多。需要遍历连续数据时，用数组、slice 和下标。

---

## 9. 什么时候使用指针

适合使用指针：

- 函数或方法需要修改调用方传入的值。
- struct 较大，频繁复制成本不合适。
- 值表示可选状态，比如 `*time.Time` 表示可能没有时间。
- 类型里包含不应该被复制的资源，如锁、文件、连接、缓冲区。

不必使用指针：

- 小的不可变值，如坐标、金额、配置快照。
- 只是为了“看起来更高效”。
- 让数据所有权变得更难判断。

---

## 10. 指针、slice、map 的区别

```go
func changeInt(n int) {
    n = 100
}

func changeIntPtr(n *int) {
    *n = 100
}
```

对于普通值，想改调用方变量通常要传指针。

slice 和 map 的变量本身也会被复制，但复制后的值仍然能访问共享的底层数据。

```go
func changeSlice(s []int) {
    s[0] = 100
}

func changeMap(m map[string]int) {
    m["x"] = 100
}
```

这不代表 Go 有引用传递。Go 统一是值传递，只是某些值内部包含对底层数据的引用。
