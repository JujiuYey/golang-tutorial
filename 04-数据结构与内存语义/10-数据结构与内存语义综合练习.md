# 10: 数据结构与内存语义综合练习

这一篇不讲新东西，用 9 道练习把数组、slice、map、struct、指针和 range 串成一张网。标准不是"代码能跑"，而是**每一步你都能回答：这里复制了什么？谁和谁在共享？**——这正是本模块和 JS 直觉差异最大的地方，也是你之前容易混淆的地方（值/引用是复制语义，栈/堆是逃逸分析的事，两码事，见第 08 篇）。

---

## 本篇重点（开工前自查）

- [ ] 能说出四种传参各复制什么：数组/struct 复制数据，指针复制地址，slice 复制头，map 复制句柄（第 08 篇的表）。
- [ ] 能预测 append 的两种结局：cap 够 → 写共享数组，不够 → 搬家（第 03 篇）。
- [ ] 记得 map 三雷：nil 不能写、元素不可取地址、遍历顺序随机（第 04、05 篇）。
- [ ] 记得 range 变量永远是副本，`&u` 是副本的地址（第 09 篇）。
- [ ] 值接收者改不到原值，指针接收者才可以（第 07 篇）。

哪条心虚就先回去翻对应篇目，再开始动手。

## 练习1: 预测输出

先不要运行，预测输出。

```go
func main() {
    a := []int{1, 2, 3, 4}
    b := a[1:3]

    b[0] = 20
    b = append(b, 30)

    fmt.Println(a)
    fmt.Println(b)
}
```

要求：

1. 画出 `a` 和 `b` 的底层数组关系。
2. 说明 `b[0] = 20` 改的是哪个元素。
3. 说明 `append` 有没有分配新数组。

提示：关键在 `b` 的 cap 是几——照第 02 篇第 7 节的画图法标出 `b` 的窗口和备用地盘，答案自然浮出来。预测完再运行对答案，错了就回第 03 篇第 2 节。

## 练习2: 安全删除 slice 元素

实现函数：

```go
func DeleteAt(nums []int, i int) ([]int, bool)
```

要求：

1. `i` 合法时删除对应元素并返回 `true`。
2. `i` 非法时返回原 slice 和 `false`。
3. 删除后保持原顺序。
4. 写一个小例子验证删除开头、中间、末尾和非法下标。

提示：`append(s[:i], s[i+1:]...)` 一行流在第 03 篇第 6 节；别忘了负数下标也是非法。

## 练习3: 克隆 slice

实现函数：

```go
func CloneStrings(s []string) []string
```

要求：

1. 返回一个新的 slice。
2. 修改返回值中的元素，不影响原 slice。
3. 对 nil slice 的行为要说清楚：返回 nil，还是返回空 slice。

提示：`copy` 版和 `append([]string(nil), s...)` 版对 nil 输入的行为不一样——两个都写出来，用 `== nil` 验证。

## 练习4: 单词统计

实现函数：

```go
func CountWords(words []string) map[string]int
```

示例：

```go
words := []string{"go", "rust", "go", "java", "go"}
fmt.Println(CountWords(words))
```

要求：

1. 使用 `map[string]int`。
2. 利用不存在 key 的零值完成计数。
3. 不依赖 map 输出顺序。

提示：核心就一行，第 04 篇第 5 节写过；别忘了函数里 map 得先初始化（nil map 的雷）。

## 练习5: 稳定打印 map

实现函数：

```go
func SortedWordCounts(counts map[string]int) []string
```

返回格式示例：

```go

```

要求：

1. 按 key 字典序排序。
2. 不直接依赖 map range 顺序。
3. 输入 map 为 nil 时返回空结果。

提示："收集 key → 排序 → 按序访问"三步在第 05 篇第 6 节；拼 `"go=3"` 用 `fmt.Sprintf`。nil map 能不能 range？第 09 篇第 8 节。

## 练习6: 学生结构体

定义：

```go
type Student struct {
    ID    int
    Name  string
    Score float64
}
```

实现：

```go
func Average(students []Student) float64
func TopStudent(students []Student) (Student, bool)
```

要求：

1. 空 slice 时 `Average` 返回 0。
2. 空 slice 时 `TopStudent` 返回 `false`。
3. `TopStudent` 不要返回 slice 中元素的指针，先用值返回练习复制语义。

提示：`(Student, bool)` 就是本模块第 04 篇的 comma ok 风格；想想调用方改了返回的 Student，会不会影响 slice 里那份——为什么？

## 练习7: 指针接收者

定义：

```go
type Counter struct {
    Value int
}
```

实现：

```go
func (c *Counter) Inc()
func (c *Counter) Add(n int)
func (c Counter) Snapshot() int
```

要求：

1. `Inc` 和 `Add` 要修改原值。
2. `Snapshot` 不修改状态。
3. 尝试把 `Inc` 改成值接收者，观察结果。

提示：第 3 步的现象在第 07 篇第 2 节演示过——连调三次 `Inc` 再打印，差距一目了然。

## 练习8: 修复 range 修改副本

下面代码想给所有学生加 5 分，但不会生效：

```go
for _, s := range students {
    s.Score += 5
}
```

要求：

1. 解释为什么不会生效。
2. 改成使用下标的版本。
3. 如果 `students` 的类型是 `[]*Student`，再写一版。

提示：第 3 问里 `s` 依然是副本——但它是**什么的**副本？副本里含不含"线索"？（第 08 篇的表 + 第 09 篇第 3 节。）

## 练习9: 联系人簿

收官题，把 struct + map + 方法 + 排序全部用上：

```go
type Contact struct {
    Name  string
    Phone string
    Email string
}

type AddressBook struct {
    contacts map[string]Contact
}
```

实现方法：

```go
func NewAddressBook() *AddressBook
func (b *AddressBook) Add(c Contact)
func (b *AddressBook) Delete(name string)
func (b *AddressBook) Find(name string) (Contact, bool)
func (b *AddressBook) ListNames() []string
```

要求：

1. `contacts` 使用 `map[string]Contact`。
2. `ListNames` 返回排序后的姓名。
3. `Find` 返回值和 `bool`，不要让调用方直接修改内部 map。
4. 思考：如果把 map 的值改成 `*Contact`，接口语义会有什么变化。

提示：`NewAddressBook` 里别忘了 make（nil map 的雷）；`contacts` 小写开头意味着什么（第 06 篇第 7 节）？第 4 问对照第 05 篇第 4 节的表格想——`Find` 返回的东西调用方一改，内部数据还安全吗。

## 验证建议

可以把练习放进一个小 module，用测试验证：

```bash
go test ./...
```

如果练习里出现 goroutine 和 map 并发读写，再使用：

```bash
go test -race ./...
```

本模块最重要的验收标准，是你能用自己的话回答两组问题：

1. 赋值、传参、range、append、map 更新——每个动作**复制了什么、谁和谁共享**。
2. 值/引用和栈/堆为什么是两回事——谁决定前者，谁决定后者。

答得上来，这个模块就真的过关了。
