# 01: 网络编程基础：地址、端口、URL 与 DNS

这篇把网络编程的"地基"打好：IP、端口、TCP/UDP、URL、DNS 这几个词到底指什么，以及 Go 里对应的工具（`net/url`、`net` 包）怎么用。学完你能正确地解析和拼装 URL（不再手写字符串拼接），并且从第一天就带上"网络调用必须有超时"的肌肉记忆。

在 Node 里你可能直接 `fetch(url)` 就完事了，很少想过下面发生了什么。Go 的网络 API 更贴近底层一点，所以先把这几层概念对齐，后面 HTTP 服务端/客户端的每个参数你都能知道它在管哪一层。

---

## 1. 地址和端口

网络服务启动时要声明"我在哪里等请求"——一个地址加一个端口：

```text
127.0.0.1:8080
//  ───┬───  ─┬─
//  ①在哪台机器  ②机器上的哪个"门牌号"
```

- `127.0.0.1`：本机回环地址，只有本机能访问（`localhost` 就是它的别名）。
- `8080`：端口，同一台机器靠端口区分不同服务。

想让局域网/外网也能访问，要监听所有网卡：

```text
:8080
0.0.0.0:8080
```

这两种写法等价。对比 Node：`server.listen(8080)` 默认监听所有网卡；Go 里 `:8080` 才是这个意思，`127.0.0.1:8080` 则只对本机开门。开发时用 `localhost:8080` 没问题，部署时要想清楚监听地址和暴露端口。

---

## 2. TCP 和 UDP

传输层就两个主角，日常打交道的协议几乎都骑在它们之一上面：

| | TCP | UDP |
|---|-----|-----|
| 连接 | 面向连接（先握手） | 无连接（拿起来就发） |
| 可靠性 | 保证送达、保证顺序 | 不保证送达、顺序、去重 |
| 数据形态 | 字节流（没有消息边界） | 数据报（一发一个包） |
| 典型场景 | HTTP、数据库、SSH | DNS、实时音视频、游戏 |

**HTTP 跑在 TCP 之上**——所以后面配置 HTTP 超时、连接复用时，底下管的其实都是 TCP 连接。TCP"字节流没有消息边界"这个特点是第 09 篇的重头戏，先记住这句话。

---

## 3. URL 结构

一个 URL 拆开是六个部分：

```text
https://example.com:443/users/42?active=true#profile
──┬──   ─────┬─────  ─┬─ ───┬───  ─────┬───  ───┬───
scheme     host     port  path      query    fragment
（协议）   （主机）  （端口）（路径）  （查询串） （锚点）
```

这和 JS 的 `new URL(...)` 拆出来的属性一一对应（`protocol`/`host`/`pathname`/`search`/`hash`）。Go 里干这活的是 `net/url` 包。

---

## 4. 解析 URL

`url.Parse` 把字符串拆成结构体，相当于 JS 的 `new URL(str)`——区别是 JS 解析失败会 throw，Go 照例返回 error 让你检查：

```go
package main

import (
	"fmt"
	"net/url"
)

func main() {
	u, err := url.Parse("https://example.com/users?id=42")
	if err != nil {
		fmt.Println("parse failed:", err)
		return
	}

	fmt.Println(u.Scheme)
	fmt.Println(u.Host)
	fmt.Println(u.Path)
	fmt.Println(u.Query().Get("id"))
}
```

```text
https
example.com
/users
42
```

`u.Query()` 返回 `url.Values`——它的底层就是 `map[string][]string`（一个 key 可以有多个值，比如 `?tag=a&tag=b`）。JS 里的对应物是 `URLSearchParams`，`Get` 同样只取第一个值。

---

## 5. 构造 URL

反过来，想拼一个带查询参数的 URL，别用 `+` 手拼字符串——填结构体、用 `url.Values` 装参数，转义交给库：

```go
package main

import (
	"fmt"
	"net/url"
)

func main() {
	u := url.URL{
		Scheme: "https",
		Host:   "api.example.com",
		Path:   "/users",
	}

	q := u.Query()
	q.Set("page", "1")
	q.Set("q", "go 语言")
	u.RawQuery = q.Encode()

	fmt.Println(u.String())
}
```

```text
https://api.example.com/users?page=1&q=go+%E8%AF%AD%E8%A8%80
```

注意 `"go 语言"` 被自动转义成了 `go+%E8%AF%AD%E8%A8%80`——空格变 `+`，中文变 UTF-8 字节的百分号编码（还记得 02-05 吗，一个中文 3 个字节，所以是 3 段 `%XX`×2 个字）。手拼字符串最容易在这里翻车。

**一句话总结：拼 URL 用 `url.URL` + `url.Values`，转义永远交给库。**

---

## 6. URL 转义

也可以单独转义一段文本，对应 JS 的 `encodeURIComponent` / `decodeURIComponent`：

```go
package main

import (
	"fmt"
	"net/url"
)

func main() {
	escaped := url.QueryEscape("Go 语言")
	fmt.Println(escaped)

	value, err := url.QueryUnescape(escaped)
	if err != nil {
		fmt.Println("unescape failed:", err)
		return
	}
	fmt.Println(value)
}
```

```text
Go+%E8%AF%AD%E8%A8%80
Go 语言
```

🕳️ 坑：路径和查询参数的转义规则**不一样**。以为一个 `QueryEscape` 走天下 → 实际上空格在 query 里转成 `+`、在路径里得是 `%20`（路径用 `url.PathEscape`）→ 因为 HTTP 规范对两个位置的保留字符定义不同。分不清就回到第 5 节的做法：让 `url.URL` 整体处理，不要自己挑函数。

## 7. DNS 查询

DNS 把域名翻译成 IP。Go 里用 `net.LookupIP`（对应 Node 的 `dns.lookup`）：

```go
package main

import (
	"fmt"
	"net"
)

func main() {
	ips, err := net.LookupIP("example.com")
	if err != nil {
		fmt.Println("lookup failed:", err)
		return
	}
	for _, ip := range ips {
		fmt.Println(ip)
	}
}
```

```text
172.66.147.243
104.20.23.154
2606:4700:10::6814:179a
2606:4700:10::ac42:93f3
```

（你跑出来的 IP 大概率不同——CDN 会按地域返回不同结果，前两个是 IPv4，后两个是 IPv6。）

和 JS 里一样，平时写业务几乎不用手动查 DNS——HTTP client 内部自动做。知道这一层存在的意义是：**DNS 解析失败是网络错误的常见来源之一**（第 10 节）。

## 8. net.Dial

`net.Dial` 是"打电话"：跟目标机器的某个端口建立一条 TCP 连接。对应 Node 的 `net.connect`：

```go
package main

import (
	"fmt"
	"net"
	"time"
)

func main() {
	dialer := net.Dialer{Timeout: 5 * time.Second}

	conn, err := dialer.Dial("tcp", "example.com:80")
	if err != nil {
		fmt.Println("dial failed:", err)
		return
	}
	defer conn.Close()

	fmt.Println("connected:", conn.RemoteAddr())
}
```

```text
connected: 172.66.147.243:80
```

拿到的 `conn` 能读能写，也要记得 `defer conn.Close()`（06-03 的套路：确认资源到手，再预约清理）。大多数 HTTP 场景不需要直接 Dial——HTTP client 内部帮你管连接;直接用它是第 09 篇 TCP 编程的内容。

## 9. 超时意识

网络和本地函数调用的本质区别：**对方可能永远不回话**。JS 的 `fetch` 默认也没有超时，页面卡住是前端常见事故；Go 服务端卡住后果更严重——goroutine 堆积、连接耗尽。

上面 `net.Dialer{Timeout: 5 * time.Second}` 就是在设超时。看看它真的生效：连一个不会应答的地址（`10.255.255.1` 是个"黑洞"内网地址）：

```go
package main

import (
	"fmt"
	"net"
	"time"
)

func main() {
	dialer := net.Dialer{Timeout: 2 * time.Second}

	start := time.Now()
	_, err := dialer.Dial("tcp", "10.255.255.1:80")
	fmt.Println("耗时:", time.Since(start).Round(time.Second))
	fmt.Println("错误:", err)
}
```

```text
耗时: 2s
错误: dial tcp 10.255.255.1:80: i/o timeout
```

正好等了 2 秒就放弃。如果不设 Timeout，这个 Dial 会卡上几十秒甚至更久。**本模块的铁律：任何网络调用都要有超时**，后面每一篇都会落实这句话。

## 10. 网络错误长什么样

网络错误五花八门，先见几个真面目。连一个没人监听的本机端口：

```go
package main

import (
	"fmt"
	"net"
)

func main() {
	_, err := net.Dial("tcp", "127.0.0.1:18099")
	fmt.Println(err)
}
```

```text
dial tcp 127.0.0.1:18099: connect: connection refused
```

常见错误来源对照：

| 错误来源 | 典型信息 |
|---------|---------|
| DNS 失败 | `no such host` |
| 连接被拒绝（端口没人听） | `connection refused` |
| 超时 | `i/o timeout` |
| 连接中途断开 | `connection reset by peer` |
| 对端返回错误状态码 | 这不是 error！要自己检查 `StatusCode`（第 06 篇） |

注意 Go 的网络错误自带上下文（操作 `dial`、协议 `tcp`、目标地址）。你自己往上包装错误时也要保持这个习惯（06 模块的 `fmt.Errorf("...: %w", err)`）：保留请求的地址、操作名称、状态码，出问题才查得动。

---

## 本篇重点

- [ ] `127.0.0.1:8080` 只对本机开门，`:8080` 监听所有网卡；部署时要想清楚用哪个。
- [ ] TCP 是可靠字节流（HTTP 的地基），UDP 是不可靠数据报；"字节流没有消息边界"后面会反复出现。
- [ ] 解析 URL 用 `url.Parse`（≈ JS 的 `new URL`），拼 URL 用 `url.URL` + `url.Values`（≈ `URLSearchParams`），转义交给库。
- [ ] 任何网络调用都要有超时，否则可能永远卡住——这是本模块的铁律。
- [ ] 网络错误常见五类：DNS 失败、拒绝连接、超时、中途断开、坏状态码；处理时保留地址和操作上下文。
