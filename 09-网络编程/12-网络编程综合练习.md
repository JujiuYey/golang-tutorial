# 12: 网络编程综合练习

这一节用练习把 HTTP server、HTTP client、JSON API、中间件、测试和 TCP/UDP 串起来。

---

## 练习1: Health 服务

实现：

```go
GET /health
```

响应：

```json
{"status":"ok"}
```

要求：

1. 使用 `http.NewServeMux`。
2. 使用 `writeJSON`。
3. 不支持其他 method 时返回 405。
4. 用 `httptest.NewRecorder` 测试。

---

## 练习2: 用户 JSON API

实现：

```text
GET    /users
POST   /users
GET    /users/{id}
DELETE /users/{id}
```

要求：

1. 使用内存 map 存储用户。
2. 用 `sync.RWMutex` 保护 map。
3. JSON 请求限制最大 1MB。
4. 错误响应使用统一 JSON 格式。
5. 创建成功返回 201。
6. 删除成功返回 204。

---

## 练习3: 中间件链

实现中间件：

1. `RequestID`
2. `Logger`
3. `Recover`
4. `Auth`

要求：

1. 能按顺序组合。
2. `Recover` 能捕获 panic 并返回 500。
3. `Auth` 缺少 token 时返回 401。
4. 日志记录 method、path、status、duration。

---

## 练习4: HTTP Client

实现：

```go
type Client struct {
    BaseURL string
    HTTP    *http.Client
}

func (c *Client) GetUser(ctx context.Context, id int) (User, error)
func (c *Client) CreateUser(ctx context.Context, req CreateUserRequest) (User, error)
```

要求：

1. 使用 `NewRequestWithContext`。
2. client 设置 timeout。
3. 关闭 response body。
4. 非 2xx 状态返回错误。
5. 用 `httptest.Server` 测试。

---

## 练习5: 文件上传

实现：

```text
POST /upload
```

要求：

1. 使用 multipart form。
2. 限制上传大小。
3. 使用服务端生成的文件名保存。
4. 返回 JSON，包含保存后的文件名和大小。
5. 写上传测试。

---

## 练习6: 静态文件服务

要求：

1. 用 `http.FileServer` 服务 `public` 目录。
2. 路由前缀是 `/static/`。
3. 用 `httptest` 测试能读取一个文件。
4. 思考路径穿越风险。

---

## 练习7: TCP echo

实现：

1. TCP server 按行读取。
2. 每行返回 `echo: <line>`。
3. client 发送一行并读取响应。
4. 设置读写 deadline。
5. 说明协议边界是换行。

---

## 练习8: UDP ping

实现：

1. UDP server 收到 `ping` 返回 `pong`。
2. UDP client 设置 2 秒 read deadline。
3. 说明 UDP 不保证送达。
4. 测试 buffer 不足时的截断行为。

---

## 练习9: 错误路径测试

为用户 API 至少测试：

1. 无效 JSON。
2. 未知字段。
3. 缺少必填字段。
4. 不存在的用户 ID。
5. 不支持的方法。

---

## 验证建议

建议运行：

```bash
go test ./...
go test -race ./...
```

如果涉及真实端口，测试结束要关闭 server 或 listener。优先使用 `httptest` 和本地临时端口，避免依赖外部网络。
