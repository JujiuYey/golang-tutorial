# 06: 错误处理与程序边界

前面已经学过函数、多返回值、struct、方法和接口，现在终于具备了理解 Go 错误处理的全部前置知识。这一章从 `error` 的真实定义开始，再学习错误返回、错误链、资源清理，以及 `panic/recover` 的边界。

这里要建立的不是几条孤立语法，而是一条完整判断：**预料中的失败返回 `error`，调用方检查并决定怎么办；程序无法正常继续时才可能 `panic`，并且只在明确边界处 `recover`。**

## 学习目标

完成本模块后，你应该能够：

- 从接口的角度解释 `error`，不再把它当成特殊语法或神秘对象。
- 按 Go 习惯返回和检查错误，不把失败结果继续传下去。
- 用 `%w` 保留错误链，并用 `errors.Is`、`errors.As` 判断错误。
- 使用 `defer` 清理资源，理解执行顺序和参数求值时机。
- 区分可预料的错误与不可恢复的 panic，并理解 recover 的作用范围。
