# 05: 错误处理综合练习

学完这个模块，是时候把错误返回、错误链、defer 和 panic/recover 组装起来了。函数和多返回值已经在第 03 章学过，这里只关注失败发生后怎样表达、传递、清理和兜底。

---

## 本篇重点（模块回顾）

动手之前，扫一眼这个模块你应该已经带走的东西：

- 错误是实现了 `Error() string` 的接口值，不是另一条神秘通道（01 篇）。
- `if err != nil` 当场处理，不随手用 `_` 吞掉（01 篇）。
- `%w` 包装保链，`errors.Is` 和 `errors.As` 沿链判断（02 篇）。
- 资源到手后再用 defer 预约清理，注意参数当场求值（03 篇）。
- 预料中的失败返回 error，不该发生的事才 panic，边界处 recover（04 篇）。

下面每道题做完，对照这张单子看看用上了哪几条。

---

## 练习1: 阶乘函数

实现函数：

```go
func factorial(n int) (int, error)
```

要求：

1. `n < 0` 时返回错误。
2. `0` 和 `1` 返回 `1`。
3. 正常返回阶乘结果。
4. 调用方必须检查错误。

提示：考察第 01 篇。错误信息里带上 `n` 的实际值；在 `main` 里用 `-1`、`0`、`5` 各测一次。

---

## 练习2: 安全除法

实现函数：

```go
func safeDivide(a, b float64) (float64, error)
```

要求：

1. `b == 0` 时返回错误。
2. 正常返回计算结果和 `nil`。
3. 调用方打印错误或结果。

提示：考察第 01 篇。出错分支的第一个返回值给什么？想想零值。

---

## 练习3: 文件读取

实现函数：

```go
func readFile(path string) ([]byte, error)
```

要求：

1. 使用 `os.Open` 打开文件。
2. 使用 `defer` 关闭文件。
3. 使用 `io.ReadAll` 读取内容。
4. 所有错误都要带上下文。

提示：考察第 02、03 篇。`defer file.Close()` 的位置在错误检查之前还是之后？先在本地建一个测试文件，再故意传一个不存在的路径，两条路都要走通。

---

## 练习4: 错误判断

实现函数：

```go
func readRequiredFile(path string) ([]byte, error)
```

要求：

1. 文件不存在时返回包装后的错误。
2. 调用方使用 `errors.Is(err, os.ErrNotExist)` 判断文件不存在。
3. 打印不同的错误提示。

提示：考察第 02 篇。包装用 `%w` 而不是 `%v`——不确定为什么的话，回去看第 2 节那个 `true / false` 对比。

---

## 练习5: panic 边界

实现函数：

```go
func runSafely(fn func()) (err error)
```

要求：

1. `fn` 正常执行时返回 `nil`。
2. `fn` 发生 panic 时，在 defer 中 recover，并把 panic 内容转成 error。
3. 分别传入一个正常函数和一个会 panic 的函数，验证两条路径。

提示：考察第 04 篇。`recover()` 只有放在延迟执行的函数里才有作用；这里是边界兜底，不是普通错误处理方式。

---

## 验证建议

每个练习都建议运行：

```bash
go run .
```

也可以为这些函数写最小测试：

```bash
go test ./...
```

不满足于"能跑"：每道题至少构造一个成功输入和一个失败输入，确认两条路径的输出都符合预期——错误路径没测过的代码，等于没写完。
