# Learning Log

这个目录记录用户在本 Go 教程仓库里的学习画像和学习日志。它是 agent 的长期上下文，不是课程正文。

agent 使用这些记录来主导下一节课：先判断当前模块、已完成主题和薄弱点，再选择一个 25 分钟内能完成的学习目标。

## 文件结构

- `profile.md`：长期学习画像，记录当前水平、偏好、薄弱点和下一步。
- `daily/YYYY-MM-DD.md`：每日学习日志。只有用户确认后，agent 才能追加。

## 使用规则

1. 每次学习辅导开始前，agent 先读 `profile.md`。
2. agent 再读最近相关的 daily 日志；如果 `profile.md` 写了当前模块，再读当前模块 README。
3. agent 主动选择本节 1 个学习目标，不要求用户每次重新规划。
4. 一次只推进 1 个认知点，不连续追问多个问题。
5. 诊断问题只作为入口；如果用户卡住，agent 要短讲解，再安排练习和验证。
6. 学习小节结束时，agent 先生成日志草稿。
7. 用户确认后，agent 才能写入 daily 日志，并按需更新 `profile.md`。

## 标准课堂闭环

默认每节课按 25 分钟标准课设计：

1. 状态确认：当前模块、上次完成主题、薄弱点、本节目标。
2. 热身诊断：只问 1 个前置问题。
3. 微讲解：用短解释和一个最小例子补当前概念。
4. 小练习：预测输出、改 1 行、写 3-8 行或解释 1 条错误。
5. 命令验证：用 `go run`、`go test`、`go vet`、`go test -race` 或 `curl` 证明结论。
6. 收束反思：只在本节完成后让用户用一句话总结。
7. 日志草稿：等待用户确认后写入。

## Profile 字段建议

`profile.md` 应逐步维护这些字段：

- `当前模块`：正在学习的模块或文章。
- `已完成主题`：已经讲过、练过、验证过的主题。
- `当前薄弱点`：从回答、代码或命令输出中暴露的问题。
- `下一节目标`：下次应完成的一个小目标。
- `验证习惯`：用户已经实际使用过的验证命令。
