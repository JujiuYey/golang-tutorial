## 主题：字符串、byte 与 rune

### 字符串底层 = 字节序列
- Go 字符串是不可变的 `[]byte`，不是 `[]rune`
- `len(s)` 数字节，不数字符
- ASCII 占 1 字节，中文 UTF-8 通常占 3 字节

### 索引 vs range
- `s[i]` 拿 byte（uint8，本质是数字），要用 `string(s[i])` 才转回可显示字符
- `for i, r := range s` 按 rune 遍历；`r` 是字符（int32），`i` 是**字节下标**而非字符下标
- 中文 3 字节，所以 range 索引会"跳"

### 易错点
- 一开始以为 `len("Go语言") = 3`（按字符数算），实际是 8
- 以为 range 的 `i` 是字符序号，实际是字节起点位置

### 验证记录
- `len("Go语言") = 8`、`s[0] = 71`、`string(s[0]) = "G"`
- `for i, r := range "Go语言"` 输出 `i=0,1,2,5`（不是 0,1,2,3）
- 统计程序：`bytes=8, chars=4`

## 主题：显式类型转换

### 核心规则
- Go 不做隐式类型转换：不同数字类型不能直接混算
- 转换不是安全检查，是"裸操作"——可能丢信息（截断、溢出）
- 设计意图：让程序员自己决定是否接受代价，避免悄悄丢精度

### 两条易混的"数字→字符串"路径
- `string(byte/rune)`：把数字当 **Unicode 码点** 翻译成 1 个字符
  - `string(65)` → `"A"`，`string(22909)` → `"好"`
- `strconv.Itoa(int)`：把整数按 **十进制文本** 转字符串
  - `strconv.Itoa(65)` → `"65"`
- 反向：`strconv.Atoi("456")` → `456, nil`
- 易错点：想把 `i := 123` 变 "123" 却写 `string(i)` → 得到字符 `{`（码点 123）

### 转换方向的"安全原则"
- 窄 → 宽（`int → int64`）永远安全
- 宽 → 窄（`int64 → int`）可能溢出，要主动确认数值范围

### 验证记录
- `string(65) = "A"`、`string(22909) = "好"`、`strconv.Itoa(65) = "65"`
- 修复：`var count int + var total int64` 报错 `mismatched types int and int64`
- 改后 `int64(count) + total = 110`

## 主题：布尔表达式与条件判断

### 核心规则：if 必须是 bool
- Go 没有 truthy / falsy：条件必须是 `bool` 类型本身
- `if (0)`、`if ("")`、`if (n)` 都编译错误（non-boolean condition in if statement）
- 想表达"n 不为 0"必须写 `if n != 0`
- 设计意图：不让语言替程序员猜"什么算真"，避免语义歧义
- 与 JS 对比：JS 里 0/""/null/undefined/NaN 全部 falsy，Go 全部编译错误

### if 初始化语句（文章第 5 节）
- 形式：`if <init>; <condition> { ... }`
  - `if n := len("golang"); n > 3 { ... }`
  - `if err := doSomething(); err != nil { ... }`
- 拆三部分：初始化语句 + 条件 + 分号分隔
- 核心好处：**变量的作用域被限制在 if/else 块内**，出 `{}` 就消失，不污染外层
- 这是 Go 的惯用法，几乎所有 error handling 都长这样

### err-handling 模式（最常见应用）
- Go 函数通常返回 `(value, err)`，调用方立刻 `if err != nil` 处理
- 标准形式：
  ```go
  if err := doSomething(); err != nil {
      return err
  }
  ```
- 为什么这样写：把 `err` 锁在 if 块里，避免外层 scope 出现一堆用完的 err 变量

### 简化惯用法
- `if 条件 { return true } else { return false }` 纯属多余
- 直接：`return age >= 18 && agreedTerms`（表达式本身就是 bool）
- `if agreedTerms` 比 `if agreedTerms == true` 更地道（bool 直接当条件）

### 验证记录
- `if n {}`（n 是 int）编译错误：non-boolean condition in if statement
- `canRegister(19, false) = false`（age 够但未同意 → 拒绝）
- canRegister 函数最终写法：`return age >= 18 && agreedTerms`（一行）
- `strconv.Atoi("42")` + if init：`parsed: 42`
