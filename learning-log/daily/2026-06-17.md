# 2026-06-17 学习日志

## 主题：值类型和字面量

### 字面量
- 代码里直接写出来的值：`25`、`"Go"`、`3.14`、`true`
- 变量名不是字面量

### 值类型 vs 引用类型
- 值类型（`b := a` 之后改 a，b 不变）：`int`、`float64`、`bool`、`string`、`[N]T`、`struct`
- 引用类型（`b := a` 之后改 a，b 变）：`[]T`、`map`、`chan`
- 记忆口诀：基础 + 数组 + 结构体 = 值类型；切片 / map / channel = 引用类型

### 易错点
- 一开始以为"基础类型走栈、复杂类型走堆"，这是错的。Go 根本没有这条规则，栈/堆是 runtime 自己决定的。
- 数组 `[3]int` 是值类型，**不是**引用类型。这点和 JS 数组完全不同。

### 与 JS 的差异
- JS 数组（对象）赋值后改一边另一边必变
- Go 数组赋值后是"复印件"，互不影响；只有切片、map、channel 才像 JS 的对象

## 主题：变量声明、赋值、零值与作用域

### 零值
- Go 规则：所有变量声明时都有可用零值（int→0, string→"", bool→false）
- 调试空字符串用 `%q`，`%s` 对空串透明

### 变量声明
- `:=` 短变量声明：只能在函数内部
- `var x = 200`：类型推断由右边字面量决定
- 字面量是"无类型"，决定权在左边（var 类型 / `:=` 目标）

### 作用域 + 变量遮蔽
- 内层声明同名变量 = 遮蔽（shadowing）
- 内层是**新建一个变量**，外层不会被修改；块级变量出 `{}` 就消失
- 易错点：以为改了外层，其实新建了一个（Go 常见 bug 源）
- 与 JS 区别：JS var 会提升/重声明；Go 不会"覆盖修改"

### 赋值 vs 声明
- `:=` = 声明 + 赋值（新建）
- `=` = 赋值（变量必须已存在）

### 验证记录
- 零值：`var i int; var s string; var b bool` → `0 "" false`，用 `%q` 看清空串
- var 类型推断：`var b = 200` → `%T` 输出 `int`
- shadowing：块内 `x := 999`，块外仍是 1
- 赋值：`x := 1; x = 2` → 输出 2

## 主题：常量与 iota

### 基本常量
- `const pi = 3.14`、`const appName = "demo"`
- const 值在编译期确定，运行时不能改

### 批量声明
- `const (...)` 块，按用途命名

### 无类型常量
- `const n = 10` 是无类型常量（和字面量一样灵活）
- 可以赋给 int / int64 / float64：左边要什么类型，n 自动适配
- 变量 `x := 10` 推断为 int（固定类型），不能直接给 int64 / float64
- 必须显式转换：`int64(x)`

### iota：const 块里的自动计数器
- 从 0 开始，每声明一行 +1
- 最常见用途：给枚举自动编号
- 跳过用 `_ = iota`，第一行就是 1

### iota 位标志
- `1 << iota` 得到 1/2/4/8（2 的幂）
- 每个权限占独立二进制位，不重叠
- 可以用 `|` 组合：`Read|Write|Execute|Delete = 1|2|4|8 = 1111 = 15`
- 用普通 iota（0/1/2/3）会让位重叠，无法区分权限

### 验证记录
- 基本 iota：`A=iota, B, C` → `0 1 2`
- 枚举：`StatusPending=iota, StatusPaid, StatusShipped, StatusCanceled` → `0 1 2 3`
- 位标志：`Read=1<<iota, Write, Execute, Delete` → `1 2 4 8`，组合 = 15
- 无类型常量：`const n = 10; var a int = n; var b int64 = n; var c float64 = n` → 输出 `10 10 10`
- 变量赋值：`x := 10; var b int64 = x` → 编译错误 `cannot use x (variable of type int) as int64 value`

## 主题：数字类型与运算

### 整数除法 + 类型转换
- 整数 ÷ 整数 = 整数（小数部分直接砍掉）
- 要保留小数必须先转 float64：`float64(a) / float64(b)`
- 例：`10 / 3 = 3`，`float64(10) / float64(3) = 3.3333333333333335`

### 浮点天生不精确
- 浮点用二进制表示小数，部分十进制小数（0.1、0.2、0.3）无法精确表示
- `0.1 + 0.2 = 0.30000000000000004`（不是 0.3）
- `0.1 + 0.2 == 0.3` 是 false
- 影响：涉及金额时不能直接用 float64 表示"分"，要用整数表示最小货币单位

### 练习：百分比函数
```go
func percent(part int, total int) float64 {
	if total == 0 {
		return 0
	}
	return float64(part) / float64(total) * 100
}
```
- 三个要求：total==0 返回 0、保留小数（先转 float64）、main 测试 percent(1,3)
- 验证：`percent(1, 3) = 33.33333333333333`（不是 33，证明没被整数除法截断）

### 验证记录
- 整数除法：`10/3 = 3`、`float64(10)/float64(3) = 3.3333333333333335`
- 浮点不精确：`0.1+0.2 = 0.30000000000000004`、`0.1+0.2 == 0.3 = false`
- 百分比：`percent(1, 3) = 33.33333333333333`
