## 主题：for 循环与 range（02-08 全节）

### range slice 三种基本形式
- `for i, v := range nums`：下标和值都要
- `for _, v := range nums`：只要值
- `for i := range nums`：只要下标
- `_` 是占位符，不是变量名；Go 不允许声明了不用的变量（之前在 02-02 见过 `declared but not used`），range 这里必须用 `_` 才能编译过

### range 的值是副本（Section 5，最易错点）
- `for _, v := range nums` 里的 `v` 是元素的副本，不是引用
- 改 `v`（包括 `v *= 10`、`v = 100`）不影响原 slice
- 想改原 slice 必须用索引：`nums[i] *= 10`
- `nums[i]` 直接定位到原 slice 的槽位，不是副本
- 设计模式区分：
  - "算出新值"的函数 → 返回结果（如 `sum`）
  - "修改原数据"的函数 → 直接改参数，不返回（如 `multiply`）

### range 遍历 string（Section 6，复习之前薄弱点）
- `range s` 按 rune（字符）遍历，不是按字节
- `i` 是每个字符的**起始字节位置**（不是字符序号）
- `r` 是该字符的 Unicode 码点（int32）
- `string(r)` 把码点转回 1 个字符的可显示字符串
- 中文 UTF-8 通常 3 字节，所以索引会跳（如 "Go语言" 输出 `0 1 2 5`，不是 `0 1 2 3`）

### range 遍历 map（Section 7）
- `for k, v := range m` 遍历 map
- **遍历顺序不保证稳定**（Go 故意随机化，避免代码依赖顺序）
- 需要稳定顺序时，先取 key 排序再遍历

### 复合赋值（顺手讲）
- `v *= 10` 是复合赋值，等价于 `v = v * 10`，跟指针无关
- 常见写法对照：`v *= 10` ≡ `v = v * 10`、`v += 1` ≡ `v = v + 1`、`v -= 2` ≡ `v = v - 2`

### 易错点
- 预测 range 输出格式时，以为是两列分开，实际 `fmt.Println(i, v)` 是空格分隔、同一行
- 以为 `v *= 10` 跟指针有关（实际是复合赋值）
- `sum(1, 2, 3)` 编译报错 → slice 要显式构造 `sum([]int{1,2,3})`
- `multiply_result := multiply(nums)` 编译报错 → multiply 不返回值，不能赋给变量

## 主题：switch 与跳转语句（02-09 部分）

### 关键点
- **Go switch 默认不贯穿**：匹配到 case 后只执行这一支，不会继续执行下一个 case（与 C/Java/JS 不同，那些要写 `break`）
- **多值 case**：`case 1, 2, 3, 4, 5:` 一个 case 匹配多个值，用逗号分隔
- **无表达式 switch**：`switch {}` 不写表达式，每个 case 自己写条件；是 Go 替代长 if-else 链的惯用法
- **case 顺序很重要**：`>= 90` 必须在 `>= 80` 前面，否则永远不会到 A

### grade 练习
- 用 `switch {}` + 多个 case 实现
- 先处理非法分数：`case score < 0 || score > 100: return "invalid"`
- 顺序：`>= 90` → A，`>= 80` → B，`>= 60` → C，`default` → D
- main 里用 `[]int` 装测试分数 + `for _, v := range` 遍历调用（复用 02-08 知识）
- 测试 7 个分数：100、90、89、60、59、-1、101，全部正确

### 易错点（之前踩过类似的）
- 多值 case 时容易把范围写错（如 `case 6, 7:` 而不是 `case 6, 7:` 漏写逗号）
- grade 函数里 `score >= 90 && score <= 100` 的 `score <= 100` 严格说是冗余的（前面已经过滤了 > 100），但写全没错
