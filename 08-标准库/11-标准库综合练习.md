# 11: 标准库综合练习

这一节用几个小工具把标准库串起来。目标是练习真实代码里的错误处理、文件路径、JSON、日志、测试和命令行参数。

---

## 练习1: JSON 配置加载器

实现：

```go
type Config struct {
    AppName string `json:"app_name"`
    Port    int    `json:"port"`
    Debug   bool   `json:"debug"`
}

func LoadConfig(path string) (Config, error)
func SaveConfig(path string, cfg Config) error
```

要求：

1. 使用 `os.ReadFile` 和 `os.WriteFile`。
2. 使用 `encoding/json`。
3. 写入时使用缩进 JSON。
4. 对文件路径和 JSON 错误添加上下文。
5. 写测试，使用 `t.TempDir()`。

---

## 练习2: 文件复制工具

实现：

```go
func CopyFile(dst, src string) error
```

要求：

1. 使用 `os.Open`、`os.Create`、`io.Copy`。
2. 正确关闭文件。
3. 源文件不存在时可以用 `errors.Is(err, os.ErrNotExist)` 判断。
4. 写测试验证复制内容一致。

---

## 练习3: 日志 CLI

实现命令：

```bash
logtool -level=INFO -msg="server started" -json
```

要求：

1. 使用 `flag` 解析参数。
2. `-msg` 不能为空。
3. 支持文本日志和 JSON 日志。
4. 使用 `log/slog`。
5. 错误输出到 stderr。

---

## 练习4: 文本统计

实现：

```go
func CountWords(r io.Reader) (map[string]int, error)
```

要求：

1. 使用 `bufio.Scanner`。
2. 使用 `strings.Fields` 拆分单词。
3. 使用 `strings.ToLower` 统一大小写。
4. 返回 map，不依赖遍历顺序。
5. 写测试时用 `strings.NewReader`。

---

## 练习5: 稳定输出

实现：

```go
func FormatCounts(counts map[string]int) []string
```

要求：

1. 输出格式为 `word=count`。
2. 按 word 字典序排序。
3. 可以使用 `sort.Strings` 或 `slices.Sort`。
4. nil map 返回空 slice。

---

## 练习6: 目录扫描

实现：

```go
func FindFiles(root string, ext string) ([]string, error)
```

要求：

1. 使用 `filepath.WalkDir`。
2. 只返回普通文件。
3. 按扩展名过滤，比如 `.json`。
4. 返回结果排序。
5. 写测试时使用 `t.TempDir()` 创建目录结构。

---

## 练习7: 时间解析

实现：

```go
func ParseLocal(layout, value, location string) (time.Time, error)
```

要求：

1. 使用 `time.LoadLocation`。
2. 使用 `time.ParseInLocation`。
3. 错误要带上 layout、value 或 location 信息。
4. 测试上海时区和 UTC。

---

## 练习8: HTTP handler 测试预习

实现一个简单 handler：

```go
func Health(w http.ResponseWriter, r *http.Request)
```

要求：

1. 返回状态码 200。
2. 返回 JSON：`{"status":"ok"}`。
3. 使用 `httptest.NewRecorder` 测试。
4. 检查响应状态码和 JSON 内容。

---

## 验证建议

本章练习建议至少运行：

```bash
go test ./...
go test -race ./...
```

如果写了 benchmark：

```bash
go test -bench=. ./...
```

标准库练习要养成三个习惯：错误不丢、资源关闭、测试不依赖真实环境。
