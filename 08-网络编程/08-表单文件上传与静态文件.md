# 08: 表单、文件上传与静态文件

这篇解决 JSON 之外的三样日常：HTML 表单、文件上传、静态文件。Express 生态里这三样分别靠 `body-parser`、`multer`、`express.static` 三个包；Go 标准库全包了——`ParseForm`、`FormFile`、`http.FileServer`。学完你能安全地收表单、存上传文件（不被文件名坑），并把一个目录挂成静态站点。

> 本篇实跑 `localhost:18015`，所有输出为真实结果。

---

## 1. 解析普通表单

`<form>` 提交的 `application/x-www-form-urlencoded` 数据，先 `ParseForm` 再取值：

```go
func submit(w http.ResponseWriter, r *http.Request) {
	if err := r.ParseForm(); err != nil {
		http.Error(w, "invalid form", http.StatusBadRequest)
		return
	}
	fmt.Fprintln(w, "Form:", r.Form.Get("name"))
	fmt.Fprintln(w, "PostForm:", r.PostForm.Get("name"))
}
```

```bash
curl -d 'name=alice' http://localhost:18015/submit
```

```text
Form: alice
PostForm: alice
```

🕳️ 坑：不先调 `ParseForm` 就去读 `r.Form`，拿到的**永远是空**——不报错、不 panic，就是安静的空字符串。Express 里中间件帮你解析好了，Go 里这一步是自己的活。

---

## 2. Form 和 PostForm 的区别

`ParseForm` 其实解析了两处：URL 查询串 + 表单 body。两个字段各看一部分。用"只有 query、没有 body"的请求现形：

```bash
curl -X POST 'http://localhost:18015/submit?name=fromquery'
```

```text
Form: fromquery
PostForm: 
```

| 字段 | 包含什么 | 用在什么时候 |
|------|---------|-------------|
| `r.Form` | URL query + 表单 body | 不在乎来源时 |
| `r.PostForm` | 只有表单 body | 敏感操作——防止有人把参数塞进 URL 冒充表单提交 |

**一句话总结：读表单提交的数据用 `PostForm`，别让 URL 里的同名参数混进来。**

---

## 3. 限制请求体大小

第 03 篇的规矩在上传场景加倍重要——表单接口 1MB 够用，上传接口按业务给（比如 10MB）：

```go
r.Body = http.MaxBytesReader(w, r.Body, 10<<20) // 10MB
```

`10<<20` 就是 10×2²⁰ = 10MB（位运算写法，Go 圈的习惯记号）。不限制的话，一个异常客户端就能用超大请求吃光你的内存和磁盘。

---

## 4. 文件上传

浏览器上传文件用的是 `multipart/form-data`。服务端三步：限大小 → 解析 multipart → 取文件：

```go
func upload(w http.ResponseWriter, r *http.Request) {
	r.Body = http.MaxBytesReader(w, r.Body, 10<<20)

	if err := r.ParseMultipartForm(10 << 20); err != nil {
		http.Error(w, "invalid multipart form", http.StatusBadRequest)
		return
	}

	file, header, err := r.FormFile("file") // "file" 是表单字段名
	if err != nil {
		http.Error(w, "missing file", http.StatusBadRequest)
		return
	}
	defer file.Close()

	// ...保存,见下一节...
	fmt.Fprintf(w, "uploaded: %s (...)\n", header.Filename)
}
```

curl 的 `-F` 就是模拟表单上传：

```bash
curl -F 'file=@hello.txt' http://localhost:18015/upload
```

```text
uploaded: hello.txt (13 bytes) -> uploads/upload-1221521535
```

字段名对不上时（客户端用了别的名字）：

```bash
curl -i -F 'other=@hello.txt' http://localhost:18015/upload
```

```text
HTTP/1.1 400 Bad Request
Content-Type: text/plain; charset=utf-8
X-Content-Type-Options: nosniff
Content-Length: 13

missing file
```

`header.Filename` 是**客户端随口说的**，跟 `User-Agent` 一样想填什么填什么——它只能当展示用的元数据，绝不能直接当保存路径（第 9 节展开）。

## 5. 保存上传文件

保存时让**服务端自己起文件名**，用 `os.CreateTemp` 生成不重复的名字：

```go
dst, err := os.CreateTemp("uploads", "upload-*") // * 会被随机串替换
if err != nil {
	http.Error(w, "create file failed", http.StatusInternalServerError)
	return
}
defer dst.Close()

if _, err := io.Copy(dst, file); err != nil { // 流式拷贝,不整个读进内存
	http.Error(w, "save file failed", http.StatusInternalServerError)
	return
}
```

上一节输出里的 `uploads/upload-1221521535` 就是它起的名。`io.Copy` 边读边写，大文件也不会撑爆内存。**服务端起名 + 原始文件名只存元数据**，路径穿越问题从根上消失。

## 6. 静态文件服务

`express.static('public')` 的 Go 版：

```go
fs := http.FileServer(http.Dir("public"))
mux.Handle("/static/", http.StripPrefix("/static/", fs))
```

拆解这两行：

```text
请求 /static/app.css
    → StripPrefix 剥掉 "/static/"  → 剩 "app.css"
    → FileServer 在 public/ 里找   → public/app.css
```

```bash
curl -i http://localhost:18015/static/app.css
```

```text
HTTP/1.1 200 OK
Accept-Ranges: bytes
Content-Length: 22
Content-Type: text/css; charset=utf-8

body { color: teal; }
```

Content-Type 根据扩展名自动设好了。忘写 `StripPrefix` 是经典 bug：FileServer 会拿着完整的 `/static/app.css` 去 `public/static/app.css` 找，全员 404。

顺带试试路径穿越能不能得逞：

```bash
curl --path-as-is -i 'http://localhost:18015/static/../reports/report.txt'
```

```text
HTTP/1.1 301 Moved Permanently
Content-Type: text/html; charset=utf-8
Location: /reports/report.txt
```

`..` 被规范化掉、重定向到一个不存在的路由（最终 404）——标准库替你挡了这一枪。但这不代表高枕无忧：**自己拼文件路径的代码（下一节的下载）不享受这个保护。**

## 7. 下载文件

想让浏览器"下载"而不是"打开"，加 `Content-Disposition` 头（Express 里 `res.download` 干的事）：

```go
func download(w http.ResponseWriter, r *http.Request) {
	w.Header().Set("Content-Disposition", `attachment; filename="report.txt"`)
	http.ServeFile(w, r, "reports/report.txt")
}
```

```bash
curl -i http://localhost:18015/download
```

```text
HTTP/1.1 200 OK
Accept-Ranges: bytes
Content-Disposition: attachment; filename="report.txt"
Content-Length: 28
Content-Type: text/plain; charset=utf-8

quarterly numbers: all good
```

🕳️ 坑：如果 `ServeFile` 的路径里拼了用户输入（`"reports/" + r.URL.Query().Get("f")`），`f=../main.go` 就能读走你的源码。用户可控的文件名必须严格校验、限制在允许目录内——或者干脆用 ID 查表映射到真实路径。

## 8. Multipart 客户端

Go 程序自己当上传方（对应 JS 的 `FormData` + `fetch`），用 `mime/multipart` 攒请求体：

```go
var body bytes.Buffer
writer := multipart.NewWriter(&body)

part, err := writer.CreateFormFile("file", "hello.txt")
if err != nil {
	return err
}
part.Write([]byte("hello from Go client"))
writer.Close() // 必须 Close:写入 multipart 的结束边界

req, err := http.NewRequest(http.MethodPost, "http://localhost:18015/upload", &body)
if err != nil {
	return err
}
req.Header.Set("Content-Type", writer.FormDataContentType())
```

配上第 06 篇的 client 发出去：

```text
状态: 200 OK
uploaded: hello.txt (20 bytes) -> uploads/upload-377529461
```

两个必做项：`writer.Close()` 不是可有可无——不 Close 就没有结束边界，服务端解析直接失败；`Content-Type` 用 `writer.FormDataContentType()`，因为 multipart 的分隔符（boundary）是随机的，得从 writer 上拿。

## 9. 文件名安全

如果确实要参考客户端文件名（比如保留扩展名），至少先剥掉路径部分：

```go
fmt.Println(filepath.Base("../../etc/passwd"))
fmt.Println(filepath.Base("photo.png"))
```

```text
passwd
photo.png
```

`filepath.Base` 把 `../../etc/passwd` 削成无害的 `passwd`。但它只解决路径穿越，重名覆盖、特殊字符、伪造扩展名、内容与类型不符这些还在。所以最稳的策略仍是第 5 节那句：**存储名服务端生成，原始文件名只作为元数据入库。**

---

## 10. 表单处理检查

1. body 是否限制了大小（表单 1MB、上传按业务）？
2. 上传的 `file` 是否 `defer file.Close()`？
3. 是否把 `header.Filename` 当成了不可信输入？
4. 保存路径是否完全由服务端控制？
5. 各失败路径的状态码是否清楚（缺字段 400、写盘失败 500）？

---

## 本篇重点

- [ ] 读表单前必须先 `ParseForm`，否则静默拿空值；`Form` 含 query，`PostForm` 只含 body。
- [ ] 上传三步：`MaxBytesReader` 限大小 → `ParseMultipartForm` → `FormFile("字段名")`；保存用 `os.CreateTemp` + `io.Copy` 流式写。
- [ ] `header.Filename` 是客户端随口说的：只当元数据，存储名由服务端生成。
- [ ] 静态服务 = `FileServer` + `StripPrefix`，忘剥前缀全员 404；标准库挡路径穿越，但自己拼路径的代码不受保护。
- [ ] Go 客户端上传：`multipart.NewWriter` 攒体，`writer.Close()` 写结束边界，Content-Type 从 `FormDataContentType()` 拿。
