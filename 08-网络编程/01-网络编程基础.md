# 01: 网络编程基础：地址、端口、URL 与 DNS

写网络程序前，要先弄清楚几个基本概念：IP、端口、协议、URL、DNS、请求和响应。

---

## 1. 地址和端口

网络服务通常监听某个地址和端口：

```text
127.0.0.1:8080
```

含义：

- `127.0.0.1`: 本机回环地址。
- `8080`: 端口。

监听所有网卡：

```text
:8080
0.0.0.0:8080
```

开发时常用 `localhost:8080`，部署时要明确监听地址和暴露端口。

---

## 2. TCP 和 UDP

TCP：

- 面向连接。
- 可靠传输。
- 字节流。
- 常用于 HTTP、数据库连接、SSH。

UDP：

- 无连接。
- 数据报。
- 不保证送达和顺序。
- 常用于 DNS、实时音视频、游戏等场景。

HTTP 通常跑在 TCP 之上。

---

## 3. URL 结构

```text
https://example.com:443/users/42?active=true#profile
```

拆开看：

- scheme: `https`
- host: `example.com`
- port: `443`
- path: `/users/42`
- query: `active=true`
- fragment: `profile`

Go 用 `net/url` 解析 URL。

---

## 4. 解析 URL

```go
u, err := url.Parse("https://example.com/users?id=42")
if err != nil {
    return err
}

fmt.Println(u.Scheme)
fmt.Println(u.Host)
fmt.Println(u.Path)
fmt.Println(u.Query().Get("id"))
```

`Query()` 返回 `url.Values`，它是 `map[string][]string`。

---

## 5. 构造 URL

```go
u := url.URL{
    Scheme: "https",
    Host:   "api.example.com",
    Path:   "/users",
}

q := u.Query()
q.Set("page", "1")
q.Set("q", "go")
u.RawQuery = q.Encode()

fmt.Println(u.String())
```

不要手写拼接 query 字符串。使用 `url.Values` 可以正确处理转义。

---

## 6. URL 转义

```go
escaped := url.QueryEscape("Go 语言")
fmt.Println(escaped)

value, err := url.QueryUnescape(escaped)
if err != nil {
    return err
}
fmt.Println(value)
```

路径和查询参数的转义规则不同。构造 URL 时优先使用 `url.URL` 和 `url.Values`。

---

## 7. DNS 查询

```go
ips, err := net.LookupIP("example.com")
if err != nil {
    return err
}
for _, ip := range ips {
    fmt.Println(ip)
}
```

DNS 把域名解析成 IP。实际 HTTP client 通常会自动做 DNS 解析，业务代码很少直接调用。

---

## 8. net.Dial

```go
conn, err := net.Dial("tcp", "example.com:80")
if err != nil {
    return err
}
defer conn.Close()
```

`net.Dial` 建立底层连接。HTTP client 在内部会帮你管理连接，大多数 HTTP 场景不需要直接 Dial。

---

## 9. 超时意识

网络操作一定要考虑超时。

```go
dialer := net.Dialer{
    Timeout: 5 * time.Second,
}

conn, err := dialer.Dial("tcp", "example.com:80")
```

没有超时的网络调用可能长时间卡住。

---

## 10. 网络错误

网络错误可能来自：

- DNS 失败。
- 连接被拒绝。
- 超时。
- 连接中途断开。
- 协议格式错误。
- 对端返回错误状态码。

处理网络错误时，要保留上下文，比如请求的地址、操作名称、状态码。
