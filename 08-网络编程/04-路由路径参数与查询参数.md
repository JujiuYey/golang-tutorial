# 04: 路由、路径参数与查询参数

路由负责把不同路径和方法的请求分发给不同 handler。标准库的 `ServeMux` 已经能完成很多基础需求。

---

## 1. 基本路由

```go
mux := http.NewServeMux()

mux.HandleFunc("GET /health", health)
mux.HandleFunc("GET /users", listUsers)
mux.HandleFunc("POST /users", createUser)
```

较新的 Go 版本中，`ServeMux` 支持带 method 的 pattern。旧版本可以在 handler 内部自己判断 `r.Method`。

---

## 2. 路径参数

```go
mux.HandleFunc("GET /users/{id}", getUser)

func getUser(w http.ResponseWriter, r *http.Request) {
    id := r.PathValue("id")
    fmt.Fprintln(w, id)
}
```

`PathValue` 返回字符串。需要数字 ID 时，要用 `strconv.Atoi`。

```go
id, err := strconv.Atoi(r.PathValue("id"))
if err != nil {
    http.Error(w, "invalid id", http.StatusBadRequest)
    return
}
```

---

## 3. 旧版本手动解析路径

如果环境不支持新 ServeMux pattern，可以手动处理：

```go
const prefix = "/users/"
if !strings.HasPrefix(r.URL.Path, prefix) {
    http.NotFound(w, r)
    return
}
id := strings.TrimPrefix(r.URL.Path, prefix)
```

手动解析要小心空 ID、多余路径段和 URL 转义。

---

## 4. 查询参数

```go
func listUsers(w http.ResponseWriter, r *http.Request) {
    q := r.URL.Query()
    page := q.Get("page")
    keyword := q.Get("q")
    fmt.Fprintln(w, page, keyword)
}
```

查询参数适合过滤、分页、排序：

```text
GET /users?page=2&q=alice
```

路径参数适合资源标识：

```text
GET /users/42
```

---

## 5. Method 分发

如果不用 method pattern，可以这样分发：

```go
func users(w http.ResponseWriter, r *http.Request) {
    switch r.Method {
    case http.MethodGet:
        listUsers(w, r)
    case http.MethodPost:
        createUser(w, r)
    default:
        w.Header().Set("Allow", "GET, POST")
        http.Error(w, "method not allowed", http.StatusMethodNotAllowed)
    }
}
```

不支持的方法应该返回 405，并设置 `Allow` header。

---

## 6. NotFound

```go
http.NotFound(w, r)
```

找不到路由或资源不存在时，都可能返回 404。API 里可以根据场景返回 JSON 格式的 404。

---

## 7. 路由顺序和匹配

标准库 `ServeMux` 会根据 pattern 匹配请求。设计路由时要避免模糊路径。

建议：

- 资源使用复数名词：`/users`。
- 子资源清楚表达层级：`/users/{id}/posts`。
- 动作用 method 表达，不要把所有动作塞进路径。

---

## 8. 路由分组

标准库没有框架式路由分组，但可以用函数组织：

```go
func registerUserRoutes(mux *http.ServeMux, h UserHandler) {
    mux.HandleFunc("GET /users", h.List)
    mux.HandleFunc("POST /users", h.Create)
    mux.HandleFunc("GET /users/{id}", h.Get)
}
```

这样能保持 main 函数清爽。

---

## 9. 版本前缀

```text
/api/v1/users
```

API 版本可以放在路径里，也可以通过 header 管理。课程阶段用路径前缀最直观。

---

## 10. 路由设计检查

1. 路径是否表达资源。
2. 方法是否表达动作。
3. ID 是否做了格式校验。
4. 方法不支持时是否返回 405。
5. 路由注册是否集中清晰。

---
