# 05: 中间件、日志、恢复与请求 ID

中间件是包装 `http.Handler` 的函数。它适合处理日志、鉴权、panic 恢复、请求 ID、CORS 等横切逻辑。

---

## 1. 中间件形态

```go
func Middleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        // before
        next.ServeHTTP(w, r)
        // after
    })
}
```

输入是 handler，输出也是 handler。

---

## 2. 日志中间件

```go
func Logger(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)
        slog.Info("request",
            "method", r.Method,
            "path", r.URL.Path,
            "duration_ms", time.Since(start).Milliseconds(),
        )
    })
}
```

这个版本还拿不到响应状态码。真实项目可以包装 `ResponseWriter` 记录 status。

---

## 3. 记录状态码

```go
type statusWriter struct {
    http.ResponseWriter
    status int
}

func (w *statusWriter) WriteHeader(status int) {
    w.status = status
    w.ResponseWriter.WriteHeader(status)
}
```

使用：

```go
sw := &statusWriter{ResponseWriter: w, status: http.StatusOK}
next.ServeHTTP(sw, r)
fmt.Println(sw.status)
```

如果 handler 没有显式调用 `WriteHeader`，第一次写 body 默认是 200，所以初始值设为 `http.StatusOK`。

---

## 4. panic 恢复

```go
func Recover(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        defer func() {
            if rec := recover(); rec != nil {
                slog.Error("panic", "value", rec)
                http.Error(w, "internal server error", http.StatusInternalServerError)
            }
        }()
        next.ServeHTTP(w, r)
    })
}
```

恢复 panic 可以避免单个请求让整个服务崩溃。普通业务错误仍然应该用 error 或明确响应处理。

---

## 5. 请求 ID

```go
type requestIDKey struct{}

func RequestID(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        id := r.Header.Get("X-Request-ID")
        if id == "" {
            id = generateID()
        }
        ctx := context.WithValue(r.Context(), requestIDKey{}, id)
        w.Header().Set("X-Request-ID", id)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}
```

context value 适合请求范围元数据，比如 request id。不要用它传普通业务参数。

---

## 6. 鉴权中间件

```go
func Auth(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        token := r.Header.Get("Authorization")
        if token == "" {
            http.Error(w, "unauthorized", http.StatusUnauthorized)
            return
        }
        next.ServeHTTP(w, r)
    })
}
```

真实鉴权会校验 token、过期时间、权限范围。这里先掌握中间件结构。

---

## 7. 链式组合

```go
handler := Recover(Logger(Auth(mux)))
```

执行顺序从外到内：

1. `Recover`
2. `Logger`
3. `Auth`
4. `mux`

顺序会影响行为。通常恢复放外层，日志也尽量靠外。

---

## 8. CORS 简介

CORS 是浏览器安全策略相关机制。简单示例：

```go
func CORS(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Access-Control-Allow-Origin", "https://example.com")
        w.Header().Set("Access-Control-Allow-Methods", "GET, POST, OPTIONS")
        if r.Method == http.MethodOptions {
            w.WriteHeader(http.StatusNoContent)
            return
        }
        next.ServeHTTP(w, r)
    })
}
```

不要在生产环境随意使用 `*`，尤其是涉及凭证和敏感接口时。

---

## 9. 中间件不要吞掉响应

中间件如果提前返回，要明确写状态码和响应体。

```go
if token == "" {
    http.Error(w, "unauthorized", http.StatusUnauthorized)
    return
}
```

调用 `next.ServeHTTP` 之后，一般不要再尝试改状态码。

---

## 10. 中间件设计检查

1. 是否只处理横切逻辑。
2. 是否明确调用或不调用 next。
3. 是否保留请求 context。
4. 是否正确写错误状态码。
5. 多个中间件顺序是否合理。

---

4. 写 `Auth` 中间件检查 Authorization。
