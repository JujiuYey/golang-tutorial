# Go Feynman 4 Questions

学完一节 Go 课程后，用这四问自检是否真的懂了。

## 触发场景

- 用户说"刚学完 02-变量声明"、"讲一下 channel"、"解释一下 interface"
- 用户完成 README 里某篇文章的"读 / 写 / 改"循环
- 用户想确认自己是否"讲得出来"

## 四问模板

按顺序问，每问用户**必须用自己的话**答完再进下一问：

1. **是什么？** —— 这个概念用一句话定义是什么？
2. **解决什么问题？** —— 没有它会怎样？它在 Go 里替代了什么其他语言的做法？
3. **最容易写错在哪？** —— 给我一个你**会写错**的具体代码片段（不是教程里的，是你自己可能犯的）。
4. **如果 AI 写错了怎么发现？** —— 看哪一行、跑哪个命令、问 AI 哪个反问能暴露问题？

## 引导策略

- 用户答得模糊时，反问"能不能更具体？"、"给个最小代码试试？"
- 用户答得对时，挑一个**反例**（来自 bad-smells 清单）让他识别
- 不要在用户没答完四问之前，给出"标准答案"

## 收尾

四问答完后，引导用户用一句话总结：

> "你现在能判断哪类 Go 代码问题？"
