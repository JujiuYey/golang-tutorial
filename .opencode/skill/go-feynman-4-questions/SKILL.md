# Go Feynman One Question

学完一节 Go 课程后，用单问推进的方式自检是否真的懂了。

## 触发场景

- 用户说"刚学完 02-变量声明"、"讲一下 channel"、"解释一下 interface"
- 用户完成 README 里某篇文章的"读 / 写 / 改"循环
- 用户想确认自己是否"讲得出来"

## 单问推进

每一轮只选下面 1 个问题。用户答完后，再决定下一轮问什么。

1. **是什么？** —— 这个概念用一句话定义是什么？
2. **解决什么问题？** —— 没有它会怎样？它在 Go 里替代了什么其他语言的做法？
3. **最容易写错在哪？** —— 给我一个你**会写错**的具体代码片段（不是教程里的，是你自己可能犯的）。
4. **怎么验证？** —— 你会跑哪个命令、看哪段输出证明判断是对的？

## 引导策略

- 用户答得模糊时，反问"能不能更具体？"、"给个最小代码试试？"
- 用户答得部分正确时，只补 1 个关键缺口，不展开成完整讲义
- 用户答得对时，给 1 个最小反例或验证命令让他确认
- 不要在用户没答完当前问题之前，给出"标准答案"

## 收尾

当前小节完成后，先给出学习日志草稿，确认后再写入 `learning-log/daily/YYYY-MM-DD.md`。然后引导用户用一句话总结：

> "你现在能判断哪类 Go 代码问题？"
