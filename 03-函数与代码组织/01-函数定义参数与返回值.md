# 01: 函数定义、参数与返回值

这篇解决一个问题：把 JS 里写函数的手感搬到 Go 上来。学完你能看懂任何 Go 函数的签名，也知道 Go 比 JS 多出的三个硬性要求——参数要写类型、返回值要写类型、说好返回就必须返回。

---

## 1. 基本函数定义

最简单的函数，和 JS 几乎一样，只是 `function` 换成了 `func`：

```go
package main

import "fmt"

func greet() {
    fmt.Println("Hello")
}

func main() {
    greet()
}
```

```text
Hello
```

带参数和返回值时，差别就出来了。第一次见 Go 的函数头，拆开看：

```go
func add(a int, b int) int {
//   ─┬─ ────┬───────  ─┬─
//    ①名字  ②参数:名字在前、类型在后   ③返回值类型
    return a + b
}
```

**和 JS 的核心差异：类型是必填项，而且写在名字后面。** JS 里 `function add(a, b)` 不用管类型；Go 里每个参数都要声明类型，返回值也要——这和 02 模块里 `var age int` 的顺序是一套规则：先名字，后类型。

相邻参数类型相同时，类型可以只写最后一次：

```go
func add(a, b int) int {   // a 和 b 都是 int
    return a + b
}
```

**一句话总结：Go 函数头 = 名字 + （参数名 类型） + 返回值类型，类型永远跟在名字后面。**

---

## 2. 无返回值函数

不写返回值类型，就是"什么都不返回"：

```go
func printUser(name string, age int) {
    fmt.Printf("%s is %d years old\n", name, age)
}
```

```text
Alice is 30 years old
```

这类函数用来"做动作"——打印、写文件、发请求。

对比 JS：JS 函数不写 `return` 会默默返回 `undefined`；Go 里无返回值函数就是真的没有值，想拿它的"结果"赋给变量会直接编译报错。Go 没有 `undefined` 这个兜底。

---

## 3. 单返回值函数

```go
func square(n int) int {
    return n * n
}

func main() {
    result := square(5)
    fmt.Println(result)
}
```

```text
25
```

`square(5)` 的结果是一个 `int`，可以赋值、可以参与运算，和 JS 用法一致。

---

## 4. 函数签名：Go 没有重载

"签名"（signature）就是函数的输入输出规格——参数类型 + 返回值类型：

```go
func add(a, b int) int
```

下面两个函数签名不同，一个吃 `int` 一个吃 `float64`：

```go
func addInt(a, b int) int
func addFloat(a, b float64) float64
```

🕳️ 坑：你可能想像某些语言那样，定义两个同名函数、靠参数类型区分（重载）。**Go 不允许**，同一个包里函数名必须唯一：

```go
func add(a, b int) int         { return a + b }
func add(a, b float64) float64 { return a + b }
```

```text
./main.go:4:6: add redeclared in this block
	./main.go:3:6: other declaration of add
```

"redeclared"（重复声明）——编译器认为你把 `add` 定义了两遍。想支持不同类型？起不同的名字（如标准库的 `strconv.ParseInt` / `ParseFloat`），或以后学泛型再解决。

顺带一提，JS 也没有重载（后定义的同名函数直接覆盖前面的），Go 只是把这件事变成了显式的编译错误。

## 5. return：说好返回，就必须返回

`return` 结束函数并交出结果，后面的代码不再执行：

```go
func abs(n int) int {
    if n < 0 {
        return -n   // 走到这就结束了
    }
    return n
}

func main() {
    fmt.Println(abs(-7), abs(7))
}
```

```text
7 7
```

🕳️ 坑：声明了返回值类型，就必须**每条路径**都有返回。JS 里漏写 `return` 会拿到 `undefined`，Go 直接编译不过：

```go
func half(n int) int {
    if n%2 == 0 {
        return n / 2
    }
    // n 是奇数时呢?没说返回什么
}
```

```text
./main.go:9:1: missing return
```

编译器帮你堵住了"忘记返回"这条路——这是好事，这类 bug 在 JS 里要到运行时才炸。

**一句话总结：返回值类型是承诺，编译器会检查每条路径是否兑现。**

---

## 6. 参数名只在函数内有效

参数就是函数自己的局部变量，出了 `{}` 就不存在：

```go
func double(n int) int {
    return n * 2
}

func main() {
    fmt.Println(double(3))
    fmt.Println(n)   // 函数外面没有 n 这个东西
}
```

```text
./main.go:11:14: undefined: n
```

这一点和 JS 完全一致——函数参数不会泄漏到外面。调用 `double(3)` 时，`3` 被交给 `n`，函数返回后 `n` 就随之消失。

---

## 本篇重点

- Go 函数头的顺序：`func 名字(参数名 类型) 返回值类型`，类型永远在名字后面。
- 相邻同类型参数可以合并写：`func add(a, b int) int`。
- Go 没有函数重载，同包内同名函数直接编译报错（`redeclared`）。
- 声明了返回值类型，每条执行路径都必须 `return`，否则报 `missing return`。
- 参数是函数的局部变量，作用域只在函数体内，和 JS 一致。

## 练习

实现两个函数：

```go
func max(a, b int) int
func isEven(n int) bool
```

要求：

1. `max` 返回两个整数中较大的一个。
2. `isEven` 判断数字是否为偶数。
3. 在 `main` 中调用并打印结果。

提示：`max` 里注意第 5 节的规则——`if/else` 两条路径都要有 `return`；`isEven` 可以直接 `return` 一个布尔表达式，不需要 `if`。
