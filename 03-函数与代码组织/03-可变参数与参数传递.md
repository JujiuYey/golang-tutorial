# 03: 参数传递与可变参数

调用函数时，看起来像是把外面的变量"交给"了函数。Go 实际做的动作更具体：**先把值复制一份，再放进函数的参数变量里**。这一篇先用已经学过的 `int`、`string` 和 `bool` 把这个规则讲清楚，再看怎样让一个函数接收数量不固定的参数。

---

## 1. 参数是函数内部的新变量

先看一个普通函数：

```go
func double(n int) int {
    return n * 2
}

func main() {
    x := 5
    result := double(x)
    fmt.Println(x, result)
}
```

```text
5 10
```

调用 `double(x)` 时，外面的变量叫 `x`，函数里面接收它的参数叫 `n`。它们不是同一个变量：

```text
x 的值是 5
    ↓ 调用时复制
n 得到一份 5
```

函数只认识自己的参数 `n`，它看不到调用方把这个值存在哪个变量里。

---

## 2. 修改参数，不会改掉外面的变量

因为参数拿到的是复制品，所以给参数重新赋值，只会修改函数内部这份：

```go
func change(n int) {
    n = 100
    fmt.Println("inside:", n)
}

func main() {
    x := 1
    change(x)
    fmt.Println("outside:", x)
}
```

```text
inside: 100
outside: 1
```

执行过程：

```text
x = 1
  → 调用 change(x)，把 1 复制给 n
  → n = 100，只改 n
  → 函数结束，n 消失
  → x 仍然是 1
```

🕳️ 坑：不要看到函数里写了 `n = 100`，就以为调用方的变量也会跟着变。先问一句：**这个参数里复制进来的到底是什么值？** 第 04 章学习 slice、map 和指针后，我们会继续用同一条规则分析更复杂的类型。

---

## 3. string 和 bool 也是同一条规则

参数复制不是 `int` 的特殊待遇：

```go
func rename(name string) {
    name = "Go"
}

func disable(enabled bool) {
    enabled = false
}

func main() {
    language := "JavaScript"
    active := true

    rename(language)
    disable(active)

    fmt.Println(language, active)
}
```

```text
JavaScript true
```

两个参数都在函数内部变过，但外面的 `language` 和 `active` 没变。规则只有一条：**调用时复制值，参数是函数自己的局部变量。**

---

## 4. 相邻参数类型相同，可以合并写

下面两种签名完全等价：

```go
func add(a int, b int) int
func add(a, b int) int
```

第二种只是省掉了重复的类型。Go 会从右往左看，知道 `a`、`b` 都是 `int`。

不同类型不能硬合并：

```go
func describe(name string, age int, active bool) string
```

函数签名应该让人一眼看出每个位置要传什么，而不是为了少写几个字挤成难读的形式。

---

## 5. 可变参数：参数数量可以不固定

有时参数类型相同，但数量不确定，比如求一组数字的和。类型前面加 `...`，表示这个参数可以接收零个或多个值：

```go
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}
```

调用时想传几个就传几个：

```go
fmt.Println(sum())
fmt.Println(sum(1, 2, 3))
fmt.Println(sum(1, 2, 3, 4, 5))
```

```text
0
6
15
```

`nums ...int` 有三条规则：

1. 每个传入值都必须是 `int`。
2. 可变参数必须放在参数列表最后。
3. 函数体可以用已经学过的 `range` 逐个读取它们。

函数体里的 `nums` 实际类型是 `[]int`，也就是下一章要正式学习的 slice。现阶段先把它理解成"Go 帮你收拢好的一组 int"即可；等学完 slice，再补上把现有 slice 用 `values...` 展开的调用方式。

对比 JS：它对应 `function sum(...nums)`。声明时 JS 把 `...` 写在名字前，Go 把 `...` 写在类型前。

---

## 本篇重点

- 参数是函数内部的新变量，调用时从外面复制一个值进来。
- 给 `int`、`string`、`bool` 参数重新赋值，不会修改调用方的变量。
- 相邻参数类型相同时可以写成 `a, b int`。
- `nums ...int` 表示接收零个或多个 `int`，并且必须是最后一个参数。
- slice、map 和指针仍然遵守值传递；等第 04 章学会这些类型后，再判断它们具体复制了什么。

## 练习

实现两个函数：

```go
func triple(n int) int
func sum(nums ...int) int
```

要求：

1. `triple(4)` 返回 `12`，调用后外面的原变量仍然是 `4`。
2. `sum()` 返回 `0`。
3. `sum(2, 4, 6)` 返回 `12`。
