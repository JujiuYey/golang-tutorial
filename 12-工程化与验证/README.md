# 12: 工程化与验证

本模块讲 Go 项目的工程化和验证。学习主线是一条可靠的验证链路：代码是否格式化，包边界是否清楚，依赖是否可复现，测试是否覆盖关键路径，并发是否安全，性能问题如何定位，CI 是否能稳定挡住回归。

## 学习目标

完成本模块后，你应该能够：

- 设计适合 Go 项目的目录结构和包边界。
- 使用 `go mod tidy`、`go mod download`、`go mod verify` 管理依赖。
- 理解 `gofmt`、`go fmt`、`go vet`、静态检查各自解决的问题。
- 编写表驱动测试、子测试、测试替身和 HTTP 测试。
- 使用覆盖率报告判断测试盲区。
- 使用 `go test -race` 检查并发数据竞争。
- 编写基准测试，并用 `pprof` 定位性能瓶颈。
- 用 Makefile 统一本地常用任务。
- 设计 CI 流水线，让格式、测试、race、构建成为稳定门禁。
- 构建带版本信息的二进制文件和可运行镜像。
- 用最小复现和日志缩小问题范围。

---

## 学习建议

工程化学习要从验证结果倒推工具：

1. 先确认代码能格式化。
2. 再确认代码能通过静态检查。
3. 再确认单元测试覆盖关键路径。
4. 再确认并发代码通过 race detector。
5. 再确认构建产物能运行。
6. 出现问题时，用日志和最小复现缩小范围。
