# 单元测试、表驱动测试与子测试

单元测试验证一个小单元的行为。Go 的测试工具简单，但组合能力很强。

---

## 1. 测试文件命名

源文件：

```go
// calculator.go
package calculator

func Add(a, b int) int {
	return a + b
}
```

测试文件：

```go
// calculator_test.go
package calculator

import "testing"

func TestAdd(t *testing.T) {
	got := Add(2, 3)
	if got != 5 {
		t.Fatalf("Add(2, 3) = %d, want %d", got, 5)
	}
}
```

规则：

- 测试文件以 `_test.go` 结尾。
- 测试函数以 `Test` 开头。
- 参数是 `*testing.T`。
- 使用 `go test` 运行。

---

## 2. 运行测试

```bash
go test ./...
```

显示详细输出：

```bash
go test -v ./...
```

只运行某个测试：

```bash
go test -run TestAdd ./...
```

只运行某个包：

```bash
go test ./internal/task
```

---

## 3. Fatal 和 Error

`t.Fatal` 会立刻停止当前测试：

```go
if err != nil {
	t.Fatalf("create task: %v", err)
}
```

`t.Error` 会记录失败并继续：

```go
if got.Title != "learn go" {
	t.Errorf("title = %q", got.Title)
}
if got.Status != StatusOpen {
	t.Errorf("status = %q", got.Status)
}
```

如果后续断言依赖前面的结果，使用 `Fatal`。多个独立字段可以用 `Error`。

## 4. 表驱动测试

```go
func TestAdd(t *testing.T) {
	tests := []struct {
		name string
		a    int
		b    int
		want int
	}{
		{name: "positive", a: 2, b: 3, want: 5},
		{name: "negative", a: -2, b: -3, want: -5},
		{name: "zero", a: 0, b: 7, want: 7},
	}

	for _, tt := range tests {
		t.Run(tt.name, func(t *testing.T) {
			got := Add(tt.a, tt.b)
			if got != tt.want {
				t.Fatalf("Add(%d, %d) = %d, want %d", tt.a, tt.b, got, tt.want)
			}
		})
	}
}
```

表驱动测试适合覆盖多个输入组合。每一行测试数据都应该有清楚名字。

## 5. 子测试

子测试可以单独运行：

```bash
go test -run 'TestAdd/positive' ./...
```

子测试也方便表达层级：

```go
func TestValidateTitle(t *testing.T) {
	t.Run("rejects empty title", func(t *testing.T) {
		err := ValidateTitle("")
		if err == nil {
			t.Fatal("expected error")
		}
	})

	t.Run("accepts normal title", func(t *testing.T) {
		err := ValidateTitle("learn go")
		if err != nil {
			t.Fatalf("validate title: %v", err)
		}
	})
}
```

## 6. 测试错误

使用 `errors.Is` 检查错误类型：

```go
func TestFindMissingTask(t *testing.T) {
	_, err := repo.FindByID(context.Background(), "missing")
	if !errors.Is(err, ErrNotFound) {
		t.Fatalf("err = %v, want ErrNotFound", err)
	}
}
```

不要只检查错误字符串。错误字符串适合给人看，错误类型适合给程序判断。

---

## 7. 测试导出 API 和内部细节

同包测试：

```go
package task
```

可以访问未导出标识符，适合测试内部复杂函数。

外部包测试：

```go
package task_test
```

只能访问导出 API，更接近真实使用者。

应用代码常用同包测试快速覆盖内部规则，公共库更需要外部包测试检验 API 体验。

---

## 8. 学习检查

检查单元测试时，重点看：

1. 测试名是否表达行为。
2. 是否覆盖正常路径。
3. 是否覆盖错误路径。
4. 表驱动测试是否有清楚 case 名。
5. 错误断言是否使用 `errors.Is` 或 `errors.As`。
6. 测试是否依赖时间、随机数或外部网络。
7. 失败信息是否能帮助定位。
