# 最小复现、日志与排障闭环

工程化的最后一环是排障能力。问题出现时，你要能缩小范围、复现问题、验证修复，并把经验沉淀回测试或文档。

---

## 1. 最小复现

最小复现是能触发问题的最小代码、输入和命令。

一个好的最小复现包含：

1. 运行命令。
2. 输入数据。
3. 期望结果。
4. 实际结果。
5. 最小代码片段。

示例：

```bash
go test -run TestParsePage ./internal/httpx
```

失败信息：

```text
ParsePage("-1") = nil error, want error
```

这比“分页有问题”清楚得多。

---

## 2. 从 bug 到测试

发现 bug 后，先写一个失败测试：

```go
func TestParsePageRejectsNegative(t *testing.T) {
	_, err := ParsePage("-1")
	if err == nil {
		t.Fatal("expected error")
	}
}
```

确认测试失败，再修代码。修完后测试通过，这个测试就变成回归保护。

---

## 3. 日志要服务定位

好日志包含稳定字段：

```go
logger.Info(
	"http request",
	slog.String("method", r.Method),
	slog.String("path", r.URL.Path),
	slog.Int("status", status),
	slog.Duration("duration", time.Since(start)),
	slog.String("request_id", requestID),
)
```

错误日志包含上下文：

```go
logger.Error(
	"create task failed",
	slog.String("request_id", requestID),
	slog.String("user_id", userID),
	slog.String("error", err.Error()),
)
```

日志不是越多越好。关键是字段稳定、上下文足够、能串起一次请求。

---

## 4. request_id

每个请求都应该有 request ID：

```go
id := r.Header.Get("X-Request-ID")
if id == "" {
	id = newRequestID()
}
w.Header().Set("X-Request-ID", id)
```

下游日志都带上同一个 ID，排查时就能串起完整链路。

---

## 5. 错误包装

底层错误要带上下文：

```go
task, err := repo.FindByID(ctx, id)
if err != nil {
	return Task{}, fmt.Errorf("find task %q: %w", id, err)
}
```

上层仍然可以判断：

```go
if errors.Is(err, ErrNotFound) {
	return http.StatusNotFound
}
```

`%w` 能同时保留上下文和错误类型。

---

## 6. 排障步骤

遇到问题时按顺序做：

1. 记录现象：命令、输入、错误输出。
2. 缩小范围：是编译、测试、运行、配置、依赖还是数据问题。
3. 本地复现：找到能稳定触发的命令。
4. 写失败测试或最小脚本。
5. 修改代码。
6. 重跑复现命令。
7. 跑相关完整验证。
8. 把测试、文档或日志补上。

这套流程能减少凭感觉改代码。

---

## 7. 综合练习

创建一个小型 Go 项目，完成完整工程化配置：

1. `cmd/myapp/main.go`。
2. 至少一个 `internal/task` 包。
3. `go.mod` 和 `go.sum`。
4. 单元测试和表驱动测试。
5. 一个使用 `httptest` 的 HTTP 测试。
6. Makefile，包含 `fmt`、`vet`、`test`、`race`、`build`、`check`。
7. GitHub Actions CI。
8. Dockerfile。
9. `README.md` 写明运行和验证命令。

完成后运行：

```bash
make check
docker build -t myapp:dev .
docker run --rm -p 8080:8080 myapp:dev
curl -i http://localhost:8080/healthz
```

---

## 8. 自查清单

完成综合练习后检查：

1. `go fmt ./...` 是否干净。
2. `go vet ./...` 是否通过。
3. `go test ./...` 是否通过。
4. `go test -race ./...` 是否通过。
5. `go build ./...` 是否通过。
6. CI 是否能复现本地验证。
7. 镜像是否能启动。
8. `curl` 是否能验证健康检查。
9. 失败日志是否包含 request ID。
10. 新 bug 是否能沉淀成测试。
