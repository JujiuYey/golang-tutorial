## 学习目标

完成本模块后，你应该能够：

- 设计一个清楚的 Web 服务目录结构。
- 区分启动代码、路由代码、handler、service、repository 的职责。
- 使用 `http.Server` 配置超时、启动服务并处理关闭信号。
- 用结构体保存应用依赖，减少全局变量。
- 把请求解析、参数校验、响应编码放在 HTTP 边界层。
- 建立统一的错误类型和状态码映射。
- 编写可组合的中间件链。
- 在 service 层表达业务规则，在 repository 层隔离持久化细节。
- 为认证、授权、日志、请求 ID 留出稳定入口。
- 使用 `httptest` 覆盖 handler、路由和完整 HTTP 流程。

---

## 学习建议

学习 Web 服务时，优先建立边界感：

1. `main` 只负责组装和启动。
2. 路由只负责把 URL 分配给 handler。
3. handler 只负责 HTTP 输入输出转换。
4. service 负责业务规则和用例编排。
5. repository 负责数据读写。
6. context 从请求入口传到下游调用。
7. 错误要能稳定映射成状态码。
8. 测试要覆盖成功路径、参数错误、业务错误和权限错误。
