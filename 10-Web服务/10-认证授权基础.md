# 认证与授权基础

认证回答“当前请求来自谁”，授权回答“这个身份能执行什么操作”。两者都属于 Web 服务常见边界能力。

---

## 1. 认证信息从请求进入

常见认证方式：

| 方式 | 常见位置 | 适合场景 |
| --- | --- | --- |
| Bearer Token | `Authorization: Bearer <token>` | API、移动端、服务间调用 |
| Cookie Session | `Cookie` | 浏览器应用 |
| Basic Auth | `Authorization: Basic ...` | 内部工具、临时保护 |

本章先用 Bearer Token 建立结构。

---

## 2. Principal 表示当前身份

```go
package auth

type Principal struct {
	UserID string
	Roles  []string
}

func (p Principal) HasRole(role string) bool {
	for _, item := range p.Roles {
		if item == role {
			return true
		}
	}
	return false
}
```

`Principal` 放入 context，后续 handler 和 service 可以读取。

---

## 3. context 中保存身份

```go
package auth

import "context"

type principalKey struct{}

func WithPrincipal(ctx context.Context, principal Principal) context.Context {
	return context.WithValue(ctx, principalKey{}, principal)
}

func PrincipalFromContext(ctx context.Context) (Principal, bool) {
	principal, ok := ctx.Value(principalKey{}).(Principal)
	return principal, ok
}
```

context key 使用未导出类型，减少冲突风险。

---

## 4. Token 校验接口

```go
package auth

import "context"

type TokenVerifier interface {
	Verify(ctx context.Context, token string) (Principal, error)
}
```

中间件只依赖接口，真实校验方式可以是数据库查询、缓存查询、签名校验或远程认证服务。

## 5. Bearer Token 中间件

```go
package auth

import (
	"net/http"
	"strings"

	"example.com/task-api/internal/httpx"
)

func Middleware(verifier TokenVerifier) func(http.Handler) http.Handler {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			header := r.Header.Get("Authorization")
			token, ok := strings.CutPrefix(header, "Bearer ")
			if !ok || token == "" {
				httpx.WriteErrorWithCode(w, http.StatusUnauthorized, "unauthorized", "missing bearer token")
				return
			}

			principal, err := verifier.Verify(r.Context(), token)
			if err != nil {
				httpx.WriteErrorWithCode(w, http.StatusUnauthorized, "unauthorized", "invalid bearer token")
				return
			}

			next.ServeHTTP(w, r.WithContext(WithPrincipal(r.Context(), principal)))
		})
	}
}
```

认证失败用 `401 Unauthorized`。已经认证但权限不够，用 `403 Forbidden`。

## 6. 授权检查

授权可以放在 handler，也可以放在 service。简单规则放 handler 即可，涉及资源归属的规则更适合放 service。

角色检查中间件：

```go
package auth

import (
	"net/http"

	"example.com/task-api/internal/httpx"
)

func RequireRole(role string) func(http.Handler) http.Handler {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			principal, ok := PrincipalFromContext(r.Context())
			if !ok {
				httpx.WriteErrorWithCode(w, http.StatusUnauthorized, "unauthorized", "missing principal")
				return
			}
			if !principal.HasRole(role) {
				httpx.WriteErrorWithCode(w, http.StatusForbidden, "forbidden", "insufficient permission")
				return
			}
			next.ServeHTTP(w, r)
		})
	}
}
```

资源归属检查放 service。下面假设任务模型中有 `OwnerID` 字段：

```go
func (s *Service) Delete(ctx context.Context, principal auth.Principal, id string) error {
	task, err := s.repo.FindByID(ctx, id)
	if err != nil {
		return err
	}
	if task.OwnerID != principal.UserID && !principal.HasRole("admin") {
		return ErrForbidden
	}
	return s.repo.Delete(ctx, id)
}
```

## 7. HMAC 签名 token 示例

简单服务可以用 HMAC 签名短 token。生产系统还要考虑轮换、撤销、过期、审计和密钥管理。

签名：

```go
package auth

import (
	"crypto/hmac"
	"crypto/sha256"
	"encoding/base64"
)

func sign(secret []byte, payload string) string {
	mac := hmac.New(sha256.New, secret)
	mac.Write([]byte(payload))
	sum := mac.Sum(nil)
	return base64.RawURLEncoding.EncodeToString(sum)
}
```

验证：

```go
func validSignature(secret []byte, payload, signature string) bool {
	expected := sign(secret, payload)
	return hmac.Equal([]byte(expected), []byte(signature))
}
```

比较签名时使用 `hmac.Equal`，避免普通字符串比较带来的时序侧信道风险。

## 8. Cookie 安全属性

浏览器场景设置 cookie 时要关注：

```go
http.SetCookie(w, &http.Cookie{
	Name:     "session",
	Value:    sessionID,
	Path:     "/",
	HttpOnly: true,
	Secure:   true,
	SameSite: http.SameSiteLaxMode,
	MaxAge:   3600,
})
```

含义：

- `HttpOnly`：浏览器脚本不能读取。
- `Secure`：只通过 HTTPS 发送。
- `SameSite`：降低跨站请求风险。
- `MaxAge`：控制过期时间。

## 9. 学习检查

检查认证授权代码时，重点看：

1. 认证失败是否返回 `401`。
2. 权限不足是否返回 `403`。
3. 当前身份是否通过 context 传递。
4. context key 是否使用未导出类型。
5. token 校验是否通过接口隔离。
6. 资源归属检查是否放在 service。
7. cookie 是否设置 `HttpOnly`、`Secure`、`SameSite`。
