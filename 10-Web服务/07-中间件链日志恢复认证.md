# 中间件链：日志、恢复与认证入口

中间件处理横切逻辑：请求日志、panic 恢复、请求 ID、认证、压缩、CORS、超时等。它们不属于具体业务 handler，却要作用在很多路由上。

---

## 1. 中间件类型

在标准库里，中间件可以定义为：

```go
package httpx

import "net/http"

type Middleware func(http.Handler) http.Handler
```

一个中间件接收下一个 handler，返回一个新的 handler。

---

## 2. 组合中间件

```go
package httpx

import "net/http"

func Chain(handler http.Handler, middlewares ...Middleware) http.Handler {
	for i := len(middlewares) - 1; i >= 0; i-- {
		handler = middlewares[i](handler)
	}
	return handler
}
```

使用方式：

```go
handler := httpx.Chain(
	app.Handler(),
	httpx.RequestID(),
	httpx.Recover(logger),
	httpx.AccessLog(logger),
	httpx.Auth(authenticator),
)
```

数组顺序表示请求进入时的执行顺序。上面的请求顺序是 `RequestID -> Recover -> AccessLog -> Auth -> App`。这样 panic 恢复日志里也能读到 request ID。

## 3. 请求 ID 中间件

请求 ID 用来串联日志：

```go
package httpx

import (
	"context"
	"crypto/rand"
	"encoding/hex"
	"net/http"
)

type requestIDKey struct{}

func RequestID() Middleware {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			id := r.Header.Get("X-Request-ID")
			if id == "" {
				generated, err := newRequestID()
				if err != nil {
					WriteErrorWithCode(w, http.StatusInternalServerError, "internal_error", "failed to create request id")
					return
				}
				id = generated
			}

			w.Header().Set("X-Request-ID", id)
			ctx := context.WithValue(r.Context(), requestIDKey{}, id)
			next.ServeHTTP(w, r.WithContext(ctx))
		})
	}
}

func RequestIDFromContext(ctx context.Context) string {
	value, ok := ctx.Value(requestIDKey{}).(string)
	if !ok {
		return ""
	}
	return value
}

func newRequestID() (string, error) {
	var bytes [16]byte
	if _, err := rand.Read(bytes[:]); err != nil {
		return "", err
	}
	return hex.EncodeToString(bytes[:]), nil
}
```

context key 使用未导出的空结构体类型，可以避免和其他包冲突。

## 4. 捕获状态码的 ResponseWriter

访问日志需要知道最终状态码：

```go
package httpx

import "net/http"

type statusRecorder struct {
	http.ResponseWriter
	status int
	bytes  int
}

func (r *statusRecorder) WriteHeader(status int) {
	if r.status != 0 {
		return
	}
	r.status = status
	r.ResponseWriter.WriteHeader(status)
}

func (r *statusRecorder) Write(data []byte) (int, error) {
	if r.status == 0 {
		r.status = http.StatusOK
	}
	n, err := r.ResponseWriter.Write(data)
	r.bytes += n
	return n, err
}
```

如果 handler 没有显式调用 `WriteHeader`，首次 `Write` 会隐式返回 `200`。

## 5. 访问日志中间件

```go
package httpx

import (
	"log/slog"
	"net/http"
	"time"
)

func AccessLog(logger *slog.Logger) Middleware {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			start := time.Now()
			recorder := &statusRecorder{ResponseWriter: w}

			next.ServeHTTP(recorder, r)

			status := recorder.status
			if status == 0 {
				status = http.StatusOK
			}

			logger.Info(
				"http request",
				slog.String("method", r.Method),
				slog.String("path", r.URL.Path),
				slog.Int("status", status),
				slog.Int("bytes", recorder.bytes),
				slog.Duration("duration", time.Since(start)),
				slog.String("request_id", RequestIDFromContext(r.Context())),
			)
		})
	}
}
```

日志字段要稳定，后续检索和告警会依赖这些字段。

## 6. panic 恢复中间件

```go
package httpx

import (
	"log/slog"
	"net/http"
	"runtime/debug"
)

func Recover(logger *slog.Logger) Middleware {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			defer func() {
				value := recover()
				if value == nil {
					return
				}

				logger.Error(
					"panic recovered",
					slog.Any("panic", value),
					slog.String("stack", string(debug.Stack())),
					slog.String("method", r.Method),
					slog.String("path", r.URL.Path),
					slog.String("request_id", RequestIDFromContext(r.Context())),
				)
				WriteErrorWithCode(w, http.StatusInternalServerError, "internal_error", "internal server error")
			}()

			next.ServeHTTP(w, r)
		})
	}
}
```

panic 恢复是最后防线。业务错误仍然应该正常返回 error。

## 7. 认证入口中间件

认证中间件适合做成可替换依赖：

```go
package httpx

import (
	"context"
	"net/http"
	"strings"
)

type Principal struct {
	ID    string
	Roles []string
}

type Authenticator interface {
	Authenticate(ctx context.Context, token string) (Principal, error)
}

type principalKey struct{}

func WithPrincipal(ctx context.Context, principal Principal) context.Context {
	return context.WithValue(ctx, principalKey{}, principal)
}

func PrincipalFromContext(ctx context.Context) (Principal, bool) {
	principal, ok := ctx.Value(principalKey{}).(Principal)
	return principal, ok
}

func Auth(authenticator Authenticator) Middleware {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			header := r.Header.Get("Authorization")
			token, ok := strings.CutPrefix(header, "Bearer ")
			if !ok || token == "" {
				WriteErrorWithCode(w, http.StatusUnauthorized, "unauthorized", "missing bearer token")
				return
			}

			principal, err := authenticator.Authenticate(r.Context(), token)
			if err != nil {
				WriteErrorWithCode(w, http.StatusUnauthorized, "unauthorized", "invalid bearer token")
				return
			}

			ctx := WithPrincipal(r.Context(), principal)
			next.ServeHTTP(w, r.WithContext(ctx))
		})
	}
}
```

认证逻辑可以统一放在中间件中，具体 handler 只从 context 读取当前用户。

## 8. 学习检查

检查中间件时，重点看：

1. 中间件是否符合 `func(http.Handler) http.Handler`。
2. 顺序是否明确。
3. 请求 ID 是否写回响应头。
4. 日志是否记录 method、path、status、duration。
5. panic 是否被记录并返回 500。
6. 认证是否只解析凭证，授权是否留给具体业务。
7. 中间件是否避免修改不属于自己的业务状态。
