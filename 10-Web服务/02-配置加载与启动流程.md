# 配置加载与启动流程

Web 服务启动时要决定监听地址、超时、日志级别、数据库地址、关闭等待时间等参数。这些值不适合散落在代码里，应该集中到配置结构体中。

---

## 1. 配置结构体

先定义服务真正需要的配置：

```go
package config

import "time"

type Config struct {
	Addr              string
	LogLevel          string
	ReadHeaderTimeout time.Duration
	WriteTimeout      time.Duration
	ShutdownTimeout   time.Duration
	DatabaseURL       string
}
```

配置结构体应该表达业务服务的需求，而非完整复刻环境变量名称。环境变量只是来源之一。

---

## 2. 从环境变量加载

一个简单的加载函数：

```go
package config

import (
	"fmt"
	"os"
	"time"
)

func Load() (Config, error) {
	cfg := Config{
		Addr:              getenv("ADDR", ":8080"),
		LogLevel:          getenv("LOG_LEVEL", "info"),
		ReadHeaderTimeout: 5 * time.Second,
		WriteTimeout:      10 * time.Second,
		ShutdownTimeout:   10 * time.Second,
		DatabaseURL:       os.Getenv("DATABASE_URL"),
	}

	if raw := os.Getenv("READ_HEADER_TIMEOUT"); raw != "" {
		value, err := time.ParseDuration(raw)
		if err != nil {
			return Config{}, fmt.Errorf("parse READ_HEADER_TIMEOUT: %w", err)
		}
		cfg.ReadHeaderTimeout = value
	}

	if raw := os.Getenv("WRITE_TIMEOUT"); raw != "" {
		value, err := time.ParseDuration(raw)
		if err != nil {
			return Config{}, fmt.Errorf("parse WRITE_TIMEOUT: %w", err)
		}
		cfg.WriteTimeout = value
	}

	if raw := os.Getenv("SHUTDOWN_TIMEOUT"); raw != "" {
		value, err := time.ParseDuration(raw)
		if err != nil {
			return Config{}, fmt.Errorf("parse SHUTDOWN_TIMEOUT: %w", err)
		}
		cfg.ShutdownTimeout = value
	}

	if err := cfg.Validate(); err != nil {
		return Config{}, err
	}
	return cfg, nil
}

func getenv(key, fallback string) string {
	value := os.Getenv(key)
	if value == "" {
		return fallback
	}
	return value
}
```

`time.ParseDuration` 支持 `500ms`、`5s`、`2m` 这类格式，比自己解析数字安全。

---

## 3. 配置校验

配置加载后要尽早校验：

```go
package config

import "fmt"

func (c Config) Validate() error {
	if c.Addr == "" {
		return fmt.Errorf("ADDR is required")
	}
	if c.ReadHeaderTimeout <= 0 {
		return fmt.Errorf("READ_HEADER_TIMEOUT must be positive")
	}
	if c.WriteTimeout <= 0 {
		return fmt.Errorf("WRITE_TIMEOUT must be positive")
	}
	if c.ShutdownTimeout <= 0 {
		return fmt.Errorf("SHUTDOWN_TIMEOUT must be positive")
	}
	return nil
}
```

启动阶段暴露配置错误，比请求进来后才发现问题更容易排查。

## 4. 启动 http.Server

服务启动不要只调用 `http.ListenAndServe`。显式创建 `http.Server` 可以配置超时和关闭流程：

```go
package app

import (
	"log/slog"
	"net/http"
	"time"
)

type App struct {
	addr              string
	logger            *slog.Logger
	handler           http.Handler
	readHeaderTimeout time.Duration
	writeTimeout      time.Duration
	shutdownTimeout   time.Duration
}

func (a *App) server() *http.Server {
	return &http.Server{
		Addr:              a.addr,
		Handler:           a.handler,
		ReadHeaderTimeout: a.readHeaderTimeout,
		WriteTimeout:      a.writeTimeout,
	}
}
```

这里把监听地址、handler 和超时都放在 `App` 上，启动逻辑只需要向 `App` 要一个配置好的 `http.Server`。

## 5. 处理启动错误和关闭信号

完整的启动流程通常长这样：

```go
package app

import (
	"context"
	"errors"
	"fmt"
	"log/slog"
	"net/http"
	"os"
	"os/signal"
	"syscall"
)

func (a *App) Run() error {
	ctx, stop := signal.NotifyContext(context.Background(), os.Interrupt, syscall.SIGTERM)
	defer stop()

	server := a.server()
	errCh := make(chan error, 1)

	go func() {
		a.logger.Info("server listening", slog.String("addr", server.Addr))
		err := server.ListenAndServe()
		if err != nil && !errors.Is(err, http.ErrServerClosed) {
			errCh <- err
			return
		}
		errCh <- nil
	}()

	select {
	case err := <-errCh:
		return err
	case <-ctx.Done():
		a.logger.Info("shutdown signal received")
		shutdownCtx, cancel := context.WithTimeout(context.Background(), a.shutdownTimeout)
		defer cancel()

		if err := server.Shutdown(shutdownCtx); err != nil {
			return fmt.Errorf("shutdown server: %w", err)
		}
		a.logger.Info("server stopped")
		return nil
	}
}
```

这里有几个关键点：

- `ListenAndServe` 正常关闭时会返回 `http.ErrServerClosed`。
- 用 channel 把启动错误传回主 goroutine。
- 收到系统信号后调用 `Shutdown`，让正在处理的请求有机会完成。
- `Shutdown` 必须有超时，避免关闭流程无限等待。

## 6. main 中处理最终错误

`main` 不能返回 error，所以常见写法是把逻辑放进 `run`：

```go
package main

import (
	"log"
)

func main() {
	if err := run(); err != nil {
		log.Fatal(err)
	}
}

func run() error {
	cfg, err := config.Load()
	if err != nil {
		return err
	}

	application, err := buildApp(cfg)
	if err != nil {
		return err
	}

	return application.Run()
}
```

这种写法让启动流程可以被测试，也能让错误逐层包装。

## 7. 学习检查

检查一个 Web 服务启动代码时，重点看：

1. 配置是否集中加载。
2. 配置是否有默认值和校验。
3. 是否显式使用 `http.Server`。
4. 是否配置 `ReadHeaderTimeout`。
5. `ListenAndServe` 的错误是否处理。
6. 是否支持 `SIGINT` 和 `SIGTERM`。
7. `Shutdown` 是否带超时。
