# 04: 函数类型、匿名函数与闭包

好消息：这篇是你的主场。Go 里函数是"一等公民"——能赋给变量、能当参数、能当返回值，闭包也和 JS 一个原理。这篇真正要学的只有一件事：**Go 怎么给函数写类型**。

---

## 1. 函数类型

变量能装 `int`、装 `string`，也能装函数。装函数的变量，类型长这样：

```go
var op func(int, int) int
//     ───────┬─────── ─┬─
//      ①参数:两个 int   ②返回:一个 int
```

把签名里的名字都去掉，剩下的骨架就是函数类型。完整例子：

```go
func add(a, b int) int {
    return a + b
}

func main() {
    var op func(int, int) int
    op = add                 // 函数名不带括号 = 函数本身这个"值"

    fmt.Println(op(3, 4))
    fmt.Printf("%T\n", op)   // 打印 op 的类型
}
```

```text
7
func(int, int) int
```

对比 JS：`const op = add` 你写过无数遍。区别是 JS 变量什么函数都能装，Go 的 `op` **只能装签名完全一致的函数**——参数个数、类型、返回值，一个都不能差：

```go
var wrong func(int) int
wrong = add   // add 要两个参数,装不进"一个参数"的变量
```

```text
./main.go:11:10: cannot use add (value of type func(a int, b int) int) as func(int) int value in assignment
```

**一句话总结：函数类型 = 去掉名字的函数签名，签名不匹配就装不进去。**

---

## 2. 匿名函数

不起名字，当场定义、直接赋给变量：

```go
func main() {
    add := func(a, b int) int {
        return a + b
    }

    fmt.Println(add(3, 4))
}
```

```text
7
```

也可以定义完立刻调用（JS 老朋友 IIFE）：

```go
func main() {
    result := func(a, b int) int {
        return a + b
    }(3, 4)   // ← 定义完马上喂参数

    fmt.Println(result)
}
```

```text
7
```

对比 JS：`const add = (a, b) => a + b` 的 Go 版。注意 Go **没有箭头函数**，匿名函数只有 `func(...) {...}` 这一种写法，再短也不能省略 `func` 和大括号。

---

## 3. 函数作为参数

把"要做什么操作"作为参数传进去——JS 里的回调，原样搬过来：

```go
func apply(op func(int, int) int, a, b int) int {
//         ─────────┬──────────
//     第一个参数的类型是"函数类型",第 1 节的写法直接用在这
    return op(a, b)
}

func main() {
    add := func(a, b int) int {
        return a + b
    }

    fmt.Println(apply(add, 3, 4))
}
```

```text
7
```

这种写法常用于回调、策略、遍历处理等场景——你在 JS 里写 `arr.map(fn)` 时干的就是这件事。

## 4. 函数作为返回值

根据条件"生产"出不同的函数返回：

```go
func getOperator(op string) func(int, int) int {
//                          ────────┬────────
//                       返回值类型也是函数类型
    switch op {
    case "+":
        return func(a, b int) int { return a + b }
    case "-":
        return func(a, b int) int { return a - b }
    default:
        return func(a, b int) int { return 0 }
    }
}

func main() {
    add := getOperator("+")
    sub := getOperator("-")
    fmt.Println(add(3, 4))
    fmt.Println(sub(3, 4))
}
```

```text
7
-1
```

## 5. 闭包

闭包 = 函数 + 它出生时的环境。内部函数"记住"了外层的变量，即使外层函数已经返回：

```go
func counter() func() int {
    count := 0            // ← 被闭包捕获的变量
    return func() int {
        count++           // 每次调用都在改"同一个" count
        return count
    }
}

func main() {
    c := counter()
    fmt.Println(c())
    fmt.Println(c())
    fmt.Println(c())
}
```

```text
1
2
3
```

`counter` 早就返回了，按理说局部变量 `count` 该消失——但返回的函数还攥着它，Go 就让它活着。**机制和 JS 闭包完全一致**，你在 JS 里对闭包的全部直觉都能直接用。

每次调用 `counter()` 都创建一套全新的环境，多个闭包互不干扰：

```go
c1 := counter()
c2 := counter()
fmt.Println(c1(), c1(), c1())
fmt.Println(c2())
```

```text
1 2 3
1
```

`c2` 从 1 开始数——它有自己的 `count`，和 `c1` 的没关系。

---

## 6. 闭包常见用途

闭包适合携带少量"记住的状态"，比如预先固定一部分参数：

```go
func makePrefixer(prefix string) func(string) string {
    return func(s string) string {
        return prefix + s   // prefix 被记住了
    }
}

func main() {
    addID := makePrefixer("ID-")
    fmt.Println(addID("100"))
    fmt.Println(addID("101"))
}
```

```text
ID-100
ID-101
```

配一次 `prefix`，之后每次调用都自动带上——JS 里的偏函数/工厂函数套路，原样成立。

**一句话总结：闭包让函数带着自己的小仓库到处走，每次工厂调用发一个新仓库。**

---

## 本篇重点

- Go 的函数是值：可赋给变量、当参数、当返回值，和 JS 一致。
- 函数类型的写法是去掉名字的签名，如 `func(int, int) int`；签名不匹配是编译错误。
- Go 没有箭头函数，匿名函数只有 `func(...) {...}` 一种写法；支持定义后立即调用。
- 闭包机制与 JS 相同：内部函数捕获外层变量，外层返回后变量仍然活着。
- 每次调用工厂函数都产生独立的闭包环境，多个闭包互不影响。

```go
func makeCounter(start int) func() int
```

要求：

1. 第一次调用返回 `start + 1`。

3. 创建两个 counter，确认它们互不影响。
