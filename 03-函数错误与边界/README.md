# 03: 函数、错误与边界

本模块系统讲解 Go 的函数和错误处理。函数是 Go 程序组织逻辑的基本单位，错误处理则决定代码能否在失败场景下保持清晰、可恢复、可定位。

## 学习目标

完成本模块后，你应该能够：

- 熟练定义函数，理解参数、返回值和函数签名。
- 正确使用多返回值、命名返回值和可变参数。
- 理解值传递、slice/map 这类引用语义类型作为参数时的行为。
- 使用函数类型、匿名函数和闭包表达简单抽象。
- 按 Go 习惯返回和处理 `error`。
- 使用 `%w` 包装错误，并用 `errors.Is`、`errors.As` 判断错误。
- 正确使用 `defer` 清理资源，理解执行顺序和参数求值。
- 明确 `panic/recover` 的适用边界，不把普通业务错误写成 panic。
