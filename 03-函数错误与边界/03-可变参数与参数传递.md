# 03: 可变参数与参数传递

这篇解决两个实际问题：怎么写"想传几个传几个"的函数（JS 的 `...rest`），以及一个更要命的问题——**把变量传进函数后，函数改了它，外面的值到底变不变？** 判断错了，就是隐蔽的 bug。

---

## 1. 可变参数

参数类型前面加 `...`，这个参数就能收任意多个值：

```go
func sum(nums ...int) int {
//       ────┬─────
//   ①可收 0 到任意多个 int,必须是最后一个参数
    total := 0
    for _, n := range nums {
        // ② 函数体内,nums 就是一个 []int 切片
        total += n
    }
    return total
}
```

调用时传几个都行，不传也行：

```go
fmt.Println(sum(1, 2, 3))
fmt.Println(sum(1, 2, 3, 4, 5))
fmt.Println(sum())
```

```text
6
15
0
```

对比 JS：这就是 `function sum(...nums)` 的 rest 参数，连位置规则都一样——必须放在参数列表最后。区别只是 Go 要写元素类型：`...int` 说明收进来的每个值都得是 `int`。

其实你早就用过可变参数了：`fmt.Println` 能塞任意多个东西，就是因为它的参数是可变参数。

---

## 2. 传入 slice：用 `...` 展开

手里已经有一个切片（slice，Go 的动态数组），想把它喂给可变参数函数，在**后面**加 `...` 展开：

```go
nums := []int{1, 2, 3}
fmt.Println(sum(nums...))
```

```text
6
```

对比 JS：同一个思路，只是符号位置反了——JS 展开写在前面 `sum(...nums)`，Go 写在后面 `sum(nums...)`。

🕳️ 坑：忘了写 `...`，直接把切片塞进去，编译器会告诉你"我要的是 int，你给我一个 []int"：

```go
fmt.Println(sum(nums))
```

```text
./main.go:15:18: cannot use nums (variable of type []int) as int value in argument to sum
```

**一句话总结：声明用 `...int`（收），调用用 `nums...`（放），`...` 都挨着谁变谁。**

---

## 3. 值传递：函数拿到的是复印件

Go 的参数传递只有一种规则：**复制一份再交给函数**（值传递）。函数里怎么折腾，都是在折腾复印件：

```go
func change(n int) {
    n = 100   // 改的是复印件
}

func main() {
    x := 1
    change(x)
    fmt.Println(x)
}
```

```text
1
```

`x` 纹丝不动。传参那一刻，`x` 的值 `1` 被抄写给了 `n`，此后 `n` 和 `x` 各过各的。

对比 JS：JS 传数字、字符串时行为完全一样（原始类型按值传）。差异出现在下一节。

---

## 4. slice 参数：复印件里夹着"地址"

slice 传参也遵守值传递，但结果看起来像"传引用"：

```go
func change(nums []int) {
    nums[0] = 100
}

func main() {
    values := []int{1, 2, 3}
    change(values)
    fmt.Println(values)
}
```

```text
[100 2 3]
```

外面的 `values` 被改了！为什么值传递还能影响外面？

因为 slice 变量本身只是一张小小的"目录卡"（slice 头），真正的数据放在别处的底层数组里。传参时复制的是目录卡，**两张卡指向同一份数据**：

```text
values ──┐
         ├──→ [1, 2, 3]   ← 数据只有一份
nums ────┘      ↑
          nums[0]=100 改的是这份共享数据
```

所以：复制的是卡片，共享的是数据，改元素两边都看得见。

对比 JS：和 JS 传数组一个感觉——`function f(arr) { arr[0] = 100 }` 也会影响外面。JS 老手的直觉在这里直接复用。

---

## 5. map 参数：同理，改内容外面可见

map（键值对容器，类似 JS 的对象/Map）和 slice 一样，变量本身只是指向底层数据的"把手"：

```go
func change(m map[string]int) {
    m["count"] = 100
}

func main() {
    data := map[string]int{"count": 1}
    change(data)
    fmt.Println(data)
}
```

```text
map[count:100]
```

**一句话总结：传的永远是复印件；但 slice 和 map 的复印件里带着数据的地址，所以改"内容"外面看得见。**

---

## 6. 指针参数：让函数能改普通变量

第 3 节里 `int` 是纯粹的复印件，函数改不动外面。如果**就是想**让函数修改外面的普通变量，办法是把变量的**地址**传进去（指针）：

```go
func change(n *int) {   // *int:参数是"一个 int 的地址"
    *n = 100            // 顺着地址找到本体,改它
}

func main() {
    x := 1
    change(&x)          // &x:取 x 的地址
    fmt.Println(x)
}
```

```text
100
```

现在 `x` 真的变了——函数拿到的复印件是"地址"，顺着地址摸到的是本体。

`&`（取地址）和 `*`（按地址取值）是新面孔，这里先混个脸熟，**指针在后面的章节会专门细讲**，现在只需要记住结论：看到函数参数是 `*类型`，说明它有能力修改你传进去的变量。

---

## 各类型传参行为对照

| 参数类型 | 函数内修改 | 外面变不变 | 原因 |
|---------|-----------|-----------|------|
| `int` / `string` / `bool` | `n = 100` | 不变 | 复制的就是值本身 |
| `[]int`（slice） | `nums[0] = 100` | **变** | 复制的是目录卡，数据共享 |
| `map[string]int` | `m["k"] = 100` | **变** | 复制的是把手，数据共享 |
| `*int`（指针） | `*n = 100` | **变** | 复制的是地址，顺着改本体 |

---

## 本篇重点

- 可变参数：声明写 `...int`，函数体内它就是 `[]int`；调用时展开已有切片写 `nums...`（和 JS 的 `...` 位置相反）。
- Go 只有值传递：传参 = 复制一份。
- `int` 等基本类型是纯复印件，函数改不动外面。
- slice 和 map 的复印件里带着底层数据的地址，函数改"内容"外面看得见——和 JS 传数组/对象的直觉一致。
- 想让函数修改普通变量，传指针 `&x`（指针后面会细讲）。

## 练习

实现两个函数：

```go
func addOne(n int)
func addOnePtr(n *int)
```

要求：

1. 调用 `addOne(x)` 后，外部 `x` 不变。
2. 调用 `addOnePtr(&x)` 后，外部 `x` 加 `1`。
3. 解释这两个函数行为不同的原因。

提示：对照第 3 节和第 6 节；`addOnePtr` 里给谁加 1——`n` 还是 `*n`？
