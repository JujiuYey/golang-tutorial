# 02: 多返回值与命名返回值

JS 函数想返回多个值，得包一层数组或对象再解构；Go 函数**原生**就能返回多个值。学完这篇，你能看懂 Go 代码里满屏的 `value, err := doSomething()`，也能识破"命名返回值 + 裸 return"这个新手最容易懵的写法。

---

## 1. 多返回值

返回值类型从一个变成一组，用括号括起来：

```go
func divide(a, b int) (int, int) {
//                    ────┬─────
//              两个返回值:商 和 余数
    return a / b, a % b
//         ──┬──  ──┬──
//          第1个   第2个,顺序对应
}
```

调用时用几个变量接几个值：

```go
func main() {
    quotient, remainder := divide(10, 3)
    fmt.Println(quotient, remainder)
}
```

```text
3 1
```

对比 JS：这相当于 `const [quotient, remainder] = divide(10, 3)`——只是 Go 不需要先把结果塞进数组再解构，语言直接支持"一个操作产生多个结果"。

🕳️ 坑：接收变量的数量必须和返回值数量**完全对上**。想只用一个变量接住两个返回值，编译器不干：

```go
x := divide(10, 3)
```

```text
./main.go:10:7: assignment mismatch: 1 variable but divide returns 2 values
```

JS 解构少写一个变量顶多丢个值，Go 这里是硬性的编译错误。不需要的值怎么办？看第 3 节。

---

## 2. 返回结果和错误

多返回值最重要的用途：**把"结果"和"这事成没成"一起交出来**。约定是错误放最后一个：

```go
func safeDivide(a, b int) (int, error) {
    if b == 0 {
        return 0, fmt.Errorf("除数不能为 0")
    }
    return a / b, nil   // nil 表示"没出错"
}
```

调用方拿到两个值，先检查错误再用结果：

```go
result, err := safeDivide(10, 0)
if err != nil {
    fmt.Println("error:", err)
    return
}
fmt.Println(result)
```

```text
error: 除数不能为 0
```

对比 JS：JS 靠 `throw` + `try/catch` 把错误抛出去；Go 把错误当成**普通返回值递给你**，你必须当场表态。这是 Go 最核心的错误处理习惯，第 05 篇会展开细讲。

---

## 3. 忽略不需要的返回值

确实用不到某个返回值，用 `_`（下划线，空标识符）扔掉：

```go
quotient, _ := divide(10, 3)   // 只要商,不要余数
fmt.Println(quotient)
```

```text
3
```

`_` 就是个"垃圾桶变量"，接了值但永远取不出来。这解决了第 1 节的坑：变量数量必须匹配，不想要的位置就用 `_` 占住。

🕳️ 坑：`_` 可以扔余数，但**不要顺手扔 error**：

```go
result, _ := safeDivide(10, 0)   // 不推荐:出错了你也不知道
```

除法失败了，`result` 是 0，程序却继续往下跑——错误被静默吞掉。忽略 error 之前，必须能说清"为什么这里可以忽略"。

## 4. 命名返回值

返回值可以命名：

```go
func calc(a, b int) (sum int, product int) {
    sum = a + b
    product = a * b
    return
}
```

第一眼看不懂很正常，这里有两个陌生动作。拆开看：

### 陌生点 ①：签名里的 `(sum int, product int)` 是在"预先声明变量"

普通写法 `(int, int)` 只说了"返回两个 int"。命名写法等于**在函数最开头偷偷帮你声明好了两个变量**，可以脑补成：

```go
func calc(a, b int) (int, int) {
    var sum int       // ← 签名里的命名，效果等于这两行
    var product int   //   一进函数就存在，且已是零值 0
    ...
}
```

所以函数体里写的是 `sum = a + b`（赋值 `=`），**不是** `sum := a + b`——变量已经被签名声明过了，再用 `:=` 反而会出问题。

用打印验证"一进函数就存在且为零值"：

```go
func calc(a, b int) (sum int, product int) {
    fmt.Println("刚进函数时:", sum, product) // 输出：刚进函数时: 0 0
    sum = a + b
    product = a * b
    return
}
```

### 陌生点 ②：光秃秃的 `return` 是缩写

因为返回值已经有名字了，`return` 不带任何东西时，Go 自动理解为"把当前的 sum 和 product 返回出去"：

```go
return            // 等价于 ↓
return sum, product
```

这叫**裸返回（naked return）**。所以整个函数的执行流程是：

```text
调用 calc(3, 4)
  → sum=0, product=0（签名声明，零值）
  → sum = 7
  → product = 12
  → return（= return sum, product）
  → 调用方拿到 7, 12
```

### 零值直接可用

正因为命名返回值一开始就是零值，这样的函数也合法：

```go
func defaultValue() (n int) {
    return
}
```

`defaultValue()` 返回 `0`——n 声明了、没赋值、裸返回，返回的就是零值。

## 5. 什么时候使用命名返回值

命名返回值适合：

- 返回值含义不明显，名字本身就是文档（比如 `(lat, lng float64)` 比 `(float64, float64)` 清楚得多）。
- 函数很短，裸 `return` 一眼能看全。
- 配合 `defer` 在函数返回前修改返回的 error（第 07 篇 defer 会细讲这个用法，现在知道有这回事就行）。

不适合：

- 函数很长——读到底部的 `return` 时，早忘了返回的是什么。
- 返回值很多——名字越多越记不住。

拿不准时，**显式返回永远不会错**：

```go
func calc(a, b int) (int, int) {
    sum := a + b
    product := a * b
    return sum, product   // 返回什么,一目了然
}
```

**一句话总结：命名返回值是可选的便利，裸 return 是它的缩写；短函数可以用，长函数请显式返回。**

---

## 本篇重点

- Go 函数原生支持多返回值，调用时相当于 JS 的数组解构，但变量数量必须严格匹配（否则 `assignment mismatch` 编译错误）。
- 最常见的形态是 `(结果, error)`，错误约定放在最后一个返回值。
- 不需要的返回值用 `_` 丢弃，但 error 不要随手丢。
- 命名返回值 = 签名里预先声明好的变量（一进函数就是零值），函数体里用 `=` 赋值而不是 `:=`。
- 裸 `return` 是 `return 所有命名返回值` 的缩写，只建议在短函数里用。
