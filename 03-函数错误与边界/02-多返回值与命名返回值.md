# 02: 多返回值与命名返回值

Go 函数可以返回多个值。最常见的模式是返回结果和错误：

```go
value, err := doSomething()
```

---

## 1. 多返回值

```go
func divide(a, b int) (int, int) {
    return a / b, a % b
}
```

调用：

```go
func main() {
    quotient, remainder := divide(10, 3)
    fmt.Println(quotient, remainder)
}
```

多返回值让 Go 代码可以直接表达“一个操作产生多个结果”。

---

## 2. 返回结果和错误

```go
func safeDivide(a, b int) (int, error) {
    if b == 0 {
        return 0, fmt.Errorf("除数不能为 0")
    }
    return a / b, nil
}
```

调用方必须检查错误：

```go
result, err := safeDivide(10, 0)
if err != nil {
    fmt.Println("error:", err)
    return
}
fmt.Println(result)
```

这是 Go 最核心的错误处理习惯。

---

## 3. 忽略不需要的返回值

如果确实不需要某个返回值，可以用 `_`。

```go
quotient, _ := divide(10, 3)
fmt.Println(quotient)
```

但不要随便忽略 `error`：

```go
// result, _ := safeDivide(10, 0) // 不推荐
```

---

## 4. 命名返回值

返回值可以命名：

```go
func calc(a, b int) (sum int, product int) {
    sum = a + b
    product = a * b
    return
}
```

命名返回值在函数开始时会被声明并初始化为零值。

```go
func defaultValue() (n int) {
    return
}
```

`defaultValue()` 返回 `0`。

---

## 5. 什么时候使用命名返回值

命名返回值适合：

- 返回值含义不明显，需要名字说明。
- 函数很短，裸 `return` 不会降低可读性。
- 配合 `defer` 修改返回错误。

不适合：

- 函数很长。
- 返回值很多。
- 裸 `return` 会让读者不知道返回了什么。

更清晰的写法通常是显式返回：

```go
func calc(a, b int) (int, int) {
    sum := a + b
    product := a * b
    return sum, product
}
```

## 练习

实现函数：

```go
func divmod(a, b int) (int, int, error)
```

要求：

1. 返回商、余数和错误。
2. `b == 0` 时返回错误。
3. 正常情况返回 `nil` 错误。
