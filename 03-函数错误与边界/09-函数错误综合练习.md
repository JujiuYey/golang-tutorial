# 09: 函数与错误综合练习

学完这个模块，是时候把零件组装起来了。这 5 道练习会把函数定义、多返回值、错误处理、defer、panic/recover 串成完整的肌肉记忆——每道题都对应前面某一篇，卡住了就回去翻。

---

## 本篇重点（模块回顾）

动手之前，扫一眼这个模块你应该已经带走的东西：

- 函数签名：参数、返回值都要类型，多返回值用 `(值, error)` 收尾（01、02 篇）。
- 错误是普通返回值：`if err != nil` 当场处理，不随手用 `_` 吞掉（05 篇）。
- 错误链：`%w` 包装保链，`errors.Is` 判断根源（06 篇）。
- defer：资源到手后立刻预约清理，注意参数当场求值（07 篇）。
- 分工：预料中的失败返回 error，不该发生的事才 panic，边界处 recover 转 error（08 篇）。

下面每道题做完，对照这张单子看看用上了哪几条。

---

## 练习1: 阶乘函数

实现函数：

```go
func factorial(n int) (int, error)
```

要求：

1. `n < 0` 时返回错误。
2. `0` 和 `1` 返回 `1`。
3. 正常返回阶乘结果。
4. 调用方必须检查错误。

提示：考察 02、05 篇。错误信息里带上 `n` 的实际值；在 `main` 里用 `-1`、`0`、`5` 各测一次。

---

## 练习2: 安全除法

实现函数：

```go
func safeDivide(a, b float64) (float64, error)
```

要求：

1. `b == 0` 时返回错误。
2. 正常返回计算结果和 `nil`。
3. 调用方打印错误或结果。

提示：考察 02、05 篇。出错分支的第一个返回值给什么？想想零值。

---

## 练习3: 文件读取

实现函数：

```go
func readFile(path string) ([]byte, error)
```

要求：

1. 使用 `os.Open` 打开文件。
2. 使用 `defer` 关闭文件。
3. 使用 `io.ReadAll` 读取内容。
4. 所有错误都要带上下文。

提示：考察 07 篇。`defer file.Close()` 的位置在错误检查之前还是之后？先在本地建一个测试文件，再故意传一个不存在的路径，两条路都要走通。

---

## 练习4: 错误判断

实现函数：

```go
func readRequiredFile(path string) ([]byte, error)
```

要求：

1. 文件不存在时返回包装后的错误。
2. 调用方使用 `errors.Is(err, os.ErrNotExist)` 判断文件不存在。
3. 打印不同的错误提示。

提示：考察 06 篇。包装用 `%w` 而不是 `%v`——不确定为什么的话，回去看 06 篇第 2 节那个 `true / false` 对比。

---

## 练习5: 计数器闭包

实现函数：

```go
func makeCounter(start int) func() int
```

要求：

1. 第一次调用返回 `start + 1`。
2. 每次调用都在上一次基础上加 `1`。
3. 创建两个 counter，确认它们互不影响。

提示：考察 04 篇。返回值类型是函数类型；被闭包捕获的变量声明在返回的匿名函数外面还是里面，结果完全不同——两种都试试。

---

## 验证建议

每个练习都建议运行：

```bash
go run .
```

对于纯函数（练习 1、2、5），可以尝试写测试：

```bash
go test ./...
```

不满足于"能跑"：每道题至少构造一个成功输入和一个失败输入，确认两条路径的输出都符合预期——错误路径没测过的代码，等于没写完。
