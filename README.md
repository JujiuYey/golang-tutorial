# AI 时代的 Go 学习路线

这是一套面向 AI 时代的 Go 系统教程。它不是只带你过一遍语法，而是训练你建立一套判断系统：看懂 Go 代码、解释 Go 机制、拆解真实问题、验证运行结果，并审查 AI 生成的代码是否简单、正确、可维护、可测试。

当前仓库以 Markdown 教程为主，包含 11 个学习模块和 2 个综合项目。你可以把它当成一条从基础语法到 Web 服务、数据库、工程化验证的长期学习路线。

---

## 适合谁学习

- 想系统学习 Go，但不想只停留在语法记忆的人。
- 已经会一点 Go，希望补齐错误处理、并发、接口、测试和工程化的人。
- 经常使用 AI 写代码，希望自己能审查、追问、验证和改进 AI 输出的人。
- 想用 Go 做 CLI、HTTP API、数据库服务、Web 服务或后端工程实践的人。

---

## 学完后你应该能做到

1. 读懂常见 Go 项目的结构、包边界、数据流和错误路径。
2. 用自己的话解释 Go 的类型、函数、错误、slice、map、interface、goroutine、channel、context 等核心机制。
3. 判断 AI 生成的 Go 代码是否符合 Go 的惯用方式。
4. 识别常见坏味道：忽略错误、过度抽象、滥用 goroutine、丢失 context、资源泄漏、测试薄弱。
5. 使用 `go run`、`go test`、`go vet`、`go test -race`、curl、日志和最小复现验证代码。
6. 和 AI 协作完成 CLI、HTTP API、数据库、并发任务等真实项目。

## 推荐学习方法：解释、拆解、复习、验证

学习 Go 不要只追求“看完”。真正可靠的掌握，来自四件事：能讲清、能拆小、能复习、能验证。

### 1. 费曼学习法：讲得出来才算懂

每学完一篇文章，先合上内容，用自己的话回答四个问题：

1. 这个概念是什么？
2. 它解决什么问题？
3. 它最容易被写错在哪里？
4. 如果 AI 写错了，我要怎么发现？

如果你讲不清楚，不要急着进入下一篇。回到原文和代码，把卡住的地方补上，再重新解释一遍。

### 2. 西蒙式拆解：把大主题拆成小技能块

不要把“并发”“Web 服务”“数据库”当成一个整体硬啃。把它们拆成可以单独练习的小块。

比如学习并发时，可以按这个顺序拆：

1. goroutine 如何启动和退出。
2. `sync.WaitGroup` 如何等待任务完成。
3. channel 什么时候发送、接收、阻塞和关闭。
4. `select` 如何处理多个事件和超时。
5. `context.Context` 如何传递取消信号。
6. `Mutex`、`RWMutex`、`atomic` 如何保护共享状态。
7. `go test -race` 如何发现数据竞争。

每次只练一个小块，确认自己能解释、能写最小代码、能验证结果，再把小块组合成项目能力。

### 3. 遗忘曲线：用间隔复习防止学完就忘

每篇文章建议按这个节奏复习：

| 时间 | 复习方式 |
|------|----------|
| 当天 | 合上文章复述核心概念，写一个最小例子 |
| 第 2 天 | 不看原文，重新写一次关键代码 |
| 第 7 天 | 回答本节最容易出错的 3 个问题 |
| 第 14 天 | 把这个知识点放进一个小练习或项目片段里 |

复习时不要一上来重读全文。先尝试回忆、解释和写代码，卡住了再回看。这样更容易发现自己到底哪里没掌握。

### 4. 主动验证：用命令证明自己真的会

Go 的优势之一是反馈很直接。不要只相信“我感觉懂了”，尽量用命令验证：

```bash
go run .
go test ./...
go vet ./...
go test -race ./...
```

网络和 Web 服务相关内容，可以继续用 curl、日志和 `httptest` 验证请求、响应、错误路径和超时行为。

## 每篇文章的学习循环

建议每篇文章都按同一个节奏学习：

1. **读**：先读概念、规则和示例。
2. **讲**：合上文章，用自己的话解释给一个刚学 Go 的人听。
3. **写**：写一个最小 Go 程序验证概念。
4. **改**：故意改错，观察编译错误、测试失败或运行结果。
5. **测**：用 `go test`、`go vet`、`go test -race`、curl 或日志验证。
6. **审**：让 AI 写一版类似代码，检查错误处理、边界、资源释放、并发安全和测试覆盖。
7. **记**：记录一句话：“我现在能判断哪类 Go 代码问题？”

## 稳扎稳打学习计划

如果你是边工作边学习，建议按 8 到 12 周推进。每周不追求刷很多篇，而是确保每个主题都能解释、写出最小例子，并完成复习。

### 第 1-2 周：工具链与基础语法

- 学习 01、02。
- 重点掌握 Go 环境、`go run`、`go test`、变量、常量、类型转换、字符串、条件、循环。
- 每天练 1 到 2 个小例子。
- 复习目标：能解释零值、作用域、变量遮蔽、`range` 的基本行为。

### 第 3-4 周：函数、错误、数据结构

- 学习 03、04。
- 重点掌握函数签名、多返回值、错误返回、`defer`、slice、map、struct、pointer。
- 每个知识点都写一个最小复现，主动制造错误再修正。
- 复习目标：能判断一段代码到底复制了什么、共享了什么、错误有没有被正确处理。

### 第 5 周：接口、组合与泛型

- 学习 05。
- 重点掌握小接口、隐式实现、接口值 nil、组合、泛型函数、类型约束。
- 不要急着抽象。先写具体代码，再判断什么时候需要接口或泛型。
- 复习目标：能识别 AI 代码里的过度抽象和没必要的 interface。

### 第 6 周：并发模型

- 学习 06。
- 重点掌握 goroutine 生命周期、channel 关闭、`select`、`context`、锁和 race detector。
- 每个并发例子都问：谁退出？谁关闭？谁取消？共享状态有没有保护？
- 复习目标：能识别 goroutine 泄漏、数据竞争、死锁和错误的 channel 使用方式。

### 第 7 周：标准库

- 学习 07。
- 重点掌握文件 IO、JSON、时间、命令行参数、结构化日志、测试工具。
- 建立“遇到问题先找标准库”的习惯。
- 复习目标：能用标准库完成常见文本、文件、JSON、时间和测试任务。

### 第 8-9 周：网络、数据库与 Web 服务

- 学习 08、09、10。
- 重点掌握 HTTP handler、client timeout、JSON API、数据库连接池、事务、repository、Web 服务分层。
- 多写 `httptest`、curl 验证和错误路径测试。
- 复习目标：能审查一个 API 是否正确处理状态码、超时、context、SQL 参数、事务和认证边界。

### 第 10 周：工程化与验证

- 学习 11。
- 重点掌握 `gofmt`、`go vet`、表驱动测试、覆盖率、race detector、benchmark、CI。
- 把验证命令整理成自己的本地检查清单。
- 复习目标：能用工具链证明代码质量，而不是只靠口头判断。

### 第 11-12 周：综合项目

- 完成 Project 01 和 Project 02。
- 每个项目都按“需求拆解 -> 最小实现 -> 测试验证 -> AI 审查 -> 重构整理”的节奏推进。
- 不要求一次写完，优先保证每一步都能运行、能测试、能解释。
- 复习目标：能把前面学到的语法、错误处理、结构设计、并发、数据库、Web 服务和工程化验证串起来。

## 和 AI 一起学习的方式

AI 可以帮你写代码、解释概念、生成测试、制造反例，但不要把判断权完全交出去。建议每次使用 AI 时，都追问这些问题：

1. 这段代码有没有忽略错误？
2. 资源有没有关闭？
3. context 有没有传下去？
4. 并发代码有没有退出条件？
5. slice、map、pointer 有没有共享状态陷阱？
6. interface 是真的需要，还是为了显得高级？
7. 测试是否只覆盖 happy path？
8. 我能不能用命令证明它真的工作？

学习 Go 的目的，不只是写出能运行的代码，而是在使用 AI 的同时保留自己的判断权。
