# AI时代下，我为什么推荐你掌握 Golang

AI 已经能生成大量代码，学习 Go 的目标也跟着变了。今天更稀缺的能力，是判断一段 Go 代码写得是否简单、正确、可维护、可测试，以及是否真的符合 Go 的惯用方式。

本课程的目标是帮助你建立一套 Go 代码判断系统：看懂 AI 写出来的 Go，识别其中的问题，要求 AI 做出更好的修改，并用工程化手段验证它真的可用。

---

## 学习目标

学完这套课程后，你应该能够做到：

1. 读懂常见 Go 项目的结构、包边界、数据流和错误路径。
2. 判断 AI 生成的 Go 代码是否符合 Go 的语言习惯。
3. 识别常见坏味道：忽略错误、过度抽象、滥用 goroutine、没有 context、资源泄漏、测试薄弱。
4. 使用 `go test`、`go vet`、`go test -race`、curl、日志和最小复现验证代码。
5. 和 AI 协作完成 CLI、HTTP API、数据库、并发任务等真实项目。

---

## 四层学习框架

### 1. 为什么在 AI 时代学习 Go

Go 的语法简单、标准库强、并发模型明确、部署方式直接，工程约束也足够清晰。这些特点让 Go 很适合被审查和验证。AI 写 Go 时很容易写出“看起来完整”的代码，Go 的简洁性让你更容易看穿这些代码是不是真的靠谱。

### 2. Go 的核心心智模型

你需要掌握包、module、类型系统、struct、interface、error、slice/map、指针、context、goroutine/channel、测试等核心概念。重点放在机制本身：它解决什么问题，适合什么场景，滥用时会出现什么后果。

### 3. AI 生成代码的审查能力

每个主题都要带着审查视角学习：AI 是否乱抽 interface？是否忽略 error？是否为了“高级感”滥用 goroutine？是否丢失 context？是否忘记关闭资源？是否只写了 happy path 测试？

### 4. 项目级验证能力

真正的掌握要落到验证上。每个项目都要能够被构建、运行、测试和观察。你要习惯用命令和事实判断代码，少相信口头解释，多相信可重复的结果。

## 建议学习方式

每一章都按下面的顺序学习：

1. 先理解本章为什么在 AI 时代仍然重要。
2. 再掌握本章对应的 Go 心智模型。
3. 主动找出示例代码中可以被 AI 写坏的地方。
4. 用测试、命令、日志或 curl 验证你的判断。
5. 最后把同一个需求交给 AI 实现，再按清单审查它。

学习 Go 的目的，是在使用 AI 的同时保留自己的判断权。
