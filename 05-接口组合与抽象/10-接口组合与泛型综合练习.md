# 10: 接口组合与泛型综合练习

这一篇没有新知识,解决的是"学了一模块,手上到底会不会"的问题:把接口、组合、泛型放进 8 道题里连起来练。和前面照着示例敲不同,这次**每个抽象都要你自己选、自己说明为什么**——上一篇的判断表就是答题卡。

建议节奏:先自己写,卡住了再回对应篇目翻,写完对照每题的"要求"逐条自查。

---

## 本篇重点

动手前扫一遍,这 8 道题分别在验收什么:

- [ ] 接口的隐式实现与编译期检查(练习 1)——第 01 篇。
- [ ] 小接口拆分、调用方定义接口(练习 2、7)——第 04、05 篇。
- [ ] typed nil error 的复现与修复(练习 3)——第 02 篇的头号坑。
- [ ] 泛型函数、comparable 约束、泛型容器与零值可用(练习 4、5、6)——第 06-08 篇。
- [ ] 接口 vs 泛型的选择判断(练习 8)——第 09 篇。

## 练习1: Shape 接口

定义：

```go
type Shape interface {
    Area() float64
    Perimeter() float64
}
```

实现：

```go
type Circle struct {
    Radius float64
}

type Rectangle struct {
    Width  float64
    Height float64
}
```

要求：

1. `Circle` 和 `Rectangle` 都满足 `Shape`。
2. 实现 `TotalArea(shapes []Shape) float64`。
3. 写编译期检查：`var _ Shape = Circle{}`。
4. 解释为什么这里适合用接口。

提示：第 4 问用第 09 篇的判断表回答——`TotalArea` 关心的是行为还是类型?

## 练习2: 小接口拆分

已有大接口：

```go
type UserRepository interface {
    Find(id int64) (User, error)
    Save(user User) error
    Delete(id int64) error
    List() ([]User, error)
}
```

要求：

1. 拆成 `UserFinder`、`UserSaver`、`UserDeleter`。
2. 写函数 `Register(saver UserSaver, user User) error`。
3. 写函数 `Profile(finder UserFinder, id int64) (ProfileView, error)`。
4. 说明为什么调用方只接收小接口。

提示：拆完后想一想——原来那个实现了 `UserRepository` 的类型,需要为这三个小接口改一行代码吗?(隐式实现,第 01 篇)

## 练习3: typed nil error

定义一个指针错误类型：

```go
type AppError struct {
    Message string
}

func (e *AppError) Error() string {
    return e.Message
}
```

要求：

1. 写函数复现 typed nil error。
2. 打印 `err == nil` 的结果。
3. 修改函数，确保没有错误时返回真正的 nil。

提示：关键是让 `*AppError` 类型的 nil 变量"过一遍手"再 return(第 02 篇第 4 节);修复口诀——要 nil 就返回字面量 nil。

## 练习4: 泛型 First 和 Filter

实现：

```go
func First[T any](items []T) (T, bool)
func Filter[T any](items []T, keep func(T) bool) []T
```

要求：

1. `First` 支持 `[]int`、`[]string`、`[]User`。
2. 空 slice 时返回零值和 `false`。
3. `Filter` 返回值保持原元素类型。
4. 说明为什么 `[]any` 版本不合适。

提示：第 2 问用 `var zero T`;第 3 问用 `%T` 打印返回值验证;第 4 问回忆第 03 篇 `[]int` 传不进 `[]any` 的编译错误。

## 练习5: 泛型 Contains

实现：

```go
func Contains[T comparable](items []T, target T) bool
```

要求：

1. 支持 `[]int` 和 `[]string`。
2. 尝试传入 `[][]int`，观察编译错误。
3. 解释 `comparable` 约束的作用。

提示：第 2 问的报错应该含有 `does not satisfy comparable`——想想 slice 为什么不可比较(第 04 模块)。

## 练习6: 泛型 Set

实现：

```go
type Set[T comparable] struct {
    items map[T]struct{}
}
```

方法：

```go
func (s *Set[T]) Add(v T)
func (s *Set[T]) Delete(v T)
func (s *Set[T]) Has(v T) bool
func (s *Set[T]) Len() int
```

要求：

1. 零值 `var s Set[string]` 可以直接 `Add`。
2. `Has` 不修改内部状态。
3. `T` 使用 `comparable`。
4. 解释为什么 map 的 key 约束要求可比较。

提示：第 1 条是本模块埋过的雷——nil map 写入会 panic(第 08 篇第 6 节),先想清楚 `Add` 为什么必须是指针接收者。

## 练习7: 组合一个服务

定义接口：

```go
type Store interface {
    FindUser(id int64) (User, error)
}

type Logger interface {
    Log(message string)
}
```

定义：

```go
type Service struct {
    Store  Store
    Logger Logger
}
```

实现：

```go
func (s Service) Profile(id int64) (ProfileView, error)
```

要求：

1. `Profile` 从 Store 读取用户。
2. 成功时写一条日志。
3. Store 报错时返回错误。
4. 写 fake store 和 fake logger 验证逻辑。

提示：fake logger 用一个 `[]string` 字段记录收到的消息(参考第 04 篇的 fakeMailer),验证时检查这个 slice;fake 要改自己的字段,注意接收者类型。

## 练习8: 接口还是泛型

对下面问题分别判断应该用接口、泛型、具体类型还是 `any`：

1. 从任意 `io.Reader` 读取全部内容。
2. 对 `[]int`、`[]string`、`[]User` 取第一个元素。
3. 保存用户到数据库。
4. 日志函数接收任意字段值并打印。
5. 对多种有序类型求最小值。

每题写出理由。

提示：拿第 09 篇第 9 节的判断表逐行对;第 5 题别忘了约束该用哪个、要不要带 `~`。

## 验证建议

把练习代码放到一个小 module 里，用测试固定行为：

```bash
go test ./...
```

对接口题，测试重点是替身实现是否容易写——fake 写着费劲,通常说明接口拆得不够小。对泛型题，测试重点是不同类型实例化后返回值类型是否正确——`%T` 是最快的验证手段。
