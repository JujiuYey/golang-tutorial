# 05: 接口、组合与泛型

本模块讲 Go 的抽象能力：接口、组合和泛型。接口用于描述行为，泛型用于描述一组类型上的通用算法。二者都和 `interface` 有关，但解决的问题不同。

## 学习目标

完成本模块后，你应该能够：

- 理解 Go 接口的隐式实现机制。
- 判断一个类型的方法集合是否满足某个接口。
- 说清楚接口值中的动态类型和动态值。
- 识别 typed nil 放进接口后的常见陷阱。
- 使用 `any`、类型断言和 type switch 处理动态类型。
- 通过小接口和接口组合表达行为边界。
- 理解组合和嵌入，不把它们误当成继承。
- 编写基本的泛型函数和泛型类型。
- 使用类型约束、类型集合、`~` 和 `comparable`。
- 判断同一个问题更适合用接口、泛型、普通函数还是具体类型。

---

## 学习建议

这一章建议把“抽象”拆成两个问题：

1. 调用方需要什么行为。
2. 被调用方是否需要知道具体类型。

如果只需要某个行为，通常考虑接口。如果算法要保留输入输出的具体类型，通常考虑泛型。如果只有一个具体类型会用到，先写具体代码。
