# 06: 泛型函数与类型参数

泛型让函数和类型可以接受类型参数。它适合表达“同一段逻辑适用于多种具体类型，并且希望保留类型信息”的场景。

---

## 1. 一个没有泛型的问题

```go
func FirstInt(items []int) (int, bool) {
    if len(items) == 0 {
        return 0, false
    }
    return items[0], true
}

func FirstString(items []string) (string, bool) {
    if len(items) == 0 {
        return "", false
    }
    return items[0], true
}
```

两个函数逻辑一样，只是元素类型不同。

---

## 2. 泛型函数

```go
func First[T any](items []T) (T, bool) {
    var zero T
    if len(items) == 0 {
        return zero, false
    }
    return items[0], true
}
```

`T` 是类型参数。`any` 是约束，表示 T 可以是任意类型。

调用：

```go
n, ok := First([]int{1, 2, 3})
s, ok := First([]string{"go", "rust"})
```

编译器通常能从参数推断出 `T`，所以不需要手写 `First[int]`。

---

## 3. 显式类型实参

有时可以显式写出类型：

```go
n, ok := First[int]([]int{1, 2, 3})
```

多数情况下省略即可：

```go
n, ok := First([]int{1, 2, 3})
```

如果编译器无法推断类型，就需要显式写出类型实参。

---

## 4. 多个类型参数

```go
func Pair[A any, B any](a A, b B) (A, B) {
    return a, b
}
```

调用：

```go
name, age := Pair("alice", 18)
fmt.Println(name, age)
```

也可以合并相同约束：

```go
func Pair[A, B any](a A, b B) (A, B) {
    return a, b
}
```

---

## 5. 类型参数的零值

不知道 T 的具体类型时，可以用 `var zero T` 得到零值。

```go
func Last[T any](items []T) (T, bool) {
    var zero T
    if len(items) == 0 {
        return zero, false
    }
    return items[len(items)-1], true
}
```

这比返回 `nil` 更通用，因为 T 可能是 `int`、`struct` 这类不能为 nil 的类型。

---

## 6. 泛型保留类型信息

```go
func Map[T any, U any](items []T, f func(T) U) []U {
    out := make([]U, 0, len(items))
    for _, item := range items {
        out = append(out, f(item))
    }
    return out
}
```

调用：

```go
lengths := Map([]string{"go", "rust"}, func(s string) int {
    return len(s)
})
```

`lengths` 的类型是 `[]int`。函数没有把数据降级成 `[]any`。

---

## 7. 泛型函数里只能使用约束允许的操作

```go
func PrintAll[T any](items []T) {
    for _, item := range items {
        fmt.Println(item)
    }
}
```

`fmt.Println` 接受 `any`，所以可以打印任意 T。

但不能随便做加法：

```go
func Add[T any](a, b T) T {
    // return a + b // 编译错误
}
```

只有约束说明 T 支持 `+` 时，才能在函数里使用 `+`。

---

## 8. 泛型不是为了消灭重复

下面这种重复不一定需要泛型：

```go
func SaveUser(user User) error {}
func SaveOrder(order Order) error {}
```

如果两个函数只是名字相似，但业务规则不同，不要硬抽成泛型。

泛型适合算法层面的重复，例如查找、过滤、映射、集合操作、容器类型。

---

## 9. 泛型调用时的类型推断限制

```go
func Zero[T any]() T {
    var zero T
    return zero
}
```

调用时没有参数可供推断，必须显式写类型：

```go
n := Zero[int]()
s := Zero[string]()
```

类型参数要么能从参数推断，要么由调用方显式提供。

---

## 10. 泛型和可读性

类型参数名通常短：

- `T`: 一个普通类型。
- `K`: key。
- `V`: value。
- `E`: element。

如果约束有明确业务含义，也可以用更具体的名字。

泛型签名一旦过长，就要警惕抽象是否过度。

---

## 练习

2. 实现 `Last[T any](items []T) (T, bool)`。
3. 实现 `Map[T, U any](items []T, f func(T) U) []U`。
4. 实现 `Filter[T any](items []T, keep func(T) bool) []T`。
5. 写一个 `Zero[T any]() T`，调用时显式传入 `int` 和 `string`。
