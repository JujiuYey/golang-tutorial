# 06: 泛型函数与类型参数

这篇解决"同一段逻辑,想给 `[]int` 用也想给 `[]string` 用,又不想复制粘贴、也不想丢类型"的问题。上上篇末尾试过 `[]any` 走不通,泛型才是正解。学完你能写出 `First`、`Map` 这类保留类型信息的通用函数。

先校准坐标：JS 天生"泛型"——函数本来就不检查参数类型,`first(arr)` 什么数组都能收。Go 的泛型是**在保住编译期类型检查的前提下**,换取这份灵活:函数多收一个"类型参数",让编译器帮每种类型各查一遍。

---

## 1. 一个没有泛型的问题

先感受痛点。"取第一个元素"这个逻辑,不用泛型只能按类型各写一份：

```go
func FirstInt(items []int) (int, bool) {
    if len(items) == 0 {
        return 0, false
    }
    return items[0], true
}

func FirstString(items []string) (string, bool) {
    if len(items) == 0 {
        return "", false
    }
    return items[0], true
}
```

两个函数逐字符对比,只有类型不同、逻辑一模一样。再来个 `[]User` 就得再抄一份——这种"同形算法"就是泛型的靶子。

---

## 2. 泛型函数

一份代码,类型留空,调用时再填：

```go
package main

import "fmt"

func First[T any](items []T) (T, bool) {
    var zero T
    if len(items) == 0 {
        return zero, false
    }
    return items[0], true
}

func main() {
    n, ok := First([]int{1, 2, 3})
    fmt.Println(n, ok)

    s, ok := First([]string{"go", "rust"})
    fmt.Println(s, ok)

    e, ok := First([]int{})
    fmt.Println(e, ok)
}
```

```text
1 true
go true
0 false
```

拆开这个新语法：

```text
func First[T any](items []T) (T, bool) {
//        ─┬────  ────┬────   ─┬─────
//         │          │        └ ③返回值也能用 T:进什么类型,出什么类型
//         │          └ ②参数用 T:元素类型为 T 的切片
//         └ ①方括号声明类型参数:T 是个"类型占位符",any 是它的约束(下一篇细讲)
```

注意调用处**没写** `First[int]`——编译器看见实参是 `[]int`,自动推断 `T = int`(类型推断)。用起来和 JS 的 `first(arr)` 手感几乎一样,但传 `[]int` 时返回值就是 `int`,类型没丢。

---

## 3. 显式类型实参

推断不出来或想写明白时,可以手动填方括号：

```go
n, ok := First[int]([]int{1, 2, 3})   // 显式:T = int
n, ok := First([]int{1, 2, 3})        // 推断:多数情况这样写
```

对比 TS：就是 `first<number>(arr)` 和 `first(arr)` 的关系,只是 Go 用方括号不用尖括号。规则也一样——**推断得出就省略,推断不出必须写**(什么情况推断不出,见第 9 节)。

---

## 4. 多个类型参数

方括号里可以列多个占位符：

```go
package main

import "fmt"

func Pair[A, B any](a A, b B) (A, B) {
    return a, b
}

func main() {
    name, age := Pair("alice", 18)
    fmt.Println(name, age)
    fmt.Printf("%T %T\n", name, age)
}
```

```text
alice 18
string int
```

`%T` 证明两个类型各自保住了:`name` 是 string、`age` 是 int。`[A, B any]` 是 `[A any, B any]` 的合并写法——约束相同时可以共享,和参数列表 `a, b int` 同一个规律。

---

## 5. 类型参数的零值

泛型函数里有个小难题:不知道 T 具体是啥,"空手而归"时该返回什么?不能写 `return nil`——T 可能是 int、struct 这类没有 nil 的类型。通用答案是 `var zero T`：

```go
func Last[T any](items []T) (T, bool) {
    var zero T                  // T 是 int → 0;string → "";指针 → nil
    if len(items) == 0 {
        return zero, false
    }
    return items[len(items)-1], true
}
```

还记得 02 模块的规则吗——**Go 里每个类型都有零值**。`var zero T` 就是让编译器按 T 的实际类型给出对应零值。第 2 节输出里空 `[]int` 返回的 `0 false`,那个 0 就是这么来的。

**一句话总结：泛型函数里的"空结果"用 `var zero T`,别想 nil。**

---

## 6. 泛型保留类型信息

泛型对比 `any` 的核心优势:**输入输出之间的类型关系保得住**。经典例子 `Map`(对,就是 JS 的 `arr.map(fn)`)：

```go
package main

import "fmt"

func Map[T any, U any](items []T, f func(T) U) []U {
    out := make([]U, 0, len(items))
    for _, item := range items {
        out = append(out, f(item))
    }
    return out
}

func main() {
    lengths := Map([]string{"go", "rust"}, func(s string) int {
        return len(s)
    })
    fmt.Println(lengths)
    fmt.Printf("%T\n", lengths)
}
```

```text
[2 4]
[]int
```

签名说得清清楚楚:收 `[]T` 和一个 `T→U` 的函数,还你 `[]U`。传 `[]string` + `string→int` 的函数,得到的就是真正的 `[]int`——不是 `[]any`,不用断言,直接能做算术。

---

## 7. 🕳️ 坑：函数体内只能做约束允许的操作

以为会怎样：T 反正是某个具体类型,加个法应该没问题。
实际怎样：编译错误。

```go
func Add[T any](a, b T) T {
    return a + b
}

func main() {
    _ = Add(1, 2)
}
```

```text
# mod05/a06_add_bad
./main.go:4:9: invalid operation: operator + not defined on a (variable of type T constrained by any)
```

为什么：编译器检查函数体时,T 还没定——它必须保证**约束范围内的每一种类型**都能跑这段代码。`any` 包括 struct、slice、func……这些都不支持 `+`,所以 `+` 被拒。报错原文说得很直白:`not defined on a (variable of type T constrained by any)`。

打印倒是没问题,因为 `fmt.Println` 收 `any`,谁都能进：

```go
func PrintAll[T any](items []T) {
    for _, item := range items {
        fmt.Println(item)   // OK
    }
}
```

想用 `+`、`<` 这些操作,得把约束收窄到"都支持该操作的类型"——正是下一篇的主题。

**一句话总结：约束是双向合同——限制调用方能传什么,也限制函数体能做什么。**

---

## 8. 泛型不是为了消灭重复

看到两个函数长得像就想抽泛型?先分辨重复的**种类**：

```go
func SaveUser(user User) error {}
func SaveOrder(order Order) error {}
```

这两个只是名字像——保存用户要查重邮箱,保存订单要校验库存,业务规则根本不同,硬抽成 `Save[T any]` 只会得到一个啥也做不了的空壳(别忘了第 7 节:约束 any 时函数体几乎啥都不能干)。

| 重复类型 | 例子 | 该用 |
|---|---|---|
| 算法同形,只差类型 | First、Filter、Map、Contains | 泛型 |
| 名字像,业务不同 | SaveUser、SaveOrder | 各写各的 |

---

## 9. 泛型调用时的类型推断限制

类型推断只看**参数**。参数里没出现 T,编译器就没线索：

```go
func Zero[T any]() T {
    var zero T
    return zero
}

func main() {
    n := Zero()
    _ = n
}
```

```text
# mod05/a06_zero_bad
./main.go:9:11: in call to Zero, cannot infer T (declared at ./main.go:3:11)
```

`cannot infer T`——参数列表是空的,推无可推。这时必须显式填：

```go
n := Zero[int]()
s := Zero[string]()
fmt.Printf("%v %q\n", n, s)
```

```text
0 ""
```

**规律：T 出现在参数里→可推断;只出现在返回值里→调用方必须显式写。**

---

## 10. 泛型和可读性

类型参数的命名惯例(和其他语言相通)：

- `T`: 一个普通类型
- `K` / `V`: map 的 key / value
- `E`: 元素(element)

约束有明确业务含义时也可以用长名字。反过来,如果一个函数签名里挤了四五个类型参数、读者要在方括号里解谜——警惕,大概率是抽象过度了,退回具体代码往往更好。

---

## 本篇重点

- [ ] `func First[T any](items []T)`:方括号声明类型参数,一份代码服务所有类型,且编译期检查、类型不丢。
- [ ] 调用时能从参数推断 T 就不用写;T 只出现在返回值时必须显式 `Zero[int]()`。
- [ ] 泛型函数里的"空结果"用 `var zero T`,因为 T 可能是没有 nil 的类型。
- [ ] 函数体只能做约束允许的操作:`any` 约束下连 `+` 都不行(报错 `not defined on ... constrained by any`)。
- [ ] 泛型抽"算法同形"的重复(Map/Filter/查找);名字像但业务不同的函数,别硬抽。
