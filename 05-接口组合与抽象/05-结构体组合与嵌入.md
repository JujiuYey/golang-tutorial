# 05: 结构体组合与嵌入

Go 没有类继承。复用代码和能力时，主要依靠组合。嵌入是组合的一种语法形式。

---

## 1. 普通组合

```go
type Logger struct{}

func (Logger) Log(msg string) {
    fmt.Println(msg)
}

type Service struct {
    logger Logger
}

func (s Service) Start() {
    s.logger.Log("start")
}
```

`Service` 拥有一个 `Logger` 字段。调用时通过字段访问。

---

## 2. 嵌入字段

```go
type Service struct {
    Logger
}

func (s Service) Start() {
    s.Log("start")
}
```

`Logger` 是嵌入字段。它没有显式字段名，字段名默认就是类型名。

```go
s := Service{Logger: Logger{}}
s.Logger.Log("start")
s.Log("start")
```

`s.Log` 是提升方法。

---

## 3. 嵌入不是继承

嵌入不会让 `Service` 成为 `Logger` 的子类。它只是把 `Logger` 作为字段放进 `Service`。

```go
func UseLogger(l Logger) {}

s := Service{Logger: Logger{}}

// UseLogger(s) // 编译错误
UseLogger(s.Logger)
```

`Service` 可以提升 `Logger` 的方法，但它本身不是 `Logger`。

---

## 4. 嵌入接口

struct 也可以嵌入接口字段。

```go
type Service struct {
    Logger
}
```

如果这里的 `Logger` 是接口：

```go
type Logger interface {
    Log(msg string)
}
```

那么创建 `Service` 时要提供具体实现：

```go
s := Service{
    Logger: ConsoleLogger{},
}

s.Log("start")
```

这种写法可以方便地注入能力，但也要小心 nil 接口字段。

---

## 5. 匿名字段的可见性

```go
type Config struct {
    Timeout time.Duration
}

type Client struct {
    Config
}
```

可以访问：

```go
c := Client{Config: Config{Timeout: time.Second}}
fmt.Println(c.Timeout)
```

如果多个嵌入字段都提供同名字段或方法，直接访问会产生歧义，需要写完整路径。

---

## 6. 方法提升与覆盖

```go
type Logger struct{}

func (Logger) Name() string {
    return "logger"
}

type Service struct {
    Logger
}

func (Service) Name() string {
    return "service"
}
```

外层类型定义了同名方法时，直接调用优先使用外层方法：

```go
s := Service{}
fmt.Println(s.Name())        // service
fmt.Println(s.Logger.Name()) // logger
```

---

## 7. 组合接口能力

```go
type Handler struct {
    Store  UserStore
    Logger Logger
}

func (h Handler) Serve(id int64) error {
    user, err := h.Store.FindUser(id)
    if err != nil {
        return err
    }
    h.Logger.Log("found " + user.Name)
    return nil
}
```

这里的 `Handler` 组合了两个能力：查用户和写日志。每个能力都可以独立替换。

---

## 8. 组合优先于继承式思维

遇到“这个类型是什么”的问题时，先问“这个类型需要哪些能力”。

例如 HTTP 服务不需要继承数据库，它只需要拥有一个能查询数据的 Store。

```go
type Server struct {
    Store Store
}
```

组合让依赖更清晰，也让测试更容易。

---

## 9. 嵌入的使用边界

适合嵌入：

- 嵌入字段的方法确实应该成为外层类型的公开能力。
- 外层类型和嵌入能力关系稳定。
- 提升字段或方法能明显减少重复样板。

优先普通字段：

- 只是内部实现细节。
- 不希望把内部能力暴露成外层 API。
- 嵌入后方法来源会变得不清楚。

---

## 10. 组合和接口的关系

组合通常用于构造具体类型，接口用于描述调用边界。

```go
type Service struct {
    Store  Store
    Logger Logger
}

func NewService(store Store, logger Logger) Service {
    return Service{Store: store, Logger: logger}
}
```

`Service` 通过组合获得能力，调用它的代码可以依赖更小的接口。

2. 验证嵌入后可以调用提升方法。
3. 写代码证明嵌入不是继承：外层类型不能当成内层类型传参。
4. 定义两个嵌入字段，让它们有同名方法，观察直接调用时的歧义。
5. 把一个结构体中的嵌入字段改成普通字段，比较 API 暴露范围。
