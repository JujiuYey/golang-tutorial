# 08: 泛型类型、方法与零值

除了泛型函数，Go 也支持泛型类型。常见用途是集合、栈、缓存、结果包装等通用数据结构。

---

## 1. 泛型类型

```go
type Stack[T any] struct {
    items []T
}
```

`Stack[T]` 表示一个元素类型为 T 的栈。

创建：

```go
var s Stack[int]
s.Push(1)
s.Push(2)
```

---

## 2. 泛型类型的方法

```go
func (s *Stack[T]) Push(v T) {
    s.items = append(s.items, v)
}

func (s *Stack[T]) Pop() (T, bool) {
    var zero T
    if len(s.items) == 0 {
        return zero, false
    }

    last := len(s.items) - 1
    v := s.items[last]
    s.items = s.items[:last]
    return v, true
}
```

方法接收者也要带上类型参数 `T`。

---

## 3. 零值可用

```go
var s Stack[string]

s.Push("go")
v, ok := s.Pop()
```

`Stack` 的零值可以直接使用，因为 `items` 的零值 nil slice 可以 append。

设计类型时，尽量让零值可用。这样调用方可以少写构造代码。

---

## 4. 构造函数

如果需要预分配容量，可以提供构造函数：

```go
func NewStack[T any](capHint int) Stack[T] {
    return Stack[T]{
        items: make([]T, 0, capHint),
    }
}
```

调用：

```go
s := NewStack[int](10)
```

类型参数通常可以从返回值以外的参数推断。这里 `capHint` 不是 T，所以调用方需要写 `NewStack[int](10)`。

---

## 5. 泛型类型不能在方法上单独声明新类型参数

Go 的方法不能拥有接收者类型参数之外的额外类型参数。

```go
type Box[T any] struct {
    Value T
}

// func (b Box[T]) Map[U any](f func(T) U) Box[U] { ... } // 不允许
```

可以把这种逻辑写成普通泛型函数：

```go
func MapBox[T, U any](b Box[T], f func(T) U) Box[U] {
    return Box[U]{Value: f(b.Value)}
}
```

---

## 6. 泛型 set

```go
type Set[T comparable] struct {
    items map[T]struct{}
}
```

因为 map key 必须可比较，所以 T 需要 `comparable`。

```go
func NewSet[T comparable]() Set[T] {
    return Set[T]{items: make(map[T]struct{})}
}

func (s Set[T]) Add(v T) {
    s.items[v] = struct{}{}
}

func (s Set[T]) Has(v T) bool {
    _, ok := s.items[v]
    return ok
}
```

这里 `Set` 的零值不能直接 `Add`，因为 nil map 不能写入。可以让 `Add` 使用指针接收者并延迟初始化：

```go
func (s *Set[T]) Add(v T) {
    if s.items == nil {
        s.items = make(map[T]struct{})
    }
    s.items[v] = struct{}{}
}
```

---

## 7. 泛型结果类型

```go
type Result[T any] struct {
    Value T
    Err   error
}
```

这种类型可以表达“一个值或一个错误”。

```go
func Ok[T any](v T) Result[T] {
    return Result[T]{Value: v}
}

func Err[T any](err error) Result[T] {
    return Result[T]{Err: err}
}
```

Go 里普通多返回值已经很好用。只有在需要把结果放进 channel、slice 或结构体时，这类包装才更有价值。

---

## 8. 类型别名和泛型

可以为实例化后的泛型类型定义别名：

```go
type IntStack = Stack[int]
```

也可以定义新的命名类型：

```go
type UserIDSet Set[int64]
```

别名保留原类型的方法集合。新的命名类型是独立类型，不会自动拥有原类型的方法，通常需要重新定义方法。

---

## 9. 泛型类型的可读性

泛型类型不要把所有变化都塞进类型参数。

```go
type Cache[K comparable, V any] struct {
    items map[K]V
}
```

这是清晰的。

如果类型参数变成四五个，而且名字很抽象，调用方会很难读懂。优先保持数据结构小而明确。

---

## 10. 泛型类型与接口字段

泛型类型内部也可以组合接口：

```go
type Cache[K comparable, V any] struct {
    items  map[K]V
    logger Logger
}
```

泛型解决值类型的通用性，接口解决行为依赖。它们可以一起使用。

---

## 练习

1. 实现 `Stack[T]`，包含 `Push`、`Pop`、`Len`。
2. 让 `Stack[T]` 的零值可用。
3. 实现 `Set[T comparable]`，包含 `Add`、`Has`、`Delete`、`Len`。
4. 修改 `Set`，让零值也可以直接 `Add`。
5. 实现 `Cache[K comparable, V any]`，支持 `Set`、`Get`、`Delete`。
