# 08: 泛型类型、方法与零值

函数能带类型参数,类型也能。这篇解决"怎么写 `Stack[int]`、`Set[string]` 这类通用容器"的问题——TS 里你写过 `Array<number>`、`Map<string, User>`,现在轮到自己造这种容器了。学完你还会掌握 Go 特有的"零值可用"设计,以及 nil map 在泛型 Set 里埋的雷。

---

## 1. 泛型类型

在类型名后面挂方括号,struct 字段就能用类型参数：

```go
type Stack[T any] struct {
    items []T
}
```

拆开看：

```text
type Stack[T any] struct {
//   ──┬── ──┬───
//     │     └ ②类型参数和约束,语法和泛型函数一样
//     └ ①类型名:使用时要写全,如 Stack[int]、Stack[string]
    items []T
//        ─┬─
//         └ ③字段类型可以用 T
}
```

使用时必须把 T 填上——`Stack[int]` 和 `Stack[string]` 是两个互不相干的类型：

```go
var s Stack[int]
s.Push(1)
s.Push(2)
```

对比 TS：`Stack[int]` ≈ `Stack<number>`,方括号换尖括号,概念一一对应。

---

## 2. 泛型类型的方法

给泛型类型定义方法,接收者要带上类型参数：

```go
package main

import "fmt"

type Stack[T any] struct {
    items []T
}

func (s *Stack[T]) Push(v T) {
    s.items = append(s.items, v)
}

func (s *Stack[T]) Pop() (T, bool) {
    var zero T
    if len(s.items) == 0 {
        return zero, false
    }
    last := len(s.items) - 1
    v := s.items[last]
    s.items = s.items[:last]
    return v, true
}

func main() {
    var s Stack[string]
    s.Push("go")
    s.Push("rust")

    v, ok := s.Pop()
    fmt.Println(v, ok)

    s.Pop()
    v2, ok2 := s.Pop()
    fmt.Printf("%q %v\n", v2, ok2)
}
```

```text
rust true
"" false
```

注意接收者写法 `(s *Stack[T])`——`[T]` 不能省,方法体里才能用 T。栈空时 `Pop` 返回 `var zero T`(上上篇的零值技巧),`Stack[string]` 的空结果自然就是 `""`。

**一句话总结：泛型方法 = 普通方法 + 接收者带 `[T]`。**

---

## 3. 零值可用

上面 `main` 里有个容易错过的细节:`var s Stack[string]` **没有任何初始化**就直接 `Push` 了,而且没炸。

为什么能行:`Stack` 唯一的字段是 slice,而 nil slice 可以直接 `append`(04 模块的老知识)。零值状态就是合法的空栈。

这是 Go 的一个招牌设计目标——**让类型的零值开箱即用**(标准库的 `bytes.Buffer`、`sync.Mutex` 都这样),调用方少写构造代码。但不是所有字段都这么配合……第 6 节的 map 就会翻车。

---

## 4. 构造函数

零值可用不代表不需要构造函数——想预分配容量时还是得有：

```go
func NewStack[T any](capHint int) Stack[T] {
    return Stack[T]{
        items: make([]T, 0, capHint),
    }
}
```

🕳️ 坑：调用它必须写 `NewStack[int](10)`,省略类型参数编译不过：

```go
func main() {
    s := NewStack(10)
    _ = s
}
```

```text
# mod05/a08_newstack_bad
./main.go:12:15: in call to NewStack, cannot infer T (declared at ./main.go:7:15)
```

上上篇的推断规则在这里重演:推断只看参数,`capHint` 是 int、和 T 无关,T 只出现在返回值里——没线索,必须显式写。**泛型容器的构造函数,十有八九要手写类型参数。**

---

## 5. 方法不能自带新的类型参数

Go 泛型有一条边界:方法只能用接收者带来的类型参数,**不能自己再声明新的**。想给 `Box[T]` 写一个变换到 `Box[U]` 的方法:

```go
type Box[T any] struct {
    Value T
}

func (b Box[T]) Map[U any](f func(T) U) Box[U] {
    return Box[U]{Value: f(b.Value)}
}
```

```text
# mod05/a08_method_bad
./main.go:7:20: syntax error: method must have no type parameters
```

`method must have no type parameters`——直接语法错误。这在 TS/Java 里都是合法操作(方法自己带 `<U>`),Go 明确不支持。

出路是把它写成普通泛型函数,容器当第一个参数：

```go
func MapBox[T, U any](b Box[T], f func(T) U) Box[U] {
    return Box[U]{Value: f(b.Value)}
}
```

所以 Go 没有 `list.Map(f).Filter(g)` 这种链式泛型 API,标准库 `slices` 包全是 `slices.Contains(list, v)`、`slices.SortFunc(list, cmp)` 这种"容器当参数"的风格——就是被这条规则塑造的。

## 6. 泛型 Set 与 nil map 的雷

Set(去重集合)是最常写的泛型容器。key 要用 `==` 去重,所以 T 必须 `comparable`：

```go
type Set[T comparable] struct {
    items map[T]struct{}
}
```

(`map[T]struct{}` 是 Go 的惯用"只要 key 不要 value"写法,`struct{}` 零内存占用。)

### 🕳️ 坑：Set 的零值不可用

以为会怎样：Stack 零值能用(第 3 节),Set 应该也行。
实际怎样：panic。

```go
func (s Set[T]) Add(v T) {
    s.items[v] = struct{}{}
}

func main() {
    var s Set[string]
    s.Add("go")
}
```

```text
panic: assignment to entry in nil map

goroutine 1 [running]:
main.Set[...].Add(...)
	/.../main.go:8
main.main()
	/.../main.go:13 +0x38
exit status 2
```

为什么：04 模块的规则——nil slice 能 append,**nil map 不能写入**。零值 Set 里的 map 是 nil,一 `Add` 就撞上 `assignment to entry in nil map`。

### 修法：指针接收者 + 延迟初始化

```go
package main

import "fmt"

type Set[T comparable] struct {
    items map[T]struct{}
}

func (s *Set[T]) Add(v T) {
    if s.items == nil {
        s.items = make(map[T]struct{})   // 第一次用时才建 map
    }
    s.items[v] = struct{}{}
}

func (s *Set[T]) Has(v T) bool {
    _, ok := s.items[v]                  // nil map 读不炸,返回零值
    return ok
}

func main() {
    var s Set[string]
    s.Add("go")
    fmt.Println(s.Has("go"), s.Has("rust"))
}
```

```text
true false
```

两个要点:① `Add` 必须换成**指针接收者**,值接收者改的是副本的字段,初始化会丢;② `Has` 不用判 nil——nil map **读**是安全的,返回零值。

| 字段类型 | 零值行为 | 零值可用? |
|---|---|---|
| slice | nil 可 append、可 range | ✅ |
| map | nil 可读,**写入 panic** | ❌ 需延迟初始化或构造函数 |

## 7. 泛型结果类型

泛型类型不只用于容器,也能做"包装"。比如把"值或错误"打包成一个值：

```go
package main

import (
    "errors"
    "fmt"
)

type Result[T any] struct {
    Value T
    Err   error
}

func Ok[T any](v T) Result[T] {
    return Result[T]{Value: v}
}

func Err[T any](err error) Result[T] {
    return Result[T]{Err: err}
}

func main() {
    results := []Result[int]{
        Ok(42),
        Err[int](errors.New("boom")),
    }
    for _, r := range results {
        fmt.Println(r.Value, r.Err)
    }
}
```

```text
42 <nil>
0 boom
```

克制提醒:Go 的多返回值 `(T, error)` 已经很好用,别为了像 Rust 而处处 Result。它的真实价值是**当结果需要装进 slice、channel、map 时**——多返回值塞不进容器,包成一个值就行(上面的 `[]Result[int]` 正是例证)。

## 8. 类型别名和泛型

实例化后的泛型类型可以起短名,两种起法效果不同：

```go
type IntStack = Stack[int]     // 别名:有 = 号,就是同一个类型,方法全在
type UserIDSet Set[int64]      // 新命名类型:没有 = 号,是独立新类型
```

02 模块讲过这个区别,泛型这儿要格外小心:**新命名类型不继承原类型的方法**——`UserIDSet` 上没有 `Add`、`Has`,得自己重新定义。想要纯粹的简写,用带 `=` 的别名。

---

## 9. 泛型类型的可读性

类型参数数量是可读性的硬指标：

```go
type Cache[K comparable, V any] struct {   // 两个参数,清晰
    items map[K]V
}
```

到四五个类型参数、名字还都是单字母时,调用方读签名像解密。数据结构优先保持小而明确——参数多到读不懂,就该拆类型或退回具体类型了。

---

## 10. 泛型类型与接口字段

收尾把本模块两条线接上:泛型类型内部照样可以组合接口字段：

```go
type Cache[K comparable, V any] struct {
    items  map[K]V
    logger Logger      // 接口:可替换的行为依赖
}
```

- **泛型**解决"装什么类型的数据"——编译期定,类型安全。
- **接口**解决"依赖什么行为"——运行时可换,方便测试。

两者不打架,各管一段。什么时候用谁,正是下一篇的主题。

---

## 本篇重点

- [ ] 泛型类型 `Stack[T any]` ≈ TS 的 `Stack<T>`;方法接收者要带 `[T]`,空结果用 `var zero T`。
- [ ] 构造函数的 T 只出现在返回值时,调用必须显式 `NewStack[int](10)`(`cannot infer T`)。
- [ ] 方法不能自带新类型参数(`method must have no type parameters`)——变换逻辑写成普通泛型函数。
- [ ] 零值可用是好设计:slice 字段天生配合;map 字段零值写入 panic,用指针接收者 + 延迟初始化补救。
- [ ] Set 的 T 用 `comparable`(map key 要求);泛型管数据类型,接口管行为依赖,可同时使用。
